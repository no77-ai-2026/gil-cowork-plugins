# 표준 운영 절차 작성 (84-sop-writer)

> MoAI-Cowork v1.0.0 하네스 레퍼런스

## 개요

프로세스 분석, 절차 설계, 문서화, 검증을 수행하는 에이전트 팀 협업 하네스입니다.

## 전문가 역할

- **프로세스 분석가**: 현재 프로세스를 분석합니다.
  - 구조: 프로세스 흐름 파악
  - 단계: 각 단계별 활동 파악
  - 책임: 단계별 책임 정의
  - 입출력: 입력과 산출물 정의
  - 문제: 현재 문제점 파악

- **절차 설계자**: 개선된 절차를 설계합니다.
  - 단계: 최적의 단계 구성
  - 순서: 논리적 순서 결정
  - 기준: 각 단계의 명확한 기준
  - 점검: 품질 점검 포인트
  - 예외: 예외 상황 처리 방안

- **문서 작성자**: SOP 문서를 작성합니다.
  - 개요: 프로세스 개요
  - 단계: 각 단계별 상세 기술
  - 양식: 필요한 양식 포함
  - 다이어그램: 프로세스 다이어그램
  - 예시: 구체적 예시 포함

- **검증자**: SOP의 완전성을 검증합니다.
  - 명확성: 표현의 명확성
  - 완전성: 필요한 모든 내용 포함
  - 정확성: 기술의 정확성
  - 실행가능성: 실제 실행 가능성
  - 일관성: 문서 간 일관성

## 워크플로우

### 단계 1: 준비

1. 사용자 입력에서 추출:
   - **프로세스**: 프로세스명
   - **범위**: 문서화 범위
   - **현황**: 현재 절차
   - **문제**: 개선할 점
   - **사용자**: 주요 사용자

2. `_workspace/` 디렉토리 생성

### 단계 2: 팀 구성 및 실행

| 순서 | 작업 | 담당 | 의존성 | 산출물 |
|------|------|------|--------|--------|
| 1 | 분석 | 분석가 | 없음 | `01_process_analysis.md` |
| 2 | 설계 | 설계자 | 작업 1 | `02_improved_procedure.md` |
| 3 | 작성 | 작성자 | 작업 2 | `03_sop_document.md` |
| 4 | 검증 | 검증자 | 작업 3 | `04_verification_report.md` |

## 산출물

- `01_process_analysis.md` — 프로세스 분석
- `02_improved_procedure.md` — 개선 절차
- `03_sop_document.md` — SOP 문서
- `04_verification_report.md` — 검증 보고서

## 오류 처리

| 오류 유형 | 대응 전략 |
|---------|---------|
| 프로세스 복잡 | 단계별 모듈화 |
| 정보 부족 | 현장 조사 권고 |
| 에이전트 실패 | 1회 재시도 → 진행 |
