# 운영 매뉴얼 (92-operations-manual)

> MoAI-Cowork v1.0.0 하네스 참고 자료

## 개요
기존 문서/코드를 분석하여 업무 프로세스, 단계별 매뉴얼, FAQ, 교육 자료를 자동 생성하는 하네스입니다.

## 전문가 역할

- **문서 분석가**: 기존 문서, 코드, 위키 분석 전문가. 조직의 업무 프로세스를 추출하고 암묵지를 명시지로 변환
  - 출처 인벤토리: 분석 대상 문서/코드 목록 및 신뢰성 평가
  - 프로세스 추출: 문서와 코드에서 업무 절차, 의사결정점, 예외 처리 파악
  - 암묵지 발굴: 코드 주석, 커밋 로그, 설정 파일에서 미문서화된 지식 추출
  - 용어사전 작성: 도메인 용어, 약자, 내부 용어 정의
  - 갭 분석: 문서화된 절차와 실제 코드/설정 간 불일치 파악

- **FAQ 개발자**: FAQ 및 문제 해결 가이드 전문가. 업무 실행 중 발생할 질문과 문제를 예상하고 FAQ, 의사결정 트리, 에스컬레이션 가이드 작성
  - FAQ 작성: 빈번한 질문을 카테고리별로 정리하고 명확한 답변 작성
  - 문제 해결 가이드: 증상→원인→해결 구조의 문제 해결 가이드 작성
  - 의사결정 트리: 복잡한 판단이 필요한 상황을 Mermaid 다이어그램으로 표현
  - 에스컬레이션 매트릭스: 자체 해결 불가 사항의 연락처 및 절차 정리
  - 용어 설명: 매뉴얼의 기술 용어를 전문가가 아닌 관점에서 재설명

- **플로우차트 설계자**: 업무 프로세스 시각화 전문가. Mermaid 다이어그램으로 분기 로직, 병렬 처리, 예외 흐름을 포함한 완전한 프로세스 맵 작성
  - 프로세스 맵 설계: 전체 업무 흐름을 고수준으로 시각화
  - 상세 플로우차트: 개별 단계, 분기, 반복을 Mermaid 플로우차트로 표현
  - RACI 매트릭스: 각 단계의 담당자(R), 책임자(A), 상담 대상(C), 보고 대상(I) 정의
  - 예외 흐름 설계: 오류, 롤백, 에스컬레이션 경로를 정상 흐름과 분리해 표현
  - 시퀀스 다이어그램: 시스템 간, 팀 간 상호작용이 필요한 프로세스에 시퀀스 다이어그램 추가

- **매뉴얼 작성자**: 단계별 운영 매뉴얼 작성 전문가. 누구나 따를 수 있는 절차, 체크리스트, 스크린샷 가이드 작성
  - 단계별 절차: 각 프로세스를 실행 가능한 단계로 분해하고 순서대로 설명
  - 사전 요구사항: 각 절차 전 필요한 권한, 도구, 사전 지식 명시
  - 체크리스트 생성: 각 절차별 완료 검증 체크리스트 제공
  - 스크린샷 가이드: UI 조작이 필요한 단계에 스크린샷 삽입 지점 표시
  - 버전 관리: 매뉴얼 버전, 작성일, 검토 주기 관리

- **교육 개발자**: 교육 자료 개발 전문가. 매뉴얼을 학습 목표 기반 교육 자료로 변환하여 퀴즈, 실습, 요약 카드, 온보딩 체크리스트 생성
  - 학습 목표 설계: Bloom 분류법에 따라 지식/이해/적용/분석 수준의 목표 정의
  - 요약 카드 제작: 핵심 프로세스를 1장 요약 카드(치트시트)로 압축
  - 퀴즈 작성: 객관식, 시나리오 기반 퀴즈로 이해도 확인
  - 실습 설계: 실제 업무 환경을 모뭐하는 실용적 시나리오 설계
  - 온보딩 체크리스트: 신입이 점진적으로 학습할 로드맵 작성

## 워크플로우

### Phase 1: 준비 (오케스트레이터 직접 수행)

1. 사용자 입력에서 추출:
   - **대상 프로세스**: 문서화 범위
   - **소스 자료**: 기존 문서 경로, 코드 저장소, 위키 URL
   - **대상 사용자**: 신입/경력/관리자 - 매뉴얼 사용자
   - **제약사항** (선택): 특정 형식, 기존 템플릿

2. 프로젝트 루트에 `_workspace/` 디렉토리 생성
3. 입력값을 정리하여 `_workspace/00_input.md`에 저장
4. 기존 매뉴얼이 있으면 복사 후 수정 모드로 전환
5. 요청 범위에 따라 **실행 모드 결정**

### Phase 2: 팀 구성 및 실행

| 순서 | 작업 | 담당 | 의존성 | 산출물 |
|------|------|------|--------|--------|
| 1 | 문서/코드 분석 | 분석가 | 없음 | `_workspace/01_document_analysis.md` |
| 2 | 프로세스 플로우차트 | 설계자 | 작업 1 | `_workspace/02_process_flowcharts.md` |
| 3 | 운영 매뉴얼 | 작성자 | 작업 1, 2 | `_workspace/03_operations_manual.md` |
| 4 | FAQ 및 문제 해결 | FAQ 개발자 | 작업 1, 3 | `_workspace/04_faq_troubleshooting.md` |
| 5 | 교육 자료 | 교육 개발자 | 작업 3 | `_workspace/05_training_materials.md` |

작업 2(플로우차트)와 4(FAQ)는 **병렬 실행** 가능합니다.

**팀 간 협업 흐름:**
- 분석가 완료 → 설계자에 프로세스 목록, 작성자에 절차 상세, FAQ 개발자에 예외사항 전달
- 설계자 + 작성자 완료 → 교육 개발자에 매뉴얼 및 학습 목표 전달
- FAQ 개발자 완료 → 최종 매뉴얼에 통합
- 모든 산출물 검증 후 사용자에 보고

### Phase 3: 통합 및 최종 산출

1. `_workspace/` 모든 파일 검증
2. 매뉴얼, FAQ, 교육 자료 간 일관성 확인
3. 최종 요약 사용자에 보고

## 산출물

모든 산출물은 `_workspace/`에 저장됩니다:
- `00_input.md` — 사용자 입력 정리
- `01_document_analysis.md` — 문서/코드 분석 결과
- `02_process_flowcharts.md` — 프로세스 플로우차트 및 RACI 매트릭스
- `03_operations_manual.md` — 단계별 운영 매뉴얼
- `04_faq_troubleshooting.md` — FAQ 및 문제 해결 가이드
- `05_training_materials.md` — 교육 자료 및 퀴즈

## 오류 처리

| 오류 유형 | 대응 전략 |
|----------|----------|
| 소스 자료 불충분 | 분석가가 일반 템플릿 및 샘플 프로세스 제공 |
| 대상 사용자 불명확 | 3가지 버전(신입/경력/관리자) 병렬 제작 |
| 코드 접근 불가 | 기존 문서 기반으로 진행, "[코드 미분석]" 표시 |
| 프로세스 불명확 | 분석가가 대상 프로세스 상세 인터뷰 요청 |
| 플로우차트 복잡 | 상세 레벨별로 단계적으로 공개 |
