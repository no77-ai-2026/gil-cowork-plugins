# 프로세스 관리자 (process-manager)

## 전략 가이드

| ID | 한국명 | 설명 |
|----|--------|------|
| operations-manual | 운영 매뉴얼 | 업무 프로세스, 운영 규정, 워크플로우 설계 |
| procurement-docs | 조달 문서 | 구매 요청서, 발주서, 벤더 평가 |
| sop-writer | SOP 작성 | 표준 운영 절차, 매뉴얼, 가이드라인 |
| meeting-strategist | 회의 전략가 | 회의록 작성, 안건 관리, 후속 조치 |

→ 참조 파일: `references/{id}.md`

## 실행 규칙

1. 사용자 요청 수신 → 해당 전략 가이드 판별
2. `references/{id}.md` 로드 → 전략 가이드에 따라 실행
3. `--deepthink` 또는 복잡 프로세스 설계 → `mcp__sequential-thinking__sequentialthinking` 호출
4. 결과물 생성 후 사용자 검토 요청

## 트리거 키워드

프로세스, 운영 매뉴얼, 업무 절차, SOP, 조달, 구매, 발주, 회의록, 안건, 워크플로우

## 사용 예시

- "신규 직원 온보딩 SOP를 작성해줘"
- "IT 장비 구매 요청서 양식을 만들어줘"
- "오늘 회의 내용을 회의록으로 정리해줘"
- "영업팀 월간 운영 매뉴얼을 설계해줘"
- "소프트웨어 구매를 위한 벤더 평가표를 만들어줘"

## 독립 실행 워크플로우

전략 가이드 파일 없이도 다음 단계로 실행 가능합니다:

1. **요청 파악**: 문서 유형(SOP/회의록/조달/매뉴얼) 및 대상 업무 확인
2. **구조 설계**: 목적, 적용 범위, 담당자, 단계별 절차 초안 작성
3. **내용 작성**: 각 절차를 구체적인 액션 단위로 기술 (담당자·도구·산출물 명시)
4. **검토 포인트 제시**: 법적 준수 사항, 결재 라인, 비상 대응 절차 포함 여부 확인
5. **최종 확인**: 사용자 검토 후 최종본 제공

## 문제 해결

| 상황 | 해결 방법 |
|------|-----------|
| 결재 라인 정보 부족 | 조직도 또는 결재 권한 범위를 알려주시면 맞춤 작성 가능합니다 |
| 조달 규정 확인 불가 | 일반적인 내부 구매 기준으로 작성 후 담당 부서 검토를 권장합니다 |
| 기존 프로세스와 충돌 | 현행 절차 문서를 공유해 주시면 개선안으로 작성해 드립니다 |
| 업종·도메인 특화 요구 | 해당 업종 규정이나 컴플라이언스 기준을 함께 제공해 주세요 |
