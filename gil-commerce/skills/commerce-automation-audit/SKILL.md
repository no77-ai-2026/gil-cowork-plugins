# 커머스 자동화 진단·로드맵 (Commerce Automation Audit)

## 개요

이커머스 셀러가 자기 매장의 운영 자동화 가능성을 자체 진단하고 우선순위·로드맵을 자동 생성하는 컨설팅용 스킬. "커머스 업무 자동화" 24p 풀세트를 wrapping.

**책임 한 줄**: 셀러 현황 입력 → 6대 영역 진단 + 자동화 우선순위 점수 + 3 Phase 로드맵 + 5 KPI 목표치 + HITL 검수 지점 자동 생성.

## 6대 커머스 업무 영역

| 영역 | 질문 | 주요 업무 | ROI |
|------|------|----------|-----|
| **A** 상품 운영 | "무엇을 팔 것인가" | 상품 소싱·등록·이미지·상세페이지·카테고리 | 중 |
| **B** 가격 & 프로모션 | "얼마에, 어떻게 팔 것인가" | 원가 분석·마진 계산·할인 정책·쿠폰·이벤트 | 중 |
| **C** 주문 & 정산 | "주문 처리와 대금 수취" | 주문 수집·송장 발행·취소/반품/교환·정산·매칭 | **🏆 최고 ROI (30-50% 절감)** |
| **D** 재고 & 물류 | "재고를 어디에 얼마나 둘 것인가" | 실시간 재고·안전재고·발주·입출고·배송사 연동 | 중 |
| **E** 마케팅 & 고객 | "고객을 데려오고 유지" | 광고 집행·퍼포먼스 마케팅·CRM·문의·리뷰 | 중 |
| **F** 데이터 & 경영 | "잘 되고 있는지 판단" | KPI 모니터링·손익·성과 분석·전략 의사결정 | 높음 |

## 자동화 유형 3분류

| 유형 | 정의 | 자동화 전략 | 도구 예시 |
|------|------|------------|----------|
| **① 반복형** (Repetitive) | 사람이 같은 일을 반복 — 데이터 입력·복사·정기 리포트 | **RPA / Workflow** | Zapier, Make, UiPath, Power Automate |
| **② 판단형** (Judgment) | 규칙·기준으로 결정 — 카테고리 분류·이상 거래·발주량 | **Rule Engine + AI** | Drools, AWS SageMaker, Custom Python |
| **③ 창의형** (Creative) | 기획·전략·카피 — 이벤트 기획·상세페이지 문구 | **AI Copilot** | ChatGPT, Claude, GPT-4 |

## 자동화 설계 4단계 프로세스

```
[STEP 1] 업무 나열 (List Up)
  - 모든 업무를 표준 포맷으로 나열
  - 업무명 / 주기 / 소요시간 / 입력(Input) / 출력(Output) / 사람이 개입하는 이유
  ↓
[STEP 2] 유형 분류 (Categorize)
  - 나열된 업무를 3가지로 분류
  - ① 반복형 ② 판단형 ③ 창의형
  ↓
[STEP 3] 우선순위 점수 (Scoring)
  - ROI 기반 점수 = (빈도 × 시간 × 오류비용) ÷ 복잡도
  - 예외 발생 비율이 높은 업무는 '복잡도' 점수를 높게 설정
  ↓
[STEP 4] 방식 결정 (Decision)
  - High: RPA / API 연동 즉시 적용
  - Mid: Rule Engine + AI 검토 후 진행
  - Low: AI Copilot 보조만 (완전 자동화 어려운 창의형)
```

## 자동화 준비도 체크리스트 (HARD)

다음 5종 중 미달 항목이 있으면 자동화 도입 전 선행 작업 필수:

| 점검 항목 | 준비도 기준 | 미달 시 선행 작업 |
|----------|-----------|------------------|
| **데이터 구조화** | 주문/상품/재고 DB 정규화 완료 | ERD 재설계 + 마이그레이션 |
| **API 연동 가능성** | 주요 쇼핑몰 API 키 확보 | 파트너십 협의 or 크롤링 대체 |
| **프로세스 표준화** | 동일 업무의 처리 방식 통일 | SOP 문서화 + 교육 |
| **권한 체계** | 시스템 간 인증 자동화 가능 | SSO 구축 or API 토큰 관리 |
| **예외 처리 정책** | 자동화 실패 시 대응 매뉴얼 | Fallback 프로세스 설계 |

> **Pro Tip**: "엑셀 → 쇼핑몰 등록" 자동화 전에 **엑셀 템플릿 표준화가 80% 더 중요**. 부서마다 다른 엑셀 양식을 통일하는 게 자동화 성공의 핵심.

## 영역별 최소 자동화 세트 ("Core 6")

전부 자동화하기보다 핵심 6가지만 확실히 구축:

| 영역 | Core Automation |
|------|----------------|
| A 상품 운영 | 대량 상품 등록 API |
| B 가격 & 프로모션 | 마진율 자동 계산기 (`commerce-margin-calculator` 연계) |
| **C 주문 & 정산** | **주문 통합 + 송장 자동 등록** (가장 먼저) |
| D 재고 & 물류 | 실시간 재고 동기화 |
| E 마케팅 & 고객 | 광고 리포트 자동 집계 (`coupang-ad-optimizer` 연계) |
| F 데이터 & 경영 | 일매출 대시보드 자동 갱신 (`commerce-morning-brief` 연계) |

> **Insight**: 100% 자동화를 목표로 하기보다, 이 6개 항목만 자동화해도 업무 효율은 3배 이상 증가.

## HITL (Human-in-the-Loop) 검수 지점 설계

### Golden Rule
완전 자동화(100%)가 아닌 **"자동화율 80% + 검수 속도 10배"**를 목표로 설정.

### 필수 검수 지점 (사람이 판단해야 함)
- 이미지 품질 및 법적 문구 (저작권·과대광고)
- 대폭 할인 가격 승인 (0원 누락 등 비정상)
- 고액 환불 케이스 (예: 50만원 이상)
- 시즌성 상품 발주 (과거 데이터로 예측 불가)

### 시스템 가드레일 (오류 확산 차단)
- **임계값 경고 (Threshold Alert)**: "전일 대비 주문량 500% 급증" 시 자동화 일시 정지
- **이중 승인 (Dual Approval)**: 핵심 설정(마진율 로직) 변경 시 2인 이상 관리자 승인
- **자동 롤백 및 재시도**: API 연동 실패 시 즉시 이전 상태 복구, 3회 재시도
- **감사 로그 (Audit Log)**: 모든 자동화 수행 내역과 사람의 개입/수정 이력 영구 보존

## 조직 규모별 자동화 전략

| Stage | 월 주문 | 핵심 전략 | 추천 도구 | 목표 |
|-------|--------|----------|----------|------|
| **Stage 1** 초기 스타트업 | <1,000건 | "반복형 업무만 집중, 판단형은 수동" | Zapier/Make, Google Sheets+Apps Script, 무료 SaaS | 생존 및 PMF 검증 |
| **Stage 2** 성장기 | 1,000-10,000건 | "반복형 완전 자동화 + Rule 엔진 도입" | 커머스 ERP(사방넷 등), Python 커스텀, RPA | 운영 효율 & 인력 최적화 |
| **Stage 3** 스케일업 | >10,000건 | "전 영역 자동화 + AI 최적화/내재화" | In-house 자체 시스템, MLOps, Data Warehouse + BI | 초격차 경쟁력 & 의사결정 속도 |

## 자동화 효과 측정 5대 KPI

| 지표 | 목표치 (Year 1) | 측정 방법 |
|------|---------------|----------|
| **시간 절감률** | 70%+ | 자동화 전후 처리 시간 비교 |
| **오류 감소율** | 90%+ | 휴먼 에러 발생 건수 비교 |
| **처리 속도** | 1/10 | 주문 100건당 소요 시간 (예: 60분 → 6분) |
| **ROI** | 280% | 시스템 구축 비용 대비 절감액 |
| **인당 처리량** | 3.0x | 동일 인력으로 3배 물량 소화 |

## 3 Phase 로드맵

```
[Phase 1] Quick Wins (1~2개월)
  목표: 반복 업무 즉시 제거 & ROI 확인
  주문 수집 통합 → 송장 번호 자동 등록 → 재고 수량 실시간 동기화
  도구: 기존 ERP 기능 + RPA

[Phase 2] Core Automation (3~6개월)
  목표: 전체 업무량 50% 감축 달성
  상품 대량 등록 API 연동 → 가격/마진 자동 계산 로직 → 정산 및 리포트 자동 생성
  도구: API 개발 + Rule Engine

[Phase 3] AI Enhancement (6~12개월)
  목표: 의사결정 속도 및 정확도 향상
  수요 예측 기반 발주량 제안 → 고객 세그먼트 자동 분류 → 동적 가격(Dynamic Pricing) 최적화
  도구: ML 모델 + A/B 테스트
```

## 입력·출력

### 입력 슬롯

| 항목 | 필수 | 예시 |
|------|------|------|
| 월 주문 건수 | 필수 | 3,500건 |
| 운영 인력 | 필수 | 정직원 2명 + 알바 1명 |
| 사용 중인 ERP·도구 | 필수 | 사방넷·카페24·쿠팡 윙 |
| 반복형 업무 비율 추정 | 권장 | 45% |
| 일평균 오류율 | 권장 | 12% (수동 입력) |
| 카테고리 | 권장 | 식품·뷰티·패션·생활용품 등 |

### 출력 (진단 보고서)

```markdown
# 자동화 도입 요약 보고

## 1. 현황 분석 (AS-IS)
- 월 주문: 3,500건 → Stage 2 성장기
- 반복형 업무 비율: 45% → 자동화 ROI 충분
- 주문 1건당 처리시간: 8분
- 수동 입력 오류율: 12%

## 2. 자동화 시나리오 (PLAN)
### Phase 1 (2개월) Quick Wins
- 송장 자동 등록 (월 120시간 절감)
- 주문 통합 자동화
### Phase 2 (6개월) Core Automation
- 상품 대량 등록 API (월 280시간 절감 누적)
- 마진 자동 계산기 (commerce-margin-calculator 연계)

## 3. 예상 효과 (TO-BE)
- 예상 1년차 ROI: 280%
- 투자금 3.2개월 내 회수
- 3명 인력 재배치 (단순→기획)
- 오류율 12% → 2% (10pp 개선)

## 4. 투자 개요
- 초기 개발비: 5,000만원
- 연간 운영비: 1,200만원
- 총 투자 (1년): 6,200만원

## 5. Next Steps
- STEP 01: 파일럿 영역 선정 → 추천: C. 주문 & 정산 영역
- STEP 02: 2주 스프린트 실행 → 송장 자동 등록 완성
- STEP 03: 효과 측정 및 확산
```
