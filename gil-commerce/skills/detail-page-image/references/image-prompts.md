# 13섹션 이미지 프롬프트 가이드

각 섹션의 비주얼 언어와 nano-banana 프롬프트 작성 패턴.

## 공통 프롬프트 구조

```
[섹션 비주얼 언어]
+ [카테고리 비주얼 브리프 (electronics/fashion/...)]
+ [상품 DNA: 형태, 색상, 소재, 시그니처 앵글]
+ [섹션 카피 인용]
+ [출력 스펙: 1080×{높이}, PNG, 한국어 타이포]
```

## 1. Hero (1080×1600) — cinematic_product_portrait

```
풀블리드 시네마틱 히어로샷. 상품이 주인공, 3/4 각도에서 드라마틱 림라이트.
좌상단 에디토리얼 타이포: 72pt 헤드라인 굵게, 36pt 서브헤드라인.
중앙좌측 CTA 버튼 캡슐. 우상단 작은 긴급성 뱃지.
여백 30% 이상. 한국 프리미엄 이커머스 무드.
```

## 2. Pain (1080×800) — emotional_photography_no_product

```
디새튜레이트된 어두운 분위기 사진 또는 어두운 비주얼 메타포.
**상품 노출 금지** — 이 섹션은 상품이 아닌 독자의 고통이 주제.
4개 불릿 32pt + 작은 아이콘. 공감 톤. 부드러운 그림자, 무드 있는 팔레트.
```

## 3. Problem (1080×800) — clinical_infographic

```
화이트 또는 크림 배경 위 깔끔한 인포그래픽 스타일. 미세한 기하 액센트.
다이어그램 형태로 3개 번호 매김 원인 (01/02/03). 단면도 또는 분자 일러스트 가능.
명확한 위계: 반전 훅 헤드라인, 번호 리스트, 마무리 시프트. 장식 최소.
```

## 4. Story (1080×1200) — editorial_split_before_after

```
좌우 분할 레이아웃: 좌측은 쿨톤 안개 낀 'Before', 우측은 웜톤 빛나는 'After'.
대각선 또는 부드러운 디바이더. 타이포가 디바이더를 가로지름.
좌상단 Before 상태, 중앙에 전환점 레이블, 우상단 After 상태,
하단 큰 배너로 증거 통계. 라이프스타일 사진 (상품 중심 X).
```

## 5. Solution (1080×800) — product_beauty_shot

```
스튜디오 뷰티샷. 상품이 깔끔한 그라디언트 또는 반사 표면 정중앙.
완벽한 라이팅 — 소프트박스, 유리 반사. 미니멀 텍스트:
상품명 (10자 이내), 한 줄 설명, 타겟 핏 태그라인. 상품의 빅 리빌.
```

## 6. How (1080×900) — illustrated_step_sequence

```
가로 또는 세로 흐름의 3단계 번호. 각 단계는 아이코닉 그래픽 + 제목 + 짧은 설명.
단계 사이 화살표. 깔끔하고 캐주얼한 톤, 친근. 하단에 결과 하이라이트.
사진 대신 일러스트 스타일 가능.
```

## 7. Proof (1080×1420) — magazine_spread

```
에디토리얼 매거진 스프레드 — 비대칭 그리드.
3개 큰 통계 블록 + 3개 리뷰어 사진 + 인용. 혼합 타이포: 굵은 큰 숫자,
이탤릭 서리프 인용. 풍부한 여백. 피처 기사 느낌.
```

## 8. Authority (1080×800) — portrait_with_quote

```
3/4 크롭 큰 포트레이트 (창업자 또는 전문가), 약간 우측 향함.
우하단 인용 박스 오버레이: 이름 + 직함 + 자격.
디새튜레이트 배경, 웜한 피사체 라이팅. 흑백 옵션 가능.
```

## 9. Benefits (1080×1200) — icon_grid_with_lifestyle

```
3×2 또는 2×3 그리드의 혜택, 각각 커스텀 아이콘 + 짧은 레이블.
중간 또는 하단에 라이프스타일 사진(실 사용 장면). 보너스 항목 액센트 박스.
하단에 총 가치 + 오퍼 가격. 컬러로 위계 구분.
```

## 10. Risk (1080×800) — document_with_seal

```
문서 또는 인증서 미학. 상단 보증 헤드라인, 하단 FAQ 리스트 (행 색 교차).
모서리에 공식 봉인 또는 뱃지. 무드 있는 코퍼레이트 팔레트.
목표: 시각적으로 '신뢰, 보증, 안전' 신호.
```

## 11. Compare (1080×800) — split_screen_vs

```
50/50 세로 분할: 좌측 '상품 없을 때' (무드 다운, 답답함) vs
우측 '상품 있을 때' (밝고 성공). 좌우 동일 구도로 직접 비교.
각각 3포인트. 명확한 VS 디바이더.
```

## 12. Filter (1080×700) — checklist_visual

```
2 컬럼: 'Recommend' (그린 체크) vs 'Not Recommend' (레드 X).
각 리스트 3개 타겟 핏 문구. 단순 리스트 위주 레이아웃. 의사결정 트리 느낌.
```

## 13. CTA (1080×900) — urgent_product_reveal

```
최종 CTA 섹션. 히어로 스타일 상품샷(Hero 각도 재사용 가능) + 긴급성 오버레이.
강조된 정가 취소선 + 할인가 (액센트 컬러). 카운트다운 또는 희소성 카피.
큰 CTA 버튼. 버튼 아래 마감 태그라인.
```

## 한국어 타이포 처리

이미지 안 한국어 텍스트가 깨지는 경우, **Higgsfield MCP**(Soul) 직접 호출로 재생성하거나, 이미지는 비주얼만 만들고 텍스트는 별도 PostScript 단계로 오버레이.

## 카테고리별 비주얼 브리프 적용

각 섹션 프롬프트에 `references/category-briefs.md`(detail-page-copy)의
카테고리 브리프를 함께 주입합니다. 예시 (electronics):

```
+ lighting: studio softbox + rim light, cool 5600K
+ palette: deep navy + warm silver + neon cyan accent
+ typography: geometric sans-serif, wide tracking
+ mood: confident, precise, slightly futuristic
```
