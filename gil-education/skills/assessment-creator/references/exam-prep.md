# 시험 대비 (57-exam-prep)

> MoAI-Cowork v1.0.0 하네스 레퍼런스

## 개요

시험 트렌드 분석, 약점 진단, 맞춤형 학습 계획, 모의고사, 오류 분석을 수행하는 에이전트 팀 협업 하네스입니다.

## 전문가 역할

- **진단 전문가**: 학습자의 현재 수준을 영역별로 측정하고 약점을 파악하여 맞춤 학습 전략의 기초를 제공합니다.
  - 영역별 숙달도 측정
  - 약점 파악
  - 강점-약점 매핑
  - 학습 스타일 분석
  - 목표 갭 분석

- **오류 분석가**: 모의고사 결과를 분석하여 오류 패턴을 파악하고 개념 결핍을 진단하며 집중 보완 전략을 제시합니다.
  - 오류 패턴 분류
  - 개념 결핍 추적
  - 오류 노트 생성
  - 약점 업데이트
  - 보완 제안

- **출제자**: 실제 시험과 동일한 형식, 난이도, 시간으로 모의고사를 설계하고 상세한 해설을 작성합니다.
  - 문제 출제
  - 난이도 설계
  - 채점 설계
  - 함정 문항 설계
  - 해설 작성

- **학습 설계자**: 진단 결과에 기반하여 맞춤형 학습 커리큘럼, 일정, 교재를 구성합니다.
  - 학습 우선순위 결정
  - 일정 작성
  - 학습 방법 매칭
  - 학습 자료 구성
  - 마일스톤 설정

- **트렌드 분석가**: 과거 시험 출제 패턴을 분석하여 자주 출제되는 분야, 난이도 추이, 예상 출제 주제를 파악합니다.
  - 과거 시험 패턴 분석
  - 자주 출제되는 분야 파악
  - 난이도 추이 추적
  - 예상 출제 주제
  - 함정 분류

## 워크플로우

### 단계 1: 준비 (오케스트레이터 직접 수행)

1. 사용자 입력에서 추출:
   - **시험명**: 대학입시/공무원/자격증/TOEIC 등
   - **과목/영역**: 전과목 또는 특정 과목
   - **현재 수준** (선택): 기존 성적, 모의고사 결과, 자가 진단
   - **목표** (선택): 목표 점수, 합격 기준
   - **남은 기간** (선택): 시험까지 남은 시간
   - **기존 데이터** (선택): 성적표, 오류 노트, 학습 이력

2. 프로젝트 루트에 `_workspace/` 디렉토리 생성
3. 입력을 정리하여 `_workspace/00_input.md`로 저장
4. 기존 데이터가 있으면 복사 후 해당 단계 조정
5. 요청 범위에 따라 실행 모드 결정

### 단계 2: 팀 구성 및 실행

| 순서 | 작업 | 에이전트 | 의존성 | 산출물 |
|------|------|--------|--------|--------|
| 1 | 트렌드 분석 | 트렌드 분석가 | 없음 | `01_trend_analysis.md` |
| 2 | 약점 진단 | 진단 전문가 | 작업 1 | `02_diagnosis_report.md` |
| 3 | 학습 계획 | 학습 설계자 | 작업 1, 2 | `03_learning_plan.md` |
| 4 | 모의고사 제작 | 출제자 | 작업 1, 2, 3 | `04_mock_exam.md`, `04_mock_exam_answer.md` |
| 5 | 오류 분석 | 오류 분석가 | 작업 4 | `05_error_analysis.md` |

**에이전트 간 커뮤니케이션 흐름:**
- 분석가 완료 → 자주 출제 분야·함정 유형을 진단가에게, 예상 주제를 설계자에게, 빈도·난이도를 출제자에게 전달
- 진단가 완료 → 약점·약점 유형·목표 갭을 설계자에게, 약점 중심 문항 요청을 출제자에게 전달
- 설계자 완료 → 학습 범위·난이도 가이드를 출제자에게 전달
- 출제자 완료 → 문항지, 정답지, 문항 설계 의도를 분석가에게 전달
- 분석가 완료 → 보완 항목을 설계자에게 피드백, 차기 시험 대비 항목을 출제자에게 전달

### 단계 3: 통합 및 최종 보고

1. `_workspace/` 내 모든 파일 확인
2. 오류 분석 보완 사항이 학습 계획에 반영되었는지 확인
3. 사용자에게 최종 요약 보고:
   - 트렌드 분석 — `01_trend_analysis.md`
   - 약점 진단 — `02_diagnosis_report.md`
   - 맞춤형 학습 계획 — `03_learning_plan.md`
   - 모의고사 문항지 — `04_mock_exam.md`
   - 모의고사 정답지 — `04_mock_exam_answer.md`
   - 오류 분석 — `05_error_analysis.md`

## 오류 처리

| 오류 유형 | 대응 전략 |
|---------|---------|
| 시험 정보 검색 실패 | 사용자에게 직접 시험 형식 확인, 일반적 지식으로 진행 |
| 과거 시험 데이터 부족 | 범위 내에서 분석, "데이터 제한" 표기 |
| 사용자 미응답(진단 항목) | 자가 진단 체크리스트 대체, 주관적 진단 수행 |
| 과목 범위 과다 | 핵심 과목부터 처리, 나머지는 후속 요청으로 안내 |
| 에이전트 실패 | 1회 재시도 → 해당 산출물 없이 진행 |
