# 학술 연구 지원 (63-research-assistant)

> MoAI-Cowork v1.0.0 하네스 레퍼런스

## 개요

문헌 검색, 읽기 노트, 비판적 종합, 참고문헌 관리를 통해 학술 연구를 지원하는 에이전트 팀 협업 하네스입니다.

## 전문가 역할

- **비판적 분석가**: 수집된 문헌을 비판적으로 분석하고 연구 공백을 파악하며 주제별 종합을 통해 문헌 검토의 핵심 서사를 구축합니다.
  - 비판적 평가: 각 연구의 방법론 엄밀성, 논리적 일관성, 일반화 가능성 평가
  - 주제별 종합: 문헌 내 핵심 주제, 패턴, 트렌드 파악 및 조직
  - 연구 공백 파악: 미탐색 영역, 상충하는 발견, 미해결 질문 파악
  - 이론적 틀 분석: 주요 이론적 관점 비교 및 적용가능성 평가
  - 종합 서사 구축: 문헌 검토의 논리적 흐름과 스토리라인 설계

- **문헌 검색 전문가**: 학술DB와 웹을 체계적으로 탐색하여 관련 논문, 책, 보고서를 수집하고 관련성을 평가합니다.
  - 검색 전략 개발: 연구 질문에서 핵심 키워드, 유의어, 관련어 도출
  - DB 탐색: Google Scholar, arXiv, PubMed, SSRN 등 검색
  - 역참고 추적: 핵심 논문의 인용 논문과 피인용 논문 추적
  - 관련성 필터링: 초록 기반 관련성 평가 및 포함/제외 기준 적용
  - 문헌 맵 생성: 연구 영역 내 세부 주제별 분류

- **읽기 노트 작성가**: 각 문헌을 정독하고 핵심 논거, 방법론, 발견, 인용 구절을 체계적으로 정리합니다.
  - 핵심 요약: 각 자료의 연구 질문, 방법론, 주요 발견, 결론 요약
  - 인용 추출: 직접 인용 가치가 있는 핵심 구절과 페이지 번호
  - 방법론 분석: 연구설계, 자료 수집, 분석 방법의 강점과 한계
  - 연결고리 파악: 자료 간 공통점, 모순, 발전 관계 태깅
  - 질문 연결: 각 자료가 사용자 연구 질문에 기여하는 방식 명시

- **참고문헌 관리자**: 모든 인용 자료의 서지 정보를 정확하게 관리하고 요청 형식으로 변환하며 중복과 누락을 확인합니다.
  - 서지 정보 수집: 각 자료의 저자, 연도, 제목, 출처, DOI, URL 정확한 기록
  - 형식 변환: APA, MLA, Chicago, Harvard, IEEE 등으로 변환
  - 본문 인용 생성: 저자-연도, 각주 등 본문 인용 형식 생성
  - 중복/누락 확인: 참고문헌 목록과 본문 인용의 일관성 확인
  - BibTeX/RIS 생성: 참고문헌 관리 소프트웨어 호환 형식 제공

- **연구 조정자**: 문헌 검색, 읽기 노트, 비판적 분석, 참고문헌 간의 일관성을 교차 검증하고 최종 보고서를 작성합니다.
  - 검색 완전성 검증: 충분한 범위의 검색 및 주요 자료 누락 확인
  - 노트-분석 일관성: 읽기 노트가 종합 분석에 정확하게 반영되었는지 검증
  - 인용 정확성 검증: 본문 인용과 참고문헌 목록의 교차 검증
  - 논리적 일관성: 종합 서사의 논리적 흐름과 근거의 충분성 평가
  - 최종 보고서 작성: 전체 연구 지원 과정 요약 및 후속 조치 제안

## 워크플로우

### 단계 1: 준비 (오케스트레이터 직접 수행)

1. 사용자 입력에서 추출:
   - **연구 주제/질문**: 무엇을 조사할 것인가
   - **연구 목적** (선택): 논문 작성, 발표, 보고서 등
   - **인용 형식** (선택): APA, MLA, Chicago 등 (기본값: APA 7판)
   - **범위 제한** (선택): 시간대, 언어, 분야, 자료 수
   - **기존 자료** (선택): 이미 수집한 논문, 노트

2. 프로젝트 루트에 `_workspace/` 디렉토리 생성
3. 입력을 정리하여 `_workspace/00_input.md`로 저장
4. 기존 자료가 있으면 복사 후 해당 단계 스킵
5. 요청 범위에 따라 실행 모드 결정

### 단계 2: 팀 구성 및 실행

| 순서 | 작업 | 담당 | 의존성 | 산출물 |
|------|------|------|--------|--------|
| 1 | 문헌 검색 | 검색 전문가 | 없음 | `01_literature_search.md` |
| 2a | 읽기 노트 | 노트 작성가 | 작업 1 | `02_reading_notes.md` |
| 2b | 초안 참고문헌 | 관리자 | 작업 1 | `04_bibliography.md` (초안) |
| 3 | 비판적 종합 | 비판적 분석가 | 작업 2a | `03_critical_synthesis.md` |
| 4 | 최종 참고문헌 | 관리자 | 작업 3 | `04_bibliography.md` (최종) |
| 5 | 연구 조정 | 조정자 | 작업 2a, 3, 4 | `05_research_summary.md` |

작업 2a(노트)와 2b(초안 참고문헌)는 **병렬 진행**.

**에이전트 간 커뮤니케이션 흐름:**
- 검색자 완료 → 자료 목록과 우선순위를 노트 작성가에게, 서지 정보를 관리자에게 전달
- 노트 작성가 완료 → 노트와 연결고리를 분석가에게 전달
- 분석가 완료 → 인용 목록을 관리자에게 전달
- 조정자 → 모든 산출물 교차 검증, RED 이슈 발견 시 관련 에이전트에 수정 요청 → 재작업 → 재검증 (최대 2회)

### 단계 3: 통합 및 최종 산출물

1. `_workspace/` 내 모든 파일 검토
2. 연구 조정 보고서의 RED 항목이 모두 처리되었는지 확인
3. 사용자에게 최종 요약 제시:
   - 문헌 검색 결과 — `01_literature_search.md`
   - 읽기 노트 — `02_reading_notes.md`
   - 비판적 종합 — `03_critical_synthesis.md`
   - 참고문헌 — `04_bibliography.md`
   - 연구 보고서 — `05_research_summary.md`

## 산출물

- `00_input.md` — 정리된 사용자 입력
- `01_literature_search.md` — 문헌 검색 결과
- `02_reading_notes.md` — 읽기 노트
- `03_critical_synthesis.md` — 비판적 종합
- `04_bibliography.md` — 참고문헌 목록
- `05_research_summary.md` — 연구 요약 보고서

## 확장 스킬

- **citation-formatter**: APA, MLA, Chicago 등 주요 인용 형식으로 정확히 변환
- **systematic-review-protocol**: PRISMA 프로토콜과 체계적 문헌 검색 전략

## 오류 처리

| 오류 유형 | 대응 전략 |
|---------|---------|
| 웹 검색 실패 | 알려진 주요 학술지와 저자 기반 제안, "검색 제한" 기록 |
| 전문 보유 불가 | 초록 기반으로 작업, "전문 미확인" 표기 |
| 자료 부족 | 인접 분야로 검색 확대, "예비 종합"으로 표기 |
| 에이전트 실패 | 1회 재시도 → 해당 산출물 없이 진행, 보고서에 기록 |
| 인용 형식 미지정 | APA 7판 기본 적용, 사용자 확인 요청 |
