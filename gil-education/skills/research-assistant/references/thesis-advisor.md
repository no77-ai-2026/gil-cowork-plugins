# 논문 작성 지도자 (58-thesis-advisor)

> MoAI-Cowork v1.0.0 하네스 레퍼런스

## 개요

주제 선정, 문헌 검토, 연구방법론, 논문 작성, 교정을 수행하는 에이전트 팀 협업 하네스입니다.

## 전문가 역할

- **문헌 분석가**: 선행 연구를 체계적으로 조사하고 비판적으로 검토하여 이론적 틀을 구축합니다.
  - 검색 전략
  - 선행 연구 분류
  - 비판적 검토
  - 이론적 틀 구축
  - 연구 공백 확인

- **방법론 전문가**: 연구 질문에 적합한 연구 설계, 데이터 수집 방법, 분석 기법을 설계하고 방법론적 엄밀성을 보장합니다.
  - 연구 설계
  - 표본 설계
  - 자료 수집 도구
  - 분석 방법 선정
  - 연구 윤리

- **교정자**: 문법·철자 수정, 학술 형식 확인, 논리적 일관성 검토, 표절 위험 평가를 수행합니다.
  - 문법·철자 수정
  - 학술 스타일 검증
  - 형식 검증
  - 논리적 일관성
  - 표절 위험 평가

- **주제 탐색자**: 연구 분야의 트렌드를 조사하고 연구 공백을 발견하며 구체적인 연구 질문과 가설을 수립합니다.
  - 분야 트렌드 분석
  - 연구 공백 파악
  - 연구 질문 수립
  - 가설 개발
  - 실행가능성 검증

- **논문 작성 코치**: 논문의 전체 구조를 설계하고 각 장을 초안 작성하며 논거를 강화합니다.
  - 논문 구조 설계
  - 서론 작성
  - 문헌 검토 장
  - 방법론 장
  - 논거 강화

## 워크플로우

### 단계 1: 준비 (오케스트레이터 직접 수행)

1. 사용자 입력에서 추출:
   - **학위 전공**: 주 전공과 세부 분야
   - **논문 유형**: 석사/박사/학술지 논문/학부 논문
   - **관심 주제** (선택): 특정 주제 또는 키워드
   - **지도교수 연구 분야** (선택): 랩 연구 방향
   - **기존 자료** (선택): 기존 원고, 참고문헌 목록, 데이터
   - **제출 기한** (선택): 제출 예정일

2. 프로젝트 루트에 `_workspace/` 디렉토리 생성
3. 입력을 정리하여 `_workspace/00_input.md`로 저장
4. 기존 자료가 있으면 복사 후 해당 단계 조정
5. 요청 범위에 따라 실행 모드 결정

### 단계 2: 팀 구성 및 실행

| 순서 | 작업 | 에이전트 | 의존성 | 산출물 |
|------|------|--------|--------|--------|
| 1 | 주제 탐색 | 주제 탐색자 | 없음 | `01_topic_proposal.md` |
| 2 | 문헌 검토 | 문헌 분석가 | 작업 1 | `02_literature_review.md` |
| 3 | 방법론 설계 | 방법론 전문가 | 작업 1, 2 | `03_methodology_design.md` |
| 4 | 초안 작성 | 논문 코치 | 작업 1, 2, 3 | `04_draft_manuscript.md` |
| 5 | 교정 | 교정자 | 작업 4 | `05_proofread_report.md` |

**에이전트 간 커뮤니케이션 흐름:**
- 탐색자 완료 → 연구 질문과 키워드를 분석가에게, 가설과 변수를 방법론가에게 전달
- 분석가 완료 → 선행 연구의 방법론 트렌드를 방법론가에게, 문헌 검토 내용을 코치에게 전달
- 방법론가 완료 → 전체 방법론 설계를 코치에게 전달
- 코치 완료 → 완성된 초안을 교정자에게 전달
- 교정자 완료 → 중대 발견 시 관련 에이전트에 수정 요청 (최대 2회)

### 단계 3: 통합 및 최종 보고

1. `_workspace/` 내 모든 파일 확인
2. 중대한 교정 항목이 모두 처리되었는지 확인
3. 사용자에게 최종 요약 보고

## 산출물

- `01_topic_exploration.md` — 주제 탐색 보고서
- `02_literature_review.md` — 문헌 분석 보고서
- `03_methodology.md` — 방법론 설계 문서
- `04_writing_guide.md` — 작성 가이드
- `05_proofreading.md` — 교정 보고서
- `06_review_report.md` — 검토 보고서

## 확장 스킬

- **research-methodology**: 연구설계 매트릭스, 표본 크기 계산, 타당성/신뢰성, 분석 방법 선정

## 오류 처리

| 오류 유형 | 대응 전략 |
|---------|---------|
| 학술DB 검색 실패 | 웹 검색으로 대체, 보고서에 "검색 제한" 기록 |
| 분야 결정 불가 | 사용자에게 질문, 관심 분야에서 추론 |
| 기존 원고 형식 불명확 | 표준 논문 형식 적용 |
| 에이전트 실패 | 1회 재시도 → 해당 산출물 없이 진행 |
| 교정 중 중대 발견 | 관련 에이전트에 수정 요청 → 재검증 (최대 2회) |
