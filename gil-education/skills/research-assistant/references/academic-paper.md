# 학술 논문 작성 (98-academic-paper)

> MoAI-Cowork v1.0.0 하네스 레퍼런스

## 개요

연구설계, 실험 관리, 통계 분석, 논문 작성, 제출 준비를 수행하는 에이전트 팀 협업 하네스입니다.

## 전문가 역할

- **실험 관리자**: 연구설계에 기반하여 실험 프로토콜, 자료 수집 절차, 측정 도구, 파일럿 테스트를 개발합니다.
  - 실험 프로토콜 작성: 재현 가능한 수준의 단계별 절차
  - 측정 도구 설계: 설문지, 인터뷰 가이드, 자극물 등
  - 자료 수집 계획: 수집 방법, 일정, 품질 관리
  - 파일럿 테스트: 본 실험 전 예비 연구 계획
  - 자료 관리 계획: 코딩, 저장, 보안, 공유 정책

- **논문 작성자**: IMRaD 구조에 따라 학술 논문을 작성하고 인용을 관리합니다.
  - 논문 구조 설계: IMRaD(도입-방법-결과-토론) 골격
  - 도입 작성: 연구 배경, 선행 문헌, 연구 목적, 가설 
  - 방법론 작성: 재현 가능한 수준의 세부 내용
  - 결과 작성: 가설 순서대로 객관적 결과 보고
  - 토론 작성: 해석, 이론·실무적 함의, 한계, 향후 연구

- **연구 설계자**: 연구 질문 수립, 가설 개발, 연구방법 선정, 변수 정의, 문헌 분석을 수행합니다.
  - 연구 질문 수립: 선행 문헌의 공백을 발견하고 구체적 질문 개발
  - 가설 개발: 이론적 근거에 기반한 검증 가능한 가설(H1, H2 등) 제시
  - 방법론 선정: 양적/질적/혼합방법, 실험/설문/사례 등 최적 방법 선택
  - 변수 정의: 독립, 종속, 통제, 매개/조절 변수의 조작적 정의
  - 문헌 분석: 관련 이론과 선행 연구의 체계적 정리

- **통계 분석가**: 연구설계에 적합한 통계 전략을 개발하고 분석 코드를 생성하며 결과를 해석합니다.
  - 분석 전략 개발: 연구설계와 자료 특성에 맞는 통계 기법
  - 전처리 계획: 결측치 처리, 이상치 탐지, 정규성 검정, 변환 전략
  - 분석 코드 생성: R 또는 Python(statsmodels/scipy) 코드 작성
  - 결과 해석: 통계적 유의성과 실제적 의미(효과크기) 구분
  - 시각화: 학술지 게재용 고품질 표와 그림

- **제출 준비자**: 목표 학술지 선정, 학술지별 형식 조정, 커버레터 작성, 심사자 추천 및 제외 요청을 관리합니다.
  - 목표 학술지 선정: 연구 분야, 임팩트팩터, 범위, 심사 기간 고려
  - 형식 조정: 학술지별 형식 요구(글자 수, 참고문헌 양식, 그림 형식)
  - 커버레터 작성: 편집장에게 연구의 신규성, 관련성, 기여도 강조
  - 심사자 추천/제외: 3-5명 추천 심사자, 제외 요청 대상
  - 제출 체크리스트: 학술지별 필수 자료 확인

## 워크플로우

### 단계 1: 준비 (오케스트레이터 직접 수행)

1. 사용자 입력에서 추출:
   - **연구 주제**: 연구 분야, 핵심 키워드
   - **연구 수준** (선택): 학부/석사/박사/교수급
   - **목표 학술지** (선택): 학술지명, 임팩트팩터 범위
   - **기존 자료** (선택): 자료, 초안, 참고문헌 목록
   - **제약 조건** (선택): 마감일, 글자 수, 특수 사항

2. 프로젝트 루트에 `_workspace/` 디렉토리 생성
3. 입력을 정리하여 `_workspace/00_input.md`로 저장
4. 기존 파일이 있으면 복사 후 해당 단계 스킵
5. 요청 범위에 따라 실행 모드 결정

### 단계 2: 팀 구성 및 실행

| 순서 | 작업 | 담당 | 의존성 | 산출물 |
|------|------|------|--------|--------|
| 1 | 연구설계 | 설계자 | 없음 | `01_research_design.md` |
| 2 | 실험 프로토콜 | 관리자 | 작업 1 | `02_experiment_protocol.md` |
| 3 | 통계 분석 | 분석가 | 작업 1, 2 | `03_analysis_report.md` |
| 4 | 원고 작성 | 작성자 | 작업 1, 2, 3 | `04_manuscript.md` |
| 5 | 제출 준비 | 준비자 | 작업 1, 4 | `05_submission_package.md` |
| 6 | 통합 검토 | 오케스트레이터 | 전체 | `06_review_report.md` |

**에이전트 간 커뮤니케이션 흐름:**
- 설계자 완료 → 변수와 표본 계획을 관리자에게, 가설과 분석 방향을 분석가에게, 이론적 배경을 작성자에게, 연구 분야를 준비자에게 전달
- 관리자 완료 → 자료 구조를 분석가에게, 방법론 자료를 작성자에게 전달
- 분석가 완료 → 결과(APA 형식 + 표/그림)을 작성자에게, 분석 코드를 준비자에게 전달
- 작성자 완료 → 완성된 원고를 준비자에게 전달
- 오케스트레이터 → 전체 일관성 검증(가설↔분석↔결과↔토론)

### 단계 3: 통합 및 최종 산출물

1. `_workspace/` 내 모든 파일의 일관성 검증
2. 가설-분석-결과-토론 연쇄의 논리적 일관성 검증
3. 통합 검토 보고서 생성
4. 사용자에게 최종 요약 보고

## 오류 처리

| 오류 유형 | 대응 전략 |
|---------|---------|
| 선행 문헌 검색 실패 | 사용자 제공 참고문헌으로 작업, "문헌 검토 제한" 기록 |
| 연구 주제 모호 | 3개 구체적 세부주제 제시 후 선택 요청 |
| 자료 미보유 | 분석 전략과 코드만 제공, "자료 수집 후 실행" 안내 |
| 에이전트 실패 | 1회 재시도 → 해당 산출물 없이 진행, 검토 보고서에 기록 |
| 가설-결과 불일치 | 작성자가 토론에서 대응, 추가 분석 제안 |
