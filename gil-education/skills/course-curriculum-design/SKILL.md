# 강의 운영 매뉴얼 자동 생성 (강사·교수·교사용)

## 개요

강사·교수·교사가 운영하는 모든 강의·수업·연수·워크숍·정규 강좌의 **운영 실무 매뉴얼**을 자동 생성합니다. 강의명·일수·세션 수·운영 인력만 알려주면 일자별 시간표, 강사·조교 동선, D-N 사전 준비물 안내, 환경·설비 체크리스트, 리스크 매트릭스 + Plan B 시나리오까지 한 번에 완성합니다.

지원 범위:
- **단기**: 1일 특강, 2-3일 집중 워크숍, 주말 부트캠프
- **중기**: 1-2주 단기 연수, 사내 교육 프로그램
- **장기**: 8주·16주 정규 강좌, 학기제 수업, 학기별 과목 운영
- **혼합**: 온라인+오프라인 블렌디드, 야간·주말 분산 수업

**체이닝 출력**
운영 매뉴얼 생성 후 `gil-office:docx-generator`를 자동 체이닝하여 Word(.docx) 형식으로 저장합니다.

**합격 기준**
모든 산출물 섹션(시간표·동선·사전 준비물·체크리스트·리스크·운영 원칙)을 포함하며 Plan B 시나리오 5건 이상을 제공합니다.

## 워크플로우

### 호출 방법

```bash
# 전체 운영 매뉴얼 생성 (기본)
/course-curriculum-design --course "사내 AI 활용 2일 워크숍"

# 사전 준비물 안내 메일만 출력
/course-curriculum-design --output prep-mail --lead-days 7

# 긴급 Plan B 추출 (실시간 리스크 대응)
/course-curriculum-design --emergency "장비 오류"

# 일수·세션 수 커스텀
/course-curriculum-design --course "1일 데이터 분석 특강" --days 1 --sessions 5
/course-curriculum-design --course "8주 디자인 정규 강좌" --days 8 --sessions 3 --cadence weekly
```

### 입력 슬롯

| 슬롯 | 필수 여부 | 기본값 | 설명 |
|------|-----------|--------|------|
| `--course` | 선택 | 없음 | 강의·수업·연수·워크숍 명칭 또는 한 줄 컨셉 |
| `--days` | 선택 | 3 | 강의 일수 (1-16) |
| `--sessions` | 선택 | 7 | 일별 세션 수 (예: S1~S7 × 50분) |
| `--cadence` | 선택 | daily | daily(연속) / weekly(주 1회) / custom |
| `--lead-days` | 선택 | 7 | 사전 준비물 안내 발송 시점 (D-N) |
| `--output` | 선택 | full | full / prep-mail / risk-only / timetable |
| `--emergency` | 선택 | 없음 | 긴급 Plan B 추출 (리스크 키워드 입력) |
| `input_pdf` | 선택 | 없음 | 강의 기획서 PDF (첨부 시 자동 파싱) |

### 처리 체인

```
Step 1: 입력 파싱 (PDF 첨부 또는 한 줄 컨셉 + 운영 파라미터)
Step 2: 일수·세션·운영 인력 기반 섹션별 콘텐츠 생성
Step 3: 운영 매뉴얼 .md 생성
Step 4: gil-office:docx-generator 자동 체이닝 → .docx 출력
```

## 산출물 섹션 상세

### §1. 표준 시간표 (일자별 × 세션별)

예시 — 1일 7세션 × 50분 구조:

```markdown
| 교시 | 시간 | 주제 | 산출물 | 강사/조교 |
|------|------|------|--------|-----------|
| S1   | 10:00–10:50 | [주제] | [산출물] | 강사 |
| S2   | 11:00–11:50 | [주제] | [산출물] | 강사 + 조교 1 |
| S3   | 12:00–12:50 | [주제] | [산출물] | 강사 |
| 점심 | 13:00–14:00 | — | — | — |
| S4   | 14:00–14:50 | [주제] | [산출물] | 강사 |
| S5   | 15:00–15:50 | [주제] | [산출물] | 강사 + 조교 2 |
| S6   | 16:00–16:50 | [주제] | [산출물] | 강사 |
| S7   | 17:00–17:50 | [주제] | [산출물] | 강사 + 전체 |
```

일수에 따라 각 일자(Day N)별로 동일 구조를 반복하며, `--cadence weekly` 시 주차(Week N) 기반으로 확장합니다.

---

### §2. 강사·조교 동선 (일자 × 역할)

```markdown
| 시점 | 메인 강사 | 조교 1 | 조교 2 |
|------|-----------|--------|--------|
| D-7 | 커리큘럼 최종 확인 | 참가자 사전 준비 메일 발송 | 설비 사전 점검 |
| D-3 | — | 환경·계정 트러블슈팅 사전 점검 | 외부 도구·API 인증 라운드 |
| Day N S1 | 강의 진행 | 실습 지원 순회 | 출결 확인 + 채팅 모니터링 |
| Day N S2 | 강의 진행 | 환경 설치 트러블슈팅 1:1 | 공통 오류 대응 |
| Day N S4 | 핵심 도구 시연 | 수강생 인증·로그인 지원 | 영상 녹화·자료 백업 |
...
```

운영 인력 수(`강사 1` / `강사 + 조교 1` / `강사 + 조교 2` 등)에 맞춰 자동 생성합니다.

---

### §3. 사전 준비물 (D-N 안내 메일) — 4분류

```markdown
## 하드웨어
- 노트북·태블릿 (운영체제 요구사항)
- 안정적인 인터넷 연결 (유선 권장 시 명시)

## 계정·소프트웨어
- 강의에서 사용할 도구 가입·로그인 (D-3 안내)
- 유료 도구는 무료 체험·평가판 안내

## 데이터·자료
- 본인 사례 데이터 1-3건 (실습 예시용)
- 사전 학습 자료(영상·읽기) 링크

## 선택
- 외장 마우스·이어폰·메모 도구
- 보조 배터리·멀티탭
```

`--lead-days N`으로 사전 안내 시점을 조정할 수 있습니다 (D-3·D-7·D-14·D-30 등).

### §4. 환경·설비 체크리스트

```markdown
## Wi-Fi
- [ ] 강의실 Wi-Fi 속도 확인 (100Mbps 이상 권장)
- [ ] 공유기 위치 확인 (강의 공간 중앙 배치)

## 프로젝터·모니터
- [ ] HDMI / USB-C 어댑터 지참
- [ ] 해상도 1920×1080 이상 설정
- [ ] 발표자 모드 / 미러링 전환 테스트

## 녹화·음향
- [ ] 화면 녹화 소프트웨어 준비
- [ ] 마이크 음질 사전 체크
- [ ] 스피커·핀마이크 백업

## 간식·비품
- [ ] 물·간식 1인당 준비
- [ ] 충전기·멀티탭 (2열 × 5구 이상)
- [ ] 명찰·교재·필기구
```

### §5. 통합 리스크 매트릭스 + Plan B (5건 이상)

```markdown
| # | 발현 신호 | 리스크 | Plan B | 책임 |
|---|-----------|--------|--------|------|
| 1 | S1 도입부 10분 안에 설치 못 함 | 수강생 환경 설치 실패 | 조교 1:1 원격 화면공유 → 안 되면 강사 환경에서 임시 진행 | 조교 1 |
| 2 | S2 외부 API 키 오류 3명 이상 | 외부 도구 인증 집단 실패 | 강사 워크스페이스에서 시연 전환 + 키 재발급 안내 메일 | 조교 2 |
| 3 | 도구 호출 30초 초과 응답 없음 | 외부 서비스 장애 | 사전 녹화 데모 영상 재생 + 대체 도구 안내 | 강사 |
| 4 | 강의실 Wi-Fi 속도 저하 | 인터넷 연결 불안정 | 핫스팟 전환 (조교 스마트폰) + 오프라인 작업 전환 | 조교 2 |
| 5 | 사후 후기·과제 발송 오류 | 공지 채널 발송 실패 | 이메일 대체 발송 + 개별 DM 안내 | 운영팀 |
| 6 | 마지막 세션 데모 실패 | 라이브 데모 환경 오류 | 사전 녹화 영상 재생 + 다음 회차 시연 재진행 안내 | 강사 |
```

**긴급 Plan B 추출 (`--emergency` 옵션)**

```
/course-curriculum-design --emergency "장비 오류"
→ 리스크 매트릭스에서 해당 키워드와 매칭되는 Plan B 즉시 추출
```

### §6. 학습 부담 최소화 운영 원칙

운영 원칙:
- 수강생은 최소한의 입력으로 결과를 얻을 수 있도록 흐름을 단순화
- 강사가 먼저 시연 → 수강생이 동일 패턴으로 따라하기 구조
- 오류 발생 시 조교 1:1 지원으로 강의 흐름 중단을 최소화
- 산출물은 당일·당주에 즉시 배포 (zip 패키지·공유 폴더)
