# 역량 모델링 (61-competency-modeler)

> MoAI-Cowork v1.0.0 하네스 레퍼런스

## 개요

직무 분석, 역량 사전, 평가 루브릭, 개발 계획, 역량 매트릭스를 통해 전체 역량 모델링 파이프라인을 수행하는 에이전트 팀 협업 하네스입니다.

## 전문가 역할

- **역량 설계자**: 직무 분석 결과에 기반하여 역량을 도출하고 행동 지표를 개발하며 숙련도 수준 프레임워크를 설계합니다.
  - 역량 도출: KSA를 통합하여 8-15개 수준의 높은 수준의 역량 정의
  - 역량 정의: 각 역량의 이름, 정의, 하위 요소 명확히 기술
  - 행동 지표 개발: 숙련도별 관찰 가능한 행동 지표
  - 숙련도 설계: 5단계 숙련도 체계 설계 및 각 수준 기준 정의
  - 역량 분류: 핵심/리더십/기능 역량으로 분류

- **개발 계획가**: 갭 분석에 기반하여 개인 및 조직의 역량 개발 계획을 수립하고 역량 매트릭스를 작성합니다.
  - 역량 갭 분석: 현재 수준과 목표 수준 간 갭 분석
  - 개발 계획: 70:20:10 원칙(경험:관계:교육) 기반 역량 개발 활동 설계
  - 학습 경로: 각 역량별 단계적 학습 경로 제공
  - 역량 매트릭스: 직무×역량×수준 3차원 매트릭스 구축
  - ROI 추정: 역량 개발 투자의 예상 수익 정량적 추정

- **직무 분석가**: 직무 설명 작성, 핵심 업무 분석, KSA 추출을 수행합니다.
  - 직무 설명: 직무의 목적, 책임, 권한, 보직 관계 체계적 기술
  - 업무 분석: 직무를 구성하는 핵심 업무 파악 및 빈도·중요도·난이도에 따른 가중치 부여
  - KSA 추출: 각 업무에 필요한 지식, 기술, 태도(Knowledge, Skill, Attitude) 파악
  - 직무 환경 분석: 조직 구조, 산업 특성, 업무 환경, 이해관계자 분석
  - 직무 분류: NCS(국가직무능력표준), O*NET 등 표준 직무 분류 체계 매핑

- **루브릭 설계자**: 각 역량별 평가 기준, 채점 가이드, 평가 도구를 설계합니다.
  - 평가 루브릭 설계: 숚련도별 구체적인 평가 기준
  - 채점 가이드: 평가자의 일관된 평정을 위한 상세한 가이드
  - 평가 도구 설계: 자기 평가, 상사 평가, 동료 평가, 360도 피드백 등 평가 방법 설계
  - 평가 항목 개발: 행동 사건 인터뷰(BEI), 상황 판단 검사(SJT) 등 평가 항목
  - 신뢰성·타당성 보장: 평가 일관성과 정확성 보장 가이드라인 제시

## 워크플로우

### 단계 1: 준비 (오케스트레이터 직접 수행)

1. 사용자 입력에서 추출:
   - **대상 직무**: 역량 모델을 구축할 직무
   - **조직 정보** (선택): 산업, 규모, 조직 문화
   - **목적** (선택): 채용/평가/개발/승진/조직설계
   - **직급 범위** (선택): 특정 직급 또는 전체
   - **기존 자료** (선택): 기존 직무 기술서, 역량 체계

2. 프로젝트 루트에 `_workspace/` 디렉토리 생성
3. 입력을 정리하여 `_workspace/00_input.md`로 저장
4. 기존 자료가 있으면 복사 후 해당 단계 조정
5. 요청 범위에 따라 실행 모드 결정

### 단계 2: 팀 구성 및 실행

| 순서 | 작업 | 담당 | 의존성 | 산출물 |
|------|------|------|--------|--------|
| 1 | 직무 분석 | 분석가 | 없음 | `01_job_analysis.md` |
| 2 | 역량 사전 | 설계자 | 작업 1 | `02_competency_dictionary.md` |
| 3 | 평가 루브릭 | 루브릭 설계자 | 작업 1, 2 | `03_assessment_rubric.md` |
| 4a | 개발 계획 | 계획가 | 작업 1, 2, 3 | `04_development_plan.md` |
| 4b | 역량 매트릭스 | 계획가 | 작업 1, 2, 3 | `05_competency_matrix.md` |

**에이전트 간 커뮤니케이션 흐름:**
- 분석가 완료 → KSA 매핑과 업무 가중치를 설계자에게, 업무별 성과 기준을 루브릭 설계자에게 전달
- 설계자 완료 → 역량 정의, 행동 지표, 숚련도 체계를 루브릭 설계자에게, 역량 목록과 체계를 계획가에게 전달
- 루브릭 설계자 완료 → 평가 결과 해석 가이드와 갭 분석 체계를 계획가에게 전달
- 계획가 → 모든 정보를 종합하여 개발 계획과 매트릭스 생성

### 단계 3: 통합 및 최종 보고

1. `_workspace/` 내 모든 파일 검토
2. 직무 분석, 역량 사전, 루브릭, 개발 계획 간 일관성 검증
3. 사용자에게 최종 요약 제시

## 산출물

- `02_competency_dictionary.md` — 역량 사전
- `03_assessment_rubric.md` — 평가 루브릭
- `04_development_plan.md` — 역량 개발 계획

## 확장 스킬

- **bars-assessment**: BARS(행동 앵커 평정척도) 평가 도구 설계 전문 스킬
- **competency-modeler**: 역량 모델링 전체 파이프라인
- **ksa-taxonomy**: 직무 분석에서 KSA 분류체계 구축 및 NCS/O*NET 매핑

## 오류 처리

| 오류 유형 | 대응 전략 |
|---------|---------|
| 직무 정보 부족 | 웹 검색으로 유사 직무 공고 수집, 사용자에게 추가 정보 요청 |
| 높은 산업 특수성 | NCS/O*NET 기준 참고, 산업별 용어 사전 활용 |
| 조직 정보 미제공 | 중견 기업 기본값 적용, 맞춤화 가이드 제공 |
| 다중 직무 요청 | 핵심 직무 우선 처리, 나머지는 매트릭스에 프레임워크 제공 |
| 에이전트 실패 | 1회 재시도 → 해당 산출물 없이 진행 |
