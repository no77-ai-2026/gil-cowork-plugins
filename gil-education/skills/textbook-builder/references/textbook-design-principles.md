# 교재 설계 원칙

> gil-education | textbook-builder 참조 | Bloom·ADDIE·UDL

## 1. ADDIE 모델 (교재 개발 표준)

```
Analysis (분석)
   ↓
Design (설계)
   ↓
Development (개발)
   ↓
Implementation (실행)
   ↓
Evaluation (평가) → Analysis 재순환
```

### Analysis (분석)
- 학습자 (연령·수준·목표·시간)
- 학습 환경 (온라인·오프라인·자기 학습)
- 자원 (예산·기술·시간)
- 제약 (저작권·언어·문화)

### Design (설계)
- 학습 목표 (Bloom)
- 단원 구조
- 평가 방법
- 학습 활동 유형

### Development (개발)
- 콘텐츠 작성
- 시각 자료 제작
- 평가 문항
- 교사용 지도서

### Implementation (실행)
- 시범 적용
- 교사 훈련
- 학습자 피드백 수집

### Evaluation (평가)
- 학습 효과 측정
- 교재 개선 사항
- 다음 버전 반영

## 2. Bloom 분류법 (학습 목표)

| 단계 | 동사 (한국어) | 활동 |
|------|-------------|------|
| 기억 | 정의·나열·이름·식별 | 플래시카드, 자동 퀴즈 |
| 이해 | 설명·요약·비교·분류 | 챗봇, 요약 작성 |
| 적용 | 사용·실행·구현·해결 | 실습, 시뮬레이션 |
| 분석 | 분류·구분·비교·체계화 | 데이터 분석, 도표 |
| 평가 | 판단·비판·정당화·평가 | 토론, 피어 리뷰 |
| 창조 | 설계·창작·생성·구성 | 프로젝트, 포트폴리오 |

좋은 교재: 단원별 6단계 균형 + 점진적 상승.

## 3. UDL (Universal Design for Learning)

모든 학습자 접근성:

### 다양한 표현 방법
- 시각 (그림·도표)
- 청각 (오디오·비디오)
- 텍스트
- 인터랙티브

### 다양한 행동·표현
- 쓰기 vs 말하기
- 그리기 vs 디지털
- 개인 vs 그룹

### 다양한 참여
- 자기 주도 vs 가이드
- 게임화 (gamification)
- 보상 시스템

## 4. 학습자 중심 설계 원칙

### 사전 지식 활용
- 진단 평가
- 사전 지식 → 새 학습 연결

### 점진적 난이도
- Scaffolding (비계 설정)
- 단순 → 복잡
- 구체 → 추상

### 활동·실습 우선 (Active Learning)
- 듣기·읽기 < 행함
- 80/20: 80% 활동, 20% 강의

### 즉시 피드백
- 자동 채점
- 오답 해설
- 학습 강화

### 의미 있는 맥락
- 실생활 연결
- 사례·시나리오
- 학습자 관심사

## 5. 시각 디자인 원칙

### 폰트
- 본문 11~14pt (성인) / 14~18pt (어린이)
- 명조 vs 고딕 (분야별)
- 한 페이지 2종 이내

### 색상
- 60-30-10 (브랜드 컬러)
- 색맹 친화 (red·green 회피 단독)
- WCAG AA 명도 대비

### 여백
- 30%+ 여백 (가독성)
- 단락 간 간격
- 좌우 여백 충분

### 시각 자료
- 1페이지 = 1개념
- 시각 자료 페이지의 30%+
- 캡션 명확

## 6. 교재 형식

### 인쇄
- A4 표준 (한국·국제)
- B5 (휴대성)
- 컬러 vs 흑백 (비용)

### 디지털
- DOCX·PDF (보편)
- HTML·EPUB (반응형)
- 인터랙티브 (H5P·Articulate)

### 오디오·비디오
- Podcast 형식
- YouTube 강의

## 7. 평가 통합

각 단원별:
- 진단 평가 (사전)
- 형성 평가 (단원 중)
- 총괄 평가 (단원 후)
- 자기 평가 체크리스트

## 8. 교사용 지도서

### 필수 항목
- 단원별 차시 계획
- 학습 목표
- 교수 방법
- 보조 자료
- 평가 기준 (rubric)
- 학습자 오개념·해결
- 답안·해설

### 형식
- 학습자용과 같은 분량 또는 약간 적게
- 페이지 매칭 (학습자 p1 = 교사 p1)
