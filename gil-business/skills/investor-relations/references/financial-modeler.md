# 재무 모델러 (financial-modeler)

> MoAI-Cowork v1.0.0 하네스 참고자료

## 개요

수익 모델, 비용 구조, 시나리오 분석, 가치 평가를 통해 전체 재무 모델링 라이프사이클을 협력하여 수행하는 하네스입니다.

## 전문가 역할

- **비용 분석가**: 비용 구조 분석 전문가, 고정/변동 비용 분류, 손익분기점 계산 비용 구조 설계, 비용 최적화 전략 도출
  - 비용 분류: 고정/변동 비용 분류
  - 비용 구조 설계: 손익분기점 계산 비용 구조 설계
  - 손익분기점 분석: Break-Even Point 분석
  - 비용 확장 모델: 규모별 비용 변화 모델
  - 마진 분석: 매출총이익, 영업이익 마진 분석

- **모델 검수자**: QA 역할, 수익 모델-비용 구조-시나리오-가치 평가 전반의 공식 정확성, 가정 일관성, 논리 타당성 교차 검증
  - 공식 정확성 검증: 공식과 계산의 정확성 확인
  - 가정 일관성 검증: 가정이 모든 부분에서 일치하는지 확인
  - 논리 타당성 검증: 모델의 논리적 타당성 확인
  - 시나리오 간 일관성: 시나리오별 일관성 확인
  - 통합 재무 요약 편집: 최종 재무 요약 편집

- **수익 모델러**: 수익 모델 설계 전문가, 비즈니스 수익 흐름 정의, 가격 전략, 판매량 추정, 성장률 시나리오를 통해 수익 예측 모델 구축
  - 수익 흐름 정의: 비즈니스 수익 흐름 명확히 정의
  - 가격 전략 분석: 가격 책정 전략 분석
  - 판매량 추정: 판매량 추정 방법론
  - 성장률 모델: 성장률 시나리오 설계
  - 단위경제: 단위당 이익, CAC, LTV 등 단위경제 분석

- **시나리오 계획자**: 재무 시나리오 분석 전문가, 기본/상향/하향 시나리오 구성, 주요 변수에 대한 민감도 분석으로 재무 성과 변동 범위 계산
  - 3개 시나리오 구성: 기본/상향/하향 시나리오 개발
  - 민감도 분석: 주요 변수별 민감도 분석
  - Tornado 차트 데이터: 영향도 분석을 위한 Tornado 차트 데이터
  - 시나리오별 재무 명세: 각 시나리오의 재무 명세
  - 확률 가중 예측: 확률을 고려한 통합 예측

- **가치 평가 전문가**: 가치 평가 전문가, DCF, 배수, 유사 기업 분석 등 다양한 가치 평가 방법론 적용하여 기업가치 계산, 투자 의사결정 기초 제공
  - DCF 분석: 현금흐름 할인법 가치 평가
  - 배수 가치 평가: 수익배수, 성장배수 등을 이용한 가치 평가
  - 유사 기업 분석: 비교 가능 기업 분석
  - 가치 평가 통합: 다양한 방법론의 가치 평가 통합
  - 투자 수익 분석: 투자 수익률 분석

## 워크플로우

### Phase 1: 준비 (오케스트레이터 직접 수행)

1. 사용자 입력에서 추출:
   - **비즈니스 정보**: 산업, 비즈니스 모델, 제품/서비스, 현재 단계(아이디어/MVP/PMF/확장)
   - **재무 데이터** (선택): 현재 수익, 비용, 투자액 등
   - **분석 목적**: 자금조달, 내부 계획, M&A, 비즈니스 실행 가능성 검증 등
   - **기존 파일** (선택): 기존 재무 모델, 비즈니스 계획 등

2. `_workspace/` 디렉토리 생성, 입력을 `_workspace/00_input.md`로 저장
3. 기존 파일 제공 시 `_workspace/`로 복사 후 해당 Phase 건너뛰기

### Phase 2: 팀 편성 및 실행

| 순서 | 작업 | 담당자 | 의존사항 | 산출물 |
|------|------|--------|---------|--------|
| 1a | 수익 모델 | 수익 모델러 | 없음 | `_workspace/01_revenue_model.md` |
| 1b | 비용 구조 | 비용 분석가 | 작업 1a | `_workspace/02_cost_structure.md` |
| 2 | 시나리오 분석 | 시나리오 계획자 | 작업 1a, 1b | `_workspace/03_scenario_analysis.md` |
| 3 | 가치 평가 | 가치 평가 전문가 | 작업 2 | `_workspace/04_valuation_report.md` |
| 4 | 통합 검수 | 모델 검수자 | 작업 1-3 | `_workspace/05_financial_summary.md` |

**에이전트 간 커뮤니케이션:**
- 수익 모델러 완료 → 수익 흐름 구조, 판매 규모 비용 분석가에게, 핵심 가정 시나리오 계획자에게 전달
- 비용 분석가 완료 → 고정/변동 비용 구조 시나리오 계획자에게 전달
- 시나리오 계획자 완료 → 시나리오별 재무 명세, 확률 가중 예측 가치 평가 전문가에게 전달
- 검수자 모든 산출물 교차 검증, 🔴 반드시 수정할 항목 발견 시 수정 요청 (최대 2회)

### Phase 3: 통합 및 최종 산출물

1. `_workspace/` 내 모든 파일 검증
2. 모든 🔴 반드시 수정할 항목 해결 확인
3. 사용자에게 최종 요약 보고

## 산출물

`_workspace/` 디렉토리에 저장된 모든 결과물:
- `00_input.md` — 정리된 사용자 입력
- `01_revenue_model.md` — 수익 모델
- `02_cost_structure.md` — 비용 구조
- `03_scenario_analysis.md` — 시나리오 분석 결과
- `04_valuation_report.md` — 가치 평가 리포트
- `05_financial_summary.md` — 재무 요약 및 검수 리포트

## 에러 처리

| 에러 유형 | 대응 전략 |
|---------|---------|
| 불충분한 시장 데이터 | 유사 산업 벤치마크 적용, "추정치" 표시 |
| 불명확한 비즈니스 모델 | 3가지 수익 모델 제안, 사용자 선택 요청 |
| 에이전트 실패 | 1회 재시도 → 계속 실패 시 진행, 리포트에 기록 |
| 수치 불일치 발견 | 원점으로 추적, 수치 수정, 수정 이력 기록 |
| 🔴 검수 항목 발견 | 수정 요청 → 재작업 → 재검증 (최대 2회) |
