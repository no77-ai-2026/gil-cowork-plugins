# KB(지식베이스) 문서 작성 표준

> gil-support | kb-article 참조 가이드 | 2026 기준

## 개요

지식베이스(Knowledge Base) 문서는 고객 셀프서비스와 상담사 응대 품질을 동시에 향상시킵니다. 잘 작성된 KB는 반복 문의를 줄이고 고객 만족도를 높이는 핵심 자산입니다.

---

## 1. KB 문서 유형

| 유형 | 구조 | 사용 목적 |
|------|------|-----------|
| How-To 가이드 | 단계별 절차 | 기능 사용법, 설정 방법 |
| FAQ | 질문-답변 쌍 | 자주 묻는 질문 모음 |
| 문제 해결 가이드 | 증상-원인-해결 | 오류 처리, 트러블슈팅 |
| 개념 설명 | 정의+예시 | 제품 개념, 정책 설명 |
| 참조 문서 | 표/목록 | 스펙, 요금, 약관 요약 |

---

## 2. FAQ 구조 원칙

### 좋은 FAQ 질문 작성법

- 고객이 실제로 사용하는 말로 작성 (기술 용어 지양)
- 질문형 문장으로 시작: "~하려면 어떻게 하나요?"
- 하나의 질문에 하나의 답변 (복합 주제 분리)
- 키워드가 질문 앞쪽에 위치 (검색 최적화)

### FAQ 작성 체크리스트

- [ ] 질문이 실제 고객 문의 데이터 기반인가?
- [ ] 답변이 200자 이내로 핵심만 담겼는가?
- [ ] 필요한 경우 관련 문서 링크가 포함되었는가?
- [ ] 마지막 업데이트 날짜가 표기되었는가?

## 3. How-To 가이드 구조

```
제목: [목표 행동] 방법 (예: "비밀번호 변경하는 방법")

난이도: 쉬움 / 보통 / 어려움
소요 시간: 약 X분
사전 조건: [필요한 경우만 작성]

단계별 안내:
1. [동작 + 위치] - "설정 메뉴를 탭합니다"
2. [동작 + 결과] - "비밀번호 입력란에 새 비밀번호를 입력하면 ✓ 표시가 나타납니다"
3. ...

완료 확인: [성공 시 나타나는 결과 설명]

문제가 생겼나요? → [트러블슈팅 문서 링크] 또는 고객센터 연락처
```

---

## 4. 검색 최적화 원칙

### 내부 검색 최적화

- 제목에 핵심 키워드 포함 (고객이 검색할 단어)
- 유의어 태그 설정: "취소"라면 "해지", "탈퇴", "환불" 태그 추가
- 관련 문서 링크: 하단에 "이 내용과 관련된 도움말" 섹션

### 외부 검색(구글·네이버) 최적화

- 문서 URL: 영문 슬러그 사용 (/how-to-reset-password)
- 메타 설명: 150자 이내, 핵심 행동과 결과 포함
- FAQ 스키마 마크업 적용 (헬프센터 솔루션에서 지원 여부 확인)

## 5. 용어 통일 원칙

### 용어 표준화

| 금지 표현 | 표준 표현 | 이유 |
|-----------|-----------|------|
| 유저 / 사용자 | 고객님 (공개 KB) / 사용자 (내부 KB) | 일관성 |
| 클릭 / 탭 / 누르기 | 탭합니다 (모바일), 클릭합니다 (PC) | 플랫폼 구분 |
| 에러 / 오류 | 오류 | 한국어 표준 |
| 로그아웃 / 로그오프 | 로그아웃 | 브랜드 통일 |

### 용어집 관리

- 서비스별 용어집 별도 파일로 관리
- 신규 용어 추가 시 팀 리뷰 필수
- 분기 1회 용어집 유효성 검토

---

## 6. 문서 품질 체크리스트

- [ ] 제목이 고객 질문 또는 목표 행동을 명확히 표현
- [ ] 200자 이내 리드 문장으로 문서 내용 요약
- [ ] 단계별 순서 번호 사용 (How-To 문서)
- [ ] 스크린샷 또는 예시 이미지 포함 (필요 시)
- [ ] 관련 문서 링크 2-3개 포함
- [ ] 마지막 업데이트 날짜 표기
- [ ] 내부 리뷰 완료 후 발행
