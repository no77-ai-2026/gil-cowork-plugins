# 피드백 분석 (93-feedback-analyzer)

> MoAI-Cowork v1.0.0 하네스 참고 자료

## 개요
고객/임직원 피드백(수집→정서 분석→주제 분류→추세 탐지→인사이트 보고)을 에이전트 팀이 협업하여 수행하는 하네스입니다.

## 전문가 역할

- **데이터 수집가**: 피드백 수집, 정제, 정규화, 익명화 전문가. 여러 채널의 피드백을 통합하여 분석 가능한 데이터셋 구축
  - 다채널 통합: 설문, 앱 리뷰, 지원 티켓, 인터뷰 노트, SNS 언급 등을 단일 형식으로 통합
  - 데이터 정제: 중복 제거, 스팸 필터링, 의미 없는 응답 제거
  - 정규화: 날짜 형식, 평점 척도(5점/10점/100점→5점), 텍스트 인코딩 통일
  - 메타데이터 태깅: 각 피드백에 채널, 날짜, 고객/임직원 분류, 제품/서비스 영역 태그
  - 기초 통계: 데이터 개수, 채널별 분포, 기간별 분포, 응답률 계산

- **감정 분석가**: 정서 분석 및 감정 분류 전문가. 피드백의 긍정/중립/부정 감정을 정량화
  - 감정 점수: 각 피드백의 감정을 -1.0(매우 부정) ~ +1.0(매우 긍정)으로 점수화
  - 감정 분포: 전체 피드백 중 긍정/중립/부정 비율 분석
  - 감정 추이: 시간 경과에 따른 감정 변화 추적
  - 세그먼트별 감정: 고객 유형, 제품, 지역별 감정 편차 분석
  - 언어 특성: 강한 어조, 반복 표현, 이모지 등 감정 신호 파악

- **주제 분류가**: 피드백의 주제를 자동으로 분류하고 태그화하는 전문가
  - 주제 정의: 상품 품질, 가격, 배송, 고객 지원, UI/UX 등 핵심 주제 정의
  - 자동 분류: 키워드, 규칙 기반 또는 머신러닝 분류기로 주제 자동 할당
  - 다중 태그: 한 피드백이 여러 주제에 해당할 경우 다중 태그 적용
  - 신규 주제 탐지: 분류되지 않는 피드백에서 새로운 주제 발견
  - 주제별 분포: 각 주제의 언급 빈도 및 감정 점수 분석

- **추세 탐지가**: 시계열 분석으로 패턴, 이상 현상, 개선/악화 신호 탐지
  - 시간 추이 분석: 월별, 주별 피드백 량과 감정 추이 관찰
  - 계절성 탐지: 특정 기간의 반복 패턴 파악
  - 이상 탐지: 정상 범위를 벗어난 급증/급감 신호 알림
  - 상관 분석: 이벤트(상품 출시, 업데이트, 홍보)와 피드백 변화 연관성 분석
  - 예측: 추이 지속 시 향후 3개월 감정/주제 추이 예측

- **인사이트 작성가**: 감정, 주제, 추세 분석 결과를 종합하여 실행 가능한 인사이트 보고서 작성
  - 인사이트 도출: 감정, 주제, 추세 분석 결과를 종합하여 핵심 인사이트 도출
  - 실행 항목: 각 인사이트에 대해 구체적이고 실행 가능한 항목 제시
  - 우선순위 매트릭스: 영향도 × 실행 용이도로 실행 항목 우선순위 결정
  - 대상별 보고서: 경영진 1페이지 요약, 실무자 상세 보고서 분리 작성
  - 모니터링 대시보드: 향후 피드백 관리를 위한 KPI 및 대시보드 구성 제안

## 워크플로우

### Phase 1: 준비 (오케스트레이터 직접 수행)

1. 사용자 입력에서 추출:
   - **분석 대상**: 고객/임직원, 제품/서비스
   - **피드백 소스**: 데이터 위치, 기간 범위
   - **분석 목표**: 개선할 영역, 의사결정 필요 사항
   - **기존 데이터** (선택): 기존 피드백 파일

2. `_workspace/` 디렉토리 생성
3. 입력값 정리하여 `_workspace/00_input.md`에 저장
4. 요청 범위에 따라 **실행 모드 결정**

### Phase 2: 팀 구성 및 실행

| 순서 | 작업 | 담당 | 의존성 | 산출물 |
|------|------|------|--------|--------|
| 1 | 피드백 수집 및 정제 | 수집가 | 없음 | `_workspace/01_cleaned_dataset.md` |
| 2a | 감정 분석 | 감정 분석가 | 작업 1 | `_workspace/02_sentiment_analysis.md` |
| 2b | 주제 분류 | 분류가 | 작업 1 | `_workspace/03_topic_classification.md` |
| 3 | 추세 탐지 | 추세 탐지가 | 작업 2a, 2b | `_workspace/04_trend_analysis.md` |
| 4 | 인사이트 보고 | 인사이트 작성가 | 작업 2a, 2b, 3 | `_workspace/05_insight_report.md` |

### Phase 3: 통합 및 최종 산출

1. 모든 분석 결과 검증
2. 인사이트 보고서 최종 검토
3. 모니터링 대시보드 설계 공유

## 산출물

모든 산출물은 `_workspace/`에 저장됩니다:
- `00_input.md` — 사용자 입력 정리
- `01_cleaned_dataset.md` — 정제된 피드백 데이터셋 및 통계
- `02_sentiment_analysis.md` — 감정 분석 결과 및 시각화
- `03_topic_classification.md` — 주제별 분류 및 분포
- `04_trend_analysis.md` — 추세 분석 및 이상 탐지 결과
- `05_insight_report.md` — 최종 인사이트 및 실행 항목

## 오류 처리

| 오류 유형 | 대응 전략 |
|----------|----------|
| 피드백 데이터 부족 | 3개월치 가정 데이터 기반 샘플 분석 수행 |
| 언어 다국어 | 주요 언어별로 별도 분석 수행 |

| 주제 분류 불명확 | 초기 주제 재정의 후 재분류 |
| 트렌드 부족 | 스냅샷 분석으로 현황 평가 |
