# 고객 지원 시스템 (49-customer-support)

> MoAI-Cowork v1.0.0 하네스 참고 자료

## 개요
고객 지원 시스템(FAQ→응답 매뉴얼→에스컬레이션→분석)을 에이전트 팀이 협업하여 구축하는 하네스입니다.

## 전문가 역할

- **CS 분석가**: 고객 지원 성과 지표, VOC 분석, 추세 분석, 개선 제안 전문가
  - 지표 체계 설계: CSAT, NPS, FCR, AHT 등 주요 CS 지표 정의 및 측정 방법
  - VOC 분석 프레임워크: 고객 피드백 수집, 분류, 분석 체계 설계
  - 추세 분석: 문의 유형 추세, 계절 패턴, 이상 현상 탐지 프레임워크
  - 개선 제안: 데이터 기반 개선 기회 파악 및 우선순위 제시
  - 보고 체계 설계: 일일/주간/월간 CS 보고서 및 대시보드 항목 정의

- **CS 검토자**: 고객 지원 시스템 일관성 검증 및 고객 경험 관점의 완전성 평가
  - FAQ ↔ 매뉴얼 일관성: FAQ 답변과 응답 매뉴얼 스크립트 정보 동일성 검증
  - 매뉴얼 ↔ 에스컬레이션 일관성: 에스컬레이션 트리거가 매뉴얼에 올바르게 반영되었는지 확인
  - 문의 커버리지: 예상되는 주요 문의가 FAQ 또는 매뉴얼에 포함되었는지 검증
  - 기술 용어 일관성: 문서 전체에서 동일한 용어 사용 확인
  - 고객 경험 흐름: 고객 관점에서 매끄러운 경험 확인

- **FAQ 개발자**: 예상 질문과 답변을 조직화하여 고객 자가 서비스를 강화하는 FAQ 작성
  - 질문 예상: 상품/서비스 관련 빈번한 질문 예상
  - 카테고리 정리: 질문을 구매 전→구매→사용→문제 단계별로 정리
  - 명확한 답변: 기술 용어 최소화, 단계별 지시 포함
  - 시각적 도구: 흐름도, 스크린샷, 비디오 링크 활용
  - 검색 최적화: 고객이 쉽게 찾을 수 있도록 키워드 포함

- **응답 매뉴얼 개발자**: 고객 문의, 불만, 문제별로 일관된 응답을 제공하는 응답 템플릿 및 스크립트 작성
  - 시나리오별 응답: 일반 문의→기술 문제→배송 문제→결함 상품 등 상황별 응답
  - 응답 구조: 공감→이해→해결책→후속 조치 순서의 일관된 구조
  - 다국어 지원: 고객 언어별 응답 템플릿
  - 톤 & 매너: 회사 브랜드에 맞는 응답 톤 유지
  - 에스컬레이션 기준: 어떤 경우 고객 지원 담당자에게 넘길지 명시

- **CS 시스템 설계자**: FAQ, 응답 매뉴얼, 에스컬레이션, 분석을 통합하는 고객 지원 운영 체계 설계
  - 채널별 운영: 이메일, 채팅, 전화, SNS 별 운영 절차
  - 대응 시간 기준: 각 채널별 1차 응답, 해결 시간 목표 설정
  - 에스컬레이션 정책: 어떤 경우 누가 어떻게 개입할지 정의
  - 팀 역할: CS 담당자, 전문가, 관리자의 책임 분명히
  - KPI 정의: 응답 시간, 해결율, 고객만족도 측정 방식

## 워크플로우

### Phase 1: 준비 (오케스트레이터 직접 수행)

1. 사용자 입력에서 추출:
   - **상품/서비스**: 지원 대상
   - **현황**: 기존 CS 운영 현황, 문제점
   - **고객**: 주요 고객층, 문의 유형 예상
   - **기존 자료** (선택): 기존 FAQ, 응답 템플릿

2. `_workspace/` 디렉토리 생성
3. 입력값 정리하여 `_workspace/00_input.md`에 저장
4. 요청 범위에 따라 **실행 모드 결정**

### Phase 2: 팀 구성 및 실행

| 순서 | 작업 | 담당 | 의존성 | 산출물 |
|------|------|------|--------|--------|
| 1 | 시스템 설계 | 설계자 | 없음 | `_workspace/01_cs_system_design.md` |
| 2a | FAQ 개발 | FAQ 개발자 | 작업 1 | `_workspace/02_faq.md` |
| 2b | 응답 매뉴얼 | 응답 개발자 | 작업 1 | `_workspace/03_response_manual.md` |
| 3 | 분석 체계 | 분석가 | 작업 1 | `_workspace/04_analytics_system.md` |
| 4 | 최종 검토 | 검토자 | 작업 2a, 2b, 3 | `_workspace/05_review_report.md` |

### Phase 3: 통합 및 최종 산출

1. 모든 파일 검증 및 일관성 확인
2. 최종 CS 시스템 가이드 작성
3. 사용자에 최종 요약 보고

## 산출물

모든 산출물은 `_workspace/`에 저장됩니다:
- `00_input.md` — 사용자 입력 정리
- `01_cs_system_design.md` — CS 시스템 설계
- `02_faq.md` — FAQ 및 자가 서비스 가이드
- `03_response_manual.md` — 상황별 응답 매뉴얼 및 스크립트
- `04_analytics_system.md` — CS 분석 및 KPI 체계
- `05_review_report.md` — 최종 검토 및 개선 제안

## 오류 처리

| 오류 유형 | 대응 전략 |
|----------|----------|
| 상품 정보 부족 | 설계자가 업종별 표준 FAQ 템플릿 제공 |
| 문의 유형 미상 | 분석가가 3개월 가정 데이터 기반 시나리오 제안 |
| 기존 자료 없음 | 일반적인 상품/서비스 기준으로 FAQ 신규 작성 |
| 에스컬레이션 정책 불명확 | 설계자가 3가지 모델 제안 (즉시/내일/외주) |
| 시스템 복잡도 높음 | 단계적 도입 계획 수립 |
