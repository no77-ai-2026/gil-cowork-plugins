# UZ 데이터 보호법·데이터 로컬리제이션 가이드

> gil-data v0.1.0 | UZ Закон о персональных данных + 한·UZ 데이터 처리

## 개요

UZ Закон о персональных данных (2019)는 UZ 시민 개인정보의 UZ 내 1차 저장을 의무화. 한국 회사가 UZ 진출·UZ 자회사 운영 시 양국 데이터 보호법 준수 필수.

---

## 1. UZ 데이터 보호법 핵심

### 명칭
**Закон Республики Узбекистан "О персональных данных"** (개인정보보호법, 2019년 시행)

### 주요 조항

#### 데이터 로컬리제이션 (제27조)

- **UZ 시민 개인정보는 UZ 영토 내 서버에 1차 저장 의무**
- 처리·저장이 UZ 내 데이터센터에서 진행
- 해외 서버는 보조 (백업·이중화)

#### 해외 송신 (제29조)

- UZ 시민 개인정보의 해외 송신 시 정부 통보·동의
- 적합 보호 수준 보장 국가만 가능
- 데이터 주체 동의

#### 데이터 주체 권리

- 정보 수집·이용·제3자 제공 동의
- 열람·정정·삭제 요청
- 처리 중지 요청

### 위반 시 제재

- 행정 책임: 벌금 (회사·임원)
- 형사 책임: 심각한 위반 (5년 이하)
- 운영 정지·라이선스 취소

## 3. 한·UZ 양국 운영 시 점검

### 한국 본사 ↔ UZ 자회사

| 데이터 | 처리 위치 | 비고 |
|--------|---------|------|
| 한국 직원·한국 고객 정보 | 한국 (가능) | 한국 PIPA |
| UZ 직원·UZ 고객 정보 | UZ 내 1차 | UZ 법 |
| 양국 인터컴퍼니 거래 | 양국 백업 | 양국 동의 |
| 매출·재무 (집계) | 양국 가능 | 익명화 |

### 양국 동의 양식

```
[한국 PIPA + UZ Закон о персональных данных 양국 동의]

1. 수집·이용 (한국·UZ)
2. 제3자 제공 (한국·UZ)
3. 해외 이전 (한국 → UZ, UZ → 한국)
4. 위탁 처리
5. 보유·이용 기간
6. 삭제·정정 권리
```

### 한국 PIPA (참고)

- 개인정보 수집·이용 동의
- 해외 이전 별도 동의
- 개인정보보호위원회 (PIPC) 감독

## 4. 데이터 분석 시 보호 조치

### PII 마스킹

```python
import pandas as pd

# 이름·전화·이메일·INN 마스킹
def mask_pii(df):
    if 'name' in df.columns:
        df['name'] = df['name'].str[:1] + '*' * (df['name'].str.len() - 1)
    if 'phone' in df.columns:
        df['phone'] = df['phone'].str[:3] + '****' + df['phone'].str[-4:]
    if 'email' in df.columns:
        df['email'] = df['email'].str.split('@').str[0].str[:2] + '***@' + df['email'].str.split('@').str[1]
    if 'inn' in df.columns:
        df['inn'] = df['inn'].str[:3] + '*****' + df['inn'].str[-1:]
    return df
```

# 그룹 사이즈 5+ 보장

## 7. 한·UZ 인터컴퍼니 데이터 흐름

### 권장 구조

```
[한국 본사] ↔ [UZ 자회사]

한국 본사:
- 한국 직원·고객 데이터 (한국 클라우드)
- UZ 매출 집계 (USD 환산, 익명화)

UZ 자회사:
- UZ 직원·고객 데이터 (UZ 클라우드)
- UZ 거래 데이터 (UZ 내 1차 저장)

양국 공유:
- 익명화·집계 데이터만
- 양국 동의 후 PII 일부 (필요 시)
```

### 데이터 이전 절차

```
1. 사전 검토 (양국 법무)
2. 데이터 카테고리 분류
3. 동의 양식 갱신
4. 이전 인프라 구축
5. UZ 정부 통보 (해당 시)
6. 정기 감사
```

## 11. 자주 하는 질문

### Q: 한국 본사가 UZ 자회사 데이터를 본사 서버에 통합할 수 있나?

A: UZ 시민 데이터는 UZ 내 1차 저장 의무. 한국 서버는 백업·이중화만. 통합 분석 시 익명화·집계 후 한국 서버.

### Q: 양국 데이터 통합 대시보드는?

A: 익명화·집계 데이터만 통합. PII는 각국 서버에 보관. 시각화 시 합계·평균만 표시.

### Q: AWS·Google Cloud는 UZ 시민 데이터에 사용 가능?

A: UZ 데이터센터 없음 → UZ 법 위반 위험. UZ 내 클라우드 (Yandex·정부) 또는 자체 서버 권장.

### Q: 한·UZ 양국 마케팅 캠페인 데이터?

A: 양국 동의 양식 + 양국 PII 분리 + 집계만 통합.
