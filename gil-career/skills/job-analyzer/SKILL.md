# 채용공고 분석기 (job-analyzer)

> 2026 한국 채용 시장은 **소규모 핀셋(정밀) 채용**으로 전환되었습니다. "수십 명 뽑는 공고에 일단 지원" → "지원 이전 단계에서 지원할지 말지 결정"하는 구조로 바뀌었으며, 본 스킬은 그 결정에 필요한 JD 분해 + 기업 리서치 + 헤드헌터 오퍼 검증을 한 번에 처리합니다.

## 지원 영역

| 영역 | 설명 |
|------|------|
| JD 분석 | 채용공고에서 필수/우대 역량 추출, 키워드 매핑 |
| 경험 매칭 | 내 경험과 JD 역량 매칭 → 자소서 소재 발굴 |
| 기업 분석 | 기업 문화, 최근 동향, 성장성, 경쟁사 비교 |
| 전략 수립 | 지원 전략 로드맵, 약점 보완 계획 |

참조 가이드: `references/jd-analysis-guide.md`, `references/realtime-patterns.md`

## 실전 분석 패턴 (한국 채용 시장)

### 1. 차별화 JD 매칭 4단계

대다수 지원자는 JD를 단순 요약. 합격자는 4단계로 분해.

```
[1] JD 키워드 추출 — 명사·동사·기술명 (15~20개)
[2] 키워드 우선순위 — 본문 첫 단락 빈도 5점, 우대사항 2점
[3] 본인 경험과 매칭 — 키워드 1개당 1 경험 (CAR 형식)
[4] 매칭 점수 70%+ 시 차별화 메시지 도출
```

매칭 점수가 70% 미만이면 다른 회사 우선 또는 갭 메우는 전략.

### 2. 기업분석 2탭 프레임 — 홈페이지 + DART

면접·자소서의 차별점은 기업분석 깊이.

| 1탭: 홈페이지 (감성·전략) | 2탭: DART (객관·재무) |
|---|---|
| 회사 비전·미션 | 매출·영업이익 3년 트렌드 |
| 사업 영역·신사업 | 사업부문별 매출 비중 |
| 인재상·문화 | R&D 투자·신규 채용 규모 |
| 최근 1년 보도자료 | 임원진 변동 |
| 채용 페이지·복지 | 경쟁사 비교 (감사보고서) |

DART에서 본 정보를 자소서·면접에 인용하면 깊이가 다름.

### 3. 채용사이트 분석으로 차별화

- 같은 회사 다른 직무 채용공고 = 회사 현재 우선순위
- 6개월-1년 채용 이력 = 조직 성장 속도·이직률
- 우대사항의 변화 = 직무 진화 방향
- 채용 광고비 투자 = 진정성·시급성

### 4. 직무 역량 파악 5단계

```
[1] 본인이 한 일 모두 나열 (3년치)
[2] 일을 행동 동사로 변환 ("회의 진행" → "이해관계자 조율")
[3] 행동에서 역량 도출 ("조율" → "커뮤니케이션·조정 역량")
[4] 역량 6~8개로 압축 + 우선순위
[5] JD 매칭 + 갭 분석
```

### 5. 수시채용 대응 (대기업·스타트업 공통)

수시채용 비중 70%+ (대기업 인사담당자 기준):

- **항상 준비된 이력서** (월 1회 업데이트)
- **타깃 회사 5-10개** + 채용 알림 설정
- **인맥 활용** — LinkedIn·전 직장 동료
- **빠른 대응** — 공고 발견 후 48시간 내 지원
- **컴팩트 자소서** (500자 버전 미리 작성)

### 6. 외국계 기업 합격 패턴

- **링크드인 프로필 우선** (이력서보다 중요)
- **영문 CV 표준 + JD별 커스터마이즈**
- **레퍼런스 체크 강함** — 전 직장 평판 핵심
- **인터뷰 5-7회** (한국 대기업 2-3회 대비 많음)
- **연봉 협상 직접** (헤드헌터보다 본인 협상)

### 7. 회계사가 알려주는 입사 전 재무 체크 포인트

- 최근 3년 영업이익률 (5% 미만 위험)
- 자본잠식 여부 (총자본 > 총부채 확인)
- 부채비율 (200% 이상 주의)
- 영업활동현금흐름 양수 여부
- 감사의견 — "한정·부적정·의견거절" 즉시 회피

## JD 분석 프레임워크

### 1단계: 구조 분해

채용공고를 다음 요소로 분해합니다.

```
[직무 개요]     → 핵심 역할과 포지션 레벨 파악
[담당 업무]     → 일상 업무 vs 프로젝트성 업무 구분
[필수 자격]     → MUST-HAVE: 없으면 지원 불가
[우대 사항]     → NICE-TO-HAVE: 있으면 차별화
[근무 조건]     → 계약 형태, 위치, 보상 구조
[기업 문화]     → 복지, 팀 분위기, 성장 지원
```

### 2단계: 역량 추출 및 분류

| 분류 | 예시 | 가중치 |
|------|------|--------|
| 하드 스킬 | Python, SQL, Figma, 재무 분석 | 높음 |
| 소프트 스킬 | 커뮤니케이션, 리더십, 문제해결 | 중간 |
| 도메인 지식 | 핀테크, 이커머스, 헬스케어 | 높음 |
| 자격/인증 | SQLD, CPA, 정보처리기사 | 조건별 |

### 3단계: 경험 매칭 매트릭스

```
역량 항목    | JD 요구 수준 | 내 보유 수준 | 갭  | 자소서 소재
─────────────┼─────────────┼─────────────┼─────┼──────────────
Python       | 상           | 중           | ▲   | 프로젝트 X 경험
데이터 분석   | 중           | 상           | ◎   | 인턴 성과 강조
팀 리더십     | 중           | 중           | ─   | 동아리 회장 경험
```

- ◎ 강점: 자소서에서 적극 어필
- ─ 충족: 간결하게 언급
- ▲ 갭: 학습 의지와 보완 계획 제시

## 기업 분석 체크리스트

| 항목 | 확인 내용 | 소스 |
|------|----------|------|
| 사업 모델 | 주요 매출원, B2B/B2C, 핵심 제품 | 사업보고서, IR 자료 |
| 재무 건전성 | 매출 추이, 영업이익률, 투자 라운드 | 전자공시(DART), 뉴스 |
| 조직 문화 | 수평/수직, 원격근무, 복지 | 잡플래닛, 블라인드, 채용 페이지 |
| 최근 동향 | 신사업, 인수합병, 리더십 변화 | 뉴스, 보도자료 |
| 경쟁 환경 | 주요 경쟁사, 시장 점유율, 차별화 | 산업 보고서 |
| 채용 패턴 | 공채/수시, 채용 주기, 선호 인재상 | 과거 공고, 채용 후기 |

## 2026년 채용 시장 핵심 트렌드 (본 스킬 반영)

- **소규모 핀셋(정밀) 채용**: 사람인 조사에서 327개사 65.7%가 2026년 채용 계획 있음 — 단, 신입은 공채보다 수시채용 중심. "한 명을 실패 없이 선점"하는 구조.
- **컬처핏 → 팀핏(Team Fit)**: 조직 전체가 아니라 실제 함께 일할 팀의 업무 스타일·갈등 대응 평가로 이동 (원티드 HR 트렌드 리포트 2026).
- **스킬 기반 채용(Skills First)**: 학벌·자격증 → 실무 과제·포트폴리오·마이크로 자격으로 평가 기준 이동. SSAFY·부스트캠프 비전공자가 명문대 졸업생보다 잘 취업하는 사례 보편화.
- **AI 리터러시 + 진정성 검증**: 챗GPT·제미나이로 포장된 자소서·답변의 진정성 검증이 면접관 41% 숙제 (한국바른채용인증원 설문).
- **타깃 기반 채용**: 지원 이전부터 소수 유력 후보를 압축해 검증. 본 스킬의 JD 분석은 **회사 입장에서 본인이 그 타깃에 들어가는지** 판단 도구로 사용.
- **AI 에이전트 소싱**: 사람인·잡코리아·원티드 모두 AI 매칭 점수가 1차 필터. ATS 키워드 + 정량 성과 노출이 결정적.

## 헤드헌터 오퍼 검증 체크리스트 (사람인·리멤버 시니어 시장)

사람인 출신 헤드헌터가 시장 오퍼의 90% 이상을 차지합니다 ([사람인 vs 잡코리아 2026 비교 참고](https://mykoreawork.com/ko/blog/saramin-jobkorea-paid-posting-comparison-2026)). 본 스킬은 오퍼를 다음 5축으로 검증합니다.

| 검증 축 | 본 스킬 자동 처리 |
|---|---|
| JD 명시성 | 직무·연봉·조직 규모·보고 라인이 구체적인가 (모호하면 추가 질문 리스트) |
| 회사 재무 | DART에서 3년 영업이익률·자본잠식·부채비율·감사의견 자동 점검 |
| 조직 분위기 | 잡플래닛 평점·블라인드 후기 사실/의견 분리 |
| 채용 이력 | 같은 회사 같은 직무 6~12개월 채용 이력 — 이직률 추정 |
| 헤드헌터 신뢰도 | 첫 미팅 = 1차 면접. 영문/국문 이력서 병행 요구 여부, 희망 연봉 ±20% 협상 가능 여부 |

## 독립 실행 워크플로우

참조 가이드(`references/`)를 사용할 수 없는 경우 다음 단계로 실행합니다.

### 1단계: 입력 확인
- 채용공고 텍스트 또는 기업명/직무명 확인
- 사용자 배경 정보 수집 (있는 경우)

### 2단계: JD 구조 분해
- 필수/우대 역량 추출
- 하드 스킬/소프트 스킬/도메인 지식 분류

### 3단계: 기업 리서치 (요청 시)
- 기업 개요, 사업 모델, 재무 상태 정리
- 조직 문화, 최근 뉴스, 채용 패턴 분석

### 4단계: 매칭 및 전략
- 사용자 경험과 JD 역량 매칭
- 강점 어필 포인트 + 갭 보완 전략 제시
- 자소서 소재 추천

## 실행 규칙

1. 사용자 요청 수신 → JD 분석 또는 기업 분석 판별
2. `references/{id}.md` 존재 시 로드 → 가이드에 따라 실행
3. `--deepthink` 또는 복잡한 다기업 비교 분석 → `mcp__sequential-thinking__sequentialthinking` 호출
4. 분석 결과를 표 형태로 정리하여 제시

## 문제 해결

- **채용공고 텍스트 없음**: 기업명과 직무명만 있어도 일반적인 해당 직무 역량 분석이 가능합니다. 정확한 분석을 위해 공고 원문을 붙여넣어 주세요.
- **비공개 기업 정보**: 스타트업이나 중소기업은 공개 정보가 제한적일 수 있습니다. 로켓펀치, 크런치베이스 등을 병행 확인하세요.
- **다수 기업 비교**: 3개 이상 기업을 비교할 때는 비교 기준(연봉, 성장성, 문화 등)을 먼저 정해주세요.
- **오래된 채용공고**: 채용 시장 트렌드는 빠르게 변합니다. 3개월 이상 된 공고는 최신 정보와 다를 수 있습니다.
