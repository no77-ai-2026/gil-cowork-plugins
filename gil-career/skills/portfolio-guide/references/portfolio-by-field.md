# 분야별 포트폴리오 가이드

> 2026년 채용 시장 기준

## 개발자

### 필수 구성

**GitHub 프로필:**
- 프로필 README: 기술 스택 뱃지, 한 줄 소개, 대표 프로젝트 링크
- 핀 프로젝트 3-5개: 가장 자신 있는 프로젝트를 상단에 고정
- 커밋 빈도: 꾸준한 활동 기록 (잔디 채우기보다 의미 있는 커밋)
- 이슈/PR 활동: 코드 리뷰, 버그 리포트 등 협업 흔적

**기술 블로그:**
- 플랫폼: velog, tistory, 개인 블로그(gatsby, next.js)
- 콘텐츠: 문제 해결 과정, 기술 선택 이유, 삽질 기록
- 빈도: 월 2-4회 (꾸준함이 핵심)
- SEO: 제목에 기술 키워드 포함, 시리즈 구성

**프로젝트 문서화:**
```
README 필수 항목:
1. 프로젝트 한 줄 설명
2. 스크린샷 또는 데모 GIF
3. 기술 스택 (아이콘 뱃지)
4. 실행 방법 (로컬 셋업)
5. 아키텍처 (간단한 다이어그램)
6. 본인 기여 부분 (팀 프로젝트 시)
7. 배포 링크 (있는 경우)
```

### 차별화 요소

- 오픈소스 기여: 작은 PR이라도 기록으로 남음
- 사이드 프로젝트: 실제 사용자가 있는 서비스
- 코딩 챌린지: 백준/프로그래머스 풀이 기록
- AI 활용 사례: AI 도구와 협업한 개발 프로세스

## 디자이너

### 필수 구성

**Behance/Dribbble:**
- 프로젝트 5-8개 (양보다 질)
- 커버 이미지: 고품질, 프로젝트 핵심을 한눈에
- 태그: 직무 관련 키워드 (UI, UX, Product Design)
- 설명: 영문+국문 병기 (글로벌 노출)

**Figma 포트폴리오:**
- 컴포넌트 라이브러리: 디자인 시스템 역량 증명
- 프로토타입: 인터랙션, 트랜지션 포함
- 오토 레이아웃: 반응형 설계 이해도
- 변수/토큰: 디자인 토큰 활용 능력

**프로세스 기반 케이스 스터디:**
```
1. 문제 정의 (Why): 어떤 사용자 문제인가
2. 리서치 (Research): 인터뷰, 데이터, 벤치마킹
3. 가설 (Hypothesis): 어떤 가설을 세웠는가
4. 설계 (Design): 와이어프레임 → 프로토타입 → 최종
5. 검증 (Validate): 유저 테스트, A/B 테스트
6. 임팩트 (Impact): 전환율 +15%, 이탈률 -20% 등
```

### 차별화 요소

- AI 활용: Midjourney 페르소나 비주얼, AI 기반 디자인 실험
- 협업 기록: Jira/Confluence 활용, 개발자와의 커뮤니케이션
- 비즈니스 임팩트: 수치로 증명된 디자인 성과
- 다학제 경험: 리서치, 카피, 프론트 구현 역량

## 마케터

### 필수 구성

**캠페인 케이스 (3-5개):**
```
캠페인명: [이름]
기간/예산: YYYY.MM - YYYY.MM / 예산 N만원
목표: 인지도 향상 / 리드 확보 / 매출 증대
전략: 어떤 채널, 어떤 메시지, 어떤 타겟
실행: 콘텐츠 유형, A/B 테스트 내용
성과: CTR 3.2%, CVR 1.8%, ROAS 450%
배운 점: 다음에는 어떻게 개선할 것인가
```

**데이터 분석 사례:**
- GA4 보고서: 사용자 행동 분석, 전환 퍼널
- SQL 쿼리: 데이터 추출/분석 역량
- 대시보드: Looker Studio, Tableau 시각화

**콘텐츠 샘플:**
- SNS 게시물: 인스타그램, 블로그, 뉴스레터
- 성과 지표: 좋아요, 공유, 댓글, 도달률

### 차별화 요소

- 그로스 해킹: AARRR 퍼널 개선 경험
- 퍼포먼스 마케팅: 광고 운영 + 최적화 성과
- 바이럴 콘텐츠: 자체 제작 콘텐츠의 확산 사례

## 기획자 (PM/PO)

### 필수 구성

**기획서 포트폴리오:**
- PRD (Product Requirements Document)
- 기능 명세서 + 와이어프레임
- 사용자 스토리 맵
- 데이터 기반 의사결정 기록

**UX 리서치:**
- 사용자 인터뷰 설계 및 분석
- 설문 조사 결과 인사이트
- 퍼소나 및 사용자 여정 맵
- 유저빌리티 테스트 결과

**프로젝트 운영:**
- 스프린트 플래닝/회고 기록
- 이해관계자 커뮤니케이션 사례
- 우선순위 프레임워크 (RICE, ICE 등)
- 출시 후 성과 측정 (DAU, 리텐션, NPS)

### 차별화 요소

- 0→1 경험: 신규 서비스 론칭 전체 과정
- 데이터 분석: 정량 지표 기반 의사결정
- 크로스펑셔널: 개발/디자인/마케팅 협업 사례
- AI 전략: AI 기능 기획 또는 AI 도구 도입 경험

## 공통 팁

1. **양보다 질**: 프로젝트 3-5개에 집중, 10개 이상은 산만
2. **최신 순 배치**: 가장 최근, 가장 강한 프로젝트를 첫 번째에
3. **성과 수치화**: "잘 했다"가 아닌 "N% 개선", "N명 사용"
4. **읽기 편한 구성**: 스캔 가능한 레이아웃, 핵심 볼드 처리
5. **모바일 확인**: 면접관이 모바일로 볼 수 있음, 반응형 필수
