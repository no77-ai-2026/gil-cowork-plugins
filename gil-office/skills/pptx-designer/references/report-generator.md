# 보고서 생성 (83-report-generator)

> MoAI-Cowork v1.0.0 하네스 레퍼런스

## 개요

보고 요구사항 파악, 자료 수집, 분석, 보고서 작성을 수행하는 에이전트 팀 협업 하네스입니다.

## 전문가 역할

- **요구사항 분석가**: 보고 요구사항을 분석합니다.
  - 목적: 보고서의 목적
  - 대상: 대상 독자
  - 범위: 보고 범위
  - 기한: 보고 기한
  - 형식: 요구 형식

- **자료 수집가**: 필요한 자료를 수집합니다.
  - 데이터: 관련 데이터 수집
  - 정보: 배경 정보 수집
  - 증거: 뒷받침하는 증거 수집
  - 검증: 자료의 신뢰성 검증
  - 정리: 자료의 체계적 정리

- **분석가**: 자료를 분석합니다.
  - 분석: 자료 분석
  - 해석: 분석 결과 해석
  - 패턴: 주요 패턴 파악
  - 시사: 시사점 도출
  - 권고: 권고사항 개발

- **보고서 작성자**: 보고서를 작성합니다.
  - 구조: 효과적인 구조 설계
  - 내용: 각 섹션별 내용 작성
  - 명확성: 명확한 표현
  - 시각화: 효과적인 데이터 시각화
  - 검증: 논리의 일관성 검증

## 워크플로우

### 단계 1: 준비

1. 사용자 입력에서 추출:
   - **보고서**: 보고서 유형
   - **목적**: 보고서 목적
   - **대상**: 대상 독자
   - **범위**: 보고 범위
   - **기한**: 제출 기한

2. `_workspace/` 디렉토리 생성

### 단계 2: 팀 구성 및 실행

| 순서 | 작업 | 담당 | 의존성 | 산출물 |
|------|------|------|--------|--------|
| 1 | 분석 | 분석가 | 없음 | `01_requirements.md` |
| 2 | 수집 | 수집가 | 작업 1 | `02_research_findings.md` |
| 3 | 분석 | 분석가 | 작업 2 | `03_analysis.md` |
| 4 | 작성 | 작성자 | 작업 3 | `04_report.md` |

## 산출물

- `01_requirements.md` — 보고 요구사항
- `02_research_findings.md` — 조사 결과
- `03_analysis.md` — 분석
- `04_report.md` — 최종 보고서

## 오류 처리

| 오류 유형 | 대응 전략 |
|---------|---------|
| 자료 부족 | 대체 자료 활용 |
| 시간 부족 | 범위 조정 |
| 에이전트 실패 | 1회 재시도 → 진행 |
