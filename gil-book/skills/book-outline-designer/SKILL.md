# book-outline-designer

> 도서 목차를 **3 레벨 트리**(부·장·꼭지)로 설계하고 **분량 배분 + 챕터 시놉시스**까지 산출하는 스킬. book-concept-planner(USP·시장 포지셔닝)와 book-target-reader(페르소나·JTBD·페인포인트) 두 스킬의 산출물을 입력받아 페르소나 여정에 맞춘 목차를 자동 분기합니다. **목차는 책의 골격** — 본문 집필 전 반드시 확정되어야 출판사 통과·완성도 모두 보장됩니다.

## 1. 개요

도서 목차는 단순한 챕터 나열이 아닙니다. 다음 **4가지 기능**을 동시에 수행합니다.

| 기능 | 의미 |
|------|------|
| **약속** | 독자가 책을 통해 얻을 결과를 미리 보여줌 |
| **여정** | 독자가 어떤 순서로 읽고 어떻게 변할지 설계 |
| **균형** | 분량을 챕터별로 배분해 한쪽이 비대해지지 않게 |
| **검증** | 목차만 봐도 USP·JTBD가 잘 구현되는지 확인 가능 |

목차 없이 본문을 쓰면 **분량 폭주·논리 비약·페르소나 이탈**이 발생합니다. 출판사 편집장이 가장 먼저 보는 것도 목차입니다.

## 3. 워크플로우 (4단계)

### Step 1: 목차 골격 설정 (3 레벨 트리)

도서 목차는 **부 → 장 → 꼭지** 3 레벨로 설계. 각 레벨의 책임:

| 레벨 | 정의 | 분량 | 예시 (실용서·부업서) |
|------|------|------|---------------------|
| **부 (Part)** | 책의 큰 단원, 독자 여정의 단계 | 3-5 부 | 1부 마인드 / 2부 실전 / 3부 자동화 |
| **장 (Chapter)** | 부의 하위 단원, 한 주제 완결 | 4-6 장/부 | 1장 부업의 종류 / 2장 시간 확보 / 3장 첫 매출 |
| **꼭지 (Section)** | 장의 하위 단원, 한 호흡으로 읽음 | 3-6 꼭지/장 | 1꼭지 부업 vs 본업 / 2꼭지 N잡 트렌드 / 3꼭지 자기 진단 |

#### 골격 결정 5 원칙

1. **MECE 원칙** — 부·장·꼭지가 서로 중복 없이, 빠짐없이 (Mutually Exclusive, Collectively Exhaustive)
2. **페르소나 여정 흐름** — 독자가 책 처음부터 끝까지 자연스럽게 이동
3. **USP 분산** — USP 3축이 부·장에 골고루 배치 (1축이 한 챕터에만 몰리지 않음)
4. **페인포인트 우선순위** — 긴급 페인을 앞 부에 배치, 잠재 페인은 뒤로
5. **JTBD 도착점** — 마지막 부·장에서 JTBD가 완료되는 느낌 (기능·감정·사회 3 차원 모두)

### Step 2: 분량 배분 (전체 매수 → 부 → 장 → 꼭지)

#### 장르별 표준 분량 (200자 원고지 기준)

| 장르 | 전체 분량 | 부당 분량 | 장당 분량 | 꼭지당 분량 |
|------|----------|----------|----------|------------|
| 실용서·자기계발 | 250-350매 | 60-90매 | 12-20매 | 3-5매 |
| 인문·교양·에세이 | 300-500매 | 80-120매 | 18-28매 | 4-7매 |
| 기술·전문서·IT | 400-800매 | 100-200매 | 25-50매 | 6-12매 |
| 소설·문학 (장편) | 500-1500매 | 120-400매 | 30-80매 | (꼭지 없음) |

#### 분량 배분 절차

1. **목표 총분량 확정** — 출판사·장르·페르소나 독서시간에 맞춤
2. **부별 비중 결정** — 보통 균등 분배, 단 도입부·결론부는 약간 가볍게
3. **장별 비중 결정** — 페인포인트 강도가 높은 장은 더 길게
4. **꼭지별 분량 책정** — 페르소나의 한 호흡 독서시간 (실용서는 5-10분/꼭지)

#### 분량 폭주·부족 검출

- **폭주**: 한 장이 표준의 1.5배 초과 → 장을 2개로 분할 검토
- **부족**: 한 장이 표준의 0.5배 미만 → 다른 장과 통합 또는 부록 처리
- **균형**: ±20% 범위 권장

### Step 3: 챕터 시놉시스 작성

각 장에 대해 **5 요소 시놉시스**(약 200-400자) 작성:

| 요소 | 질문 | 분량 |
|------|------|------|
| **도입** | 독자가 이 장에서 처음 만날 페인포인트는? | 30-50자 |
| **약속** | 이 장에서 무엇을 다룰 것인가? | 50-80자 |
| **본문 요약** | 핵심 주장·방법·사례 3개 | 80-150자 |
| **사례·인용** | 어떤 실증·인터뷰·데이터로 뒷받침? | 30-60자 |
| **다음 장 연결** | 이 장이 끝나면 어떤 질문이 남는가? | 30-50자 |

#### 꼭지별 시놉시스 (선택)

장 시놉시스가 충분하면 꼭지는 1줄 제목 + 핵심 메시지 1문장으로도 OK. 다만 출판사 투고용 제안서에는 꼭지 시놉시스도 포함 권장.

### Step 4: 흐름 검증 (페르소나 여정 시뮬레이션)

완성된 목차에 페르소나가 처음부터 끝까지 따라간다고 가정. **3 시점**에서 검증:

| 시점 | 검증 질문 | 통과 기준 |
|------|----------|----------|
| **1부 끝** | 페르소나가 책을 덮을 동기는 충분히 차단되었나? | 1부 끝에 페르소나 흥미·신뢰가 60% 이상 유지 |
| **중반** | JTBD 기능적·감정적 차원이 절반 이상 충족되었나? | 중반 챕터에 핵심 방법론·사례 배치 확인 |
| **마지막 부** | JTBD 3 차원(기능·감정·사회) 모두 완료되는가? | 마지막 부·장에 사회적 JTBD(타인 인식 변화) 명시 |

검증 실패 시 Step 1로 되돌아가 목차 재설계.

## 4. 5 장르 프리셋 (자동 분기)

### 4-1. 실용서·자기계발

| 차원 | 분기 |
|------|------|
| 목차 패턴 | "마인드 → 실전 → 자동화·확장" 3부 구조 |
| 부 개수 | 3-4부 |
| 장당 분량 | 12-20매 (200자 원고지) |
| 꼭지 길이 | 3-5매 (5-10분 독서) |
| 꼭지 안 구조 | **Q&A·실습 박스·체크리스트** 권장 |
| 시각 자료 | 도표·인포그래픽 (장당 2-4개) |
| 부록 | 템플릿·체크리스트·30일 챌린지표 |

### 4-2. 인문·교양·에세이

| 차원 | 분기 |
|------|------|
| 목차 패턴 | "관찰 → 사유 → 제안" 3부 또는 시간순·주제별 |
| 부 개수 | 3-5부 |
| 장당 분량 | 18-28매 |
| 꼭지 길이 | 4-7매 |
| 꼭지 안 구조 | **서사 흐름·인용·각주** 활용 |
| 시각 자료 | 적음 (사진·삽화 정도) |
| 부록 | 참고문헌·찾아보기 |

### 4-3. 기술·전문서·IT 입문서

| 차원 | 분기 |
|------|------|
| 목차 패턴 | "기초 → 핵심 기능 → 실전 프로젝트 → 심화" 4부 구조 |
| 부 개수 | 4-6부 |
| 장당 분량 | 25-50매 (코드·도표 포함) |
| 꼭지 길이 | 6-12매 |
| 꼭지 안 구조 | **개념 설명 → 코드 예제 → 따라하기 → 정리** |
| 시각 자료 | 도표·코드·스크린샷 (꼭지당 3-6개) |
| 부록 | 환경 설정 가이드·용어 사전·전체 코드 GitHub |

### 4-4. 소설·문학

| 차원 | 분기 |
|------|------|
| 목차 패턴 | **부 또는 장 단위만**, 꼭지 없음 (서사 흐름 유지) |
| 장 개수 | 10-30장 (장편 기준) |
| 장당 분량 | 30-80매 (시점·갈등에 따라 가변) |
| 시놉시스 | 3막 구조(설정·전개·결말) 또는 5막·7막 구조 |
| 인물 카드 | 주요 인물 3-7명 별도 카드 |
| 시점 변화 | 1인칭/3인칭·전지적/제한적 시점 명시 |

### 4-5. 경험서·실무회고 (GIL 오리지널)

| 차원 | 분기 |
|------|------|
| 집필 단위 | **장(章) 단위** (꼭지로 분해하지 않음) |
| 표준 분량 | 200-320매 / 장당 12-20매 |
| 목차 패턴 | **도입 → 시행착오 → 체계 구축 → 성찰** |
| 장 구조 | 각 장은 장면→맥락→원칙(30/40/30) — `book-chapter-writer §4-5` |

> **UZ 듀얼** — UZ 한인·고려인 경험서 맥락은 `book-chapter-writer/references/uz-fieldnote.md` 참조.

## 5. 출력 형식 (목차 트리 + 시놉시스 표)

```markdown
- 장 개수: ___
- 꼭지 개수: ___ (실용서·기술서만)
- 예상 독서 시간: ___ 시간

### 프롤로그 (___ 매)
[프롤로그 주제 1줄]

- 3꼭지. [꼭지 제목] (___ 매)

#### 2장. [장 제목] (___ 매)

### 2부. [부 제목] (___ 매)
...

### 에필로그 (___ 매)

| 2부 | ___% | ___ |
| ... |

## 5. 부록·추가 자료

- 부록 A: ___ (템플릿·체크리스트 등)
- 부록 B: ___
- 참고 자료: GitHub·웹사이트·다운로드 링크
```

## 6. 사용 예시

### 예시 1 — 실용서·자기계발 (부업서)

```
book-concept-planner와 book-target-reader 결과 입력.
"30일 챌린지로 시작하는 부업" 책 목차 설계해줘.
총 분량 300매, 3부 구조 권장.
```

→ Step 1 골격: 1부 마인드(80매) / 2부 실전(140매) / 3부 자동화(80매)  
→ Step 2 분량: 부당 평균 100매, 장당 15매, 꼭지당 4매  
→ Step 3 시놉시스: 12장 각각 5요소 시놉시스 작성  
→ Step 4 검증: 1부 끝 신뢰도 70%·중반 JTBD 60%·마지막 사회적 JTBD ✅  
→ 출력: 목차 트리 + 시놉시스 표

### 예시 2 — IT 입문서 (12 챕터·600매)

```
"Next.js 15 + shadcn/ui 실전 가이드" 목차 설계해줘. 
주니어 3년차 백엔드 개발자 페르소나, 12 챕터·600매 기준.
```

→ Step 1 골격: 1부 기초(2장)·2부 핵심 기능(4장)·3부 실전 프로젝트(4장)·4부 심화(2장)  
→ Step 2 분량: 장당 50매, 코드 예제 6-10개/장  
→ Step 3 시놉시스: 12장 각각 개념·코드·따라하기·정리 흐름  
→ Step 4 검증: 1부 후 React 차이 이해·중반 풀스택 자신감·마지막 이직 자신감 ✅

### 예시 3 — 인문·에세이 (40대 워킹맘)

```
"엄마이지만 나로 살기" 에세이집 목차 설계해줘.
400매·서사 흐름·민음사 투고 목표.
```

→ 인문 프리셋 자동 적용 → "관찰 → 사유 → 제안" 3부 → 꼭지 없이 장 단위 (12-15장)

| 창비 | 인문·아동 | 사회적 관점·각주 풍부 |

### KPIPA 표준 (한국출판문화산업진흥원)

- 표준 분량: 200자 원고지 기준 (출판사 양식에 따라 워드 카운트로 변환)
- ISBN 부가기호: 목차 확정 후 출판사 신청
- 도서정가제: 신간 18개월 정가 + 최대 10% 할인 + 5% 적립

## 8. 주의사항

### 목차 함정 회피

| 함정 | 피하는 법 |
|------|----------|
| **목차 = 챕터 나열** | 부·장·꼭지 3 레벨 + 5 원칙 적용 (MECE·여정·USP 분산·페인 우선·JTBD 도착) |
| **분량 폭주** | 한 장이 표준의 1.5배 초과 시 분할. 균형 ±20% |
| **USP 한쪽 몰림** | USP 3축이 부에 분산되었는지 검증 |
| **JTBD 미도달** | 마지막 부에 사회적 JTBD 명시 (이름 변화·관계 변화) |
| **페르소나 이탈** | 4단계 검증 (1부 끝·중반·마지막) 통과 필수 |
| **출판사 양식 무시** | 투고 출판사의 기존 신간 목차 5권 분석 후 양식 맞춤 |

### 페르소나 입력 정합성

- book-concept-planner USP 3축 → 목차의 부·장에 분산되어 들어갔는지 검증
- book-target-reader JTBD 3 차원 → 마지막 부에서 모두 완료되는지 검증
- book-target-reader 페인포인트 매트릭스 → 긴급 페인이 앞 부에 배치되었는지 검증

### 출처 정책

- vault 외부 참고 자료 직접 인용 금지
- 인사이트·프레임워크만 추출, 비유·예시 표현은 자체 작성
- 한국 출판사 목차 분석은 실제 신간 5권 직접 확인 후 인용

## 9. 관련 스킬

### Before (앞 단계)
- `book-concept-planner` — USP·시장 포지셔닝
- `book-target-reader` — 페르소나·JTBD·페인포인트

### After (체이닝)
- `book-author-bio` — 저자 약력 (목차 확정 후 저자 신뢰 신호 정의)
- `book-proposal-writer` — 출판사 투고용 제안서 (목차 + 시놉시스 포함)
- `book-chapter-writer` — 본문 챕터 집필 (꼭지 단위 반복)

### Alternative
- `gil-business:strategy-planner` — 사업·서비스 구조 설계 (책 아님)
- `gil-content:blog` — 블로그 시리즈 구조 (책 분량 미만)

### Post (검수)
- `gil-content:korean-spell-check` — 한국어 맞춤법
- `gil:ai-slop-reviewer` — 목차 자연스러움 검수
