# book-concept-planner

> 한국 출판사 제출용 **도서 컨셉서**를 단계적으로 작성하는 스킬. 의도파악 → 리서치 → 인사이트 도출의 3단계로 한 줄·30자·300자 요약, USP 3축, 시장 포지셔닝 매트릭스, 4 장르 프리셋을 산출합니다. gil-book 플러그인 워크플로우의 **첫 번째 산출물** — 이 컨셉서가 없으면 후속 스킬(book-target-reader·book-outline-designer·book-proposal-writer)이 정확하게 작동하지 않습니다.

## 1. 개요

도서 컨셉서는 출판사 투고 직전 단계에서 작성하는 **책 한 권의 정체성 문서**입니다. 다음 4가지 질문에 답해야 합니다.

| 질문 | 산출물 |
|------|--------|
| 이 책은 누구를 위한가? | 타깃 독자 페르소나 1명 (구체) |
| 왜 지금 이 책이어야 하는가? | 시장 포지셔닝 (2축 매트릭스) |
| 무엇이 다른가? | USP 3축 (경쟁작 대비 차별) |
| 한 줄로 무엇인가? | 한 줄·30자·300자 요약 (3 길이) |

한국 출판사 투고용 컨셉서는 **A4 1-2장**이 표준입니다. 분량보다 정밀도가 중요하며, USP가 약하면 어떤 좋은 본문도 출판사 통과가 어렵습니다.

## 3. 워크플로우 (3단계 인사이트 도출)

### Step 1: 의도 파악 (Why?)

"왜 이 책을 쓰는가"를 명확히. AskUserQuestion으로 다음 5 질문에 답을 받습니다.

| # | 차원 | 질문 | 옵션 예시 |
|---|------|------|-----------|
| 1 | Why (저자 동기) | 책을 쓰는 가장 큰 이유는? | ① 표현 욕구 ② 경험·노하우 공유 ③ 흔적 남기기 ④ 시장 진입·수익 |
| 2 | What (타깃) | 누가 이 책을 읽기를 바라나? | 페르소나 1명 (나이·직업·고민) 자유 기술 |
| 3 | What (페인포인트) | 독자의 어떤 문제·고민을 해결하나? | 자유 기술 (3개 이내) |
| 4 | Who (의사결정자) | 출간 여부를 누가 최종 결정하나? | ① 출판사 편집장 ② 자비 출판 (저자 본인) ③ 독립 출판 펀딩 |
| 5 | Check (누락) | 다른 책에서 다루지 않는 영역이 있는가? | 자유 기술 |

5 답변 모두 모이면 Step 2로 진행.

### Step 2: 리서치 (Research)

데스크 서치 + 필드 서치를 **MECE 원칙**(Mutually Exclusive, Collectively Exhaustive — 중복·누락 없이)으로 진행합니다.

#### 데스크 서치 (필수, 한국 2026 데이터)

| 자료 | 목적 | 출처 |
|------|------|------|
| 교보문고 베스트셀러 차트 | 장르별 최근 6개월 트렌드 | https://www.kyobobook.co.kr |
| 알라딘 도서 DB | ISBN·매출·리뷰 데이터 | https://www.aladin.co.kr |
| KPIPA 출판 통계 | 시장 사이즈·발행 부수 통계 | https://www.kpipa.or.kr |
| 도서정가제 가격 분포 | 신간·구간 가격 정책 | https://www.kpipa.or.kr |
| 한국출판콘텐츠 (KPC) | 출판사 투고 가이드 | https://www.kpc.or.kr |

검색 키워드 팁:
- 한글 키워드 + 영문 키워드 (글로벌 트렌드 비교)
- 책 제목·저자명·ISBN 모두 시도
- 출판사 자체 사이트 (해당 출판사 신간 코너)

#### 필드 서치 (권장, 가능한 범위)

- **동일 장르 저자 인터뷰** — 유튜브·블로그·SNS·강연 발췌
- **잠재 독자 1:1 인터뷰** — 5명 정도. 페인포인트·구매 동기 직접 청취
- **출판사 편집자 가벼운 확인** — 컨셉 1 페이지로 사전 의견 (수락률 향상)
- **서점 직원·MD 의견** — 매대에 어떻게 놓일지

> 💡 필드 서치 1줄 원칙: **"나보다 조금이라도 잘 아는 사람에게 묻는다"**.

#### MECE 자료 정리

- 시장·경쟁상황·고객·트렌드·리스크·외부환경·내부역량·산업구조 8 카테고리로 분류
- 중복 자료는 1개로 통합
- 누락된 카테고리는 추가 리서치

### Step 3: 컨셉 도출 (Insight)

리서치 결과를 다음 **5 산출물**로 정제합니다.

#### 3-1. USP (Unique Selling Proposition) — 차별화 3축

경쟁작 최소 3권과 비교해 차별 포인트 3개를 한 문장씩 정리:

```
USP 1 [방법론 차별]: 경쟁작 ___과 달리, 이 책은 ___ 방법론을 사용한다.
USP 2 [관점 차별]:   경쟁작 ___과 달리, 이 책은 ___ 관점에서 다룬다.
USP 3 [경험 차별]:   경쟁작 ___과 달리, 이 책은 ___ 실제 사례를 제공한다.
```

USP의 핵심은 "다름". 일본·미국 출판 사례에서 반복 확인된 명제 — **"베스트셀러의 필요 조건은 다름"**. 충분 조건은 없으나 USP가 약하면 출판사 통과가 어렵습니다.

#### 3-2. 한 줄 요약 (15자 내외)

표지·SNS·서점 진열대에 노출되는 단 한 줄. 3 원칙 준수:

| 원칙 | 적용 방법 |
|------|----------|
| **구체성** | "많다" 같은 추상어 대신 그림이 그려지는 단어 사용 |
| **낯선 조합** | 평소 만나지 않을 두 단어의 결합으로 시선 끌기 |
| **검색 우선** | 일상에서 듣는 말에서 뽑아내기 (search > make) |

한자어는 우리말로 변환 권장:
- "진가를 발휘" → "제대로 합니다"
- "역부족이었다" → "모자랐다"
- "재발 방지" → "다시는 이런 일이 없도록"

#### 3-3. 30자 카피 (소개문 1)

서점 매대·검색 결과에서 책을 집어들게 만드는 30자 이내. 베네핏·타깃·차별점 중 2개 조합:

```
"30일 만에 부업 시작하는 가장 빠른 길" (베네핏 + 시간)
"내성적인 직장인을 위한 1:1 영업 책"     (타깃 + 카테고리)
"한 권으로 끝내는 Next.js 15 실전"        (베네핏 + 완결성)
```

#### 3-4. 300자 시놉시스 (소개문 2)

책 뒷표지·온라인 서점 상세 페이지용 300자. 3단 구조:

| 단 | 분량 | 내용 |
|----|------|------|
| 1단 (도입) | 100자 | 독자의 문제·고민·페인포인트 정의 |
| 2단 (약속) | 100자 | 이 책이 무엇을 어떻게 다룰지 |
| 3단 (변화) | 100자 | 독자가 이 책을 읽고 얻을 결과 |

#### 3-5. 시장 포지셔닝 (2축 매트릭스)

경쟁작 4-8권을 2축 매트릭스에 배치:

```
            깊이 (심화)
                ↑
                │
                │  경쟁작 A    경쟁작 B
                │
실용 ←─────────┼───────────→ 서사
                │
                │  경쟁작 C    경쟁작 D
                │
                ↓
            깊이 (입문)
```

축은 장르별로 변형 가능:
- **실용서**: 깊이(입문 ↔ 심화) × 톤(실용 ↔ 서사)
- **인문서**: 시대(고대 ↔ 현대) × 관점(개인 ↔ 사회)
- **기술서**: 난이도(입문 ↔ 고급) × 범위(좁음 ↔ 넓음)
- **소설**: 분량(중편 ↔ 장편) × 시점(1인칭 ↔ 3인칭)

## 4. 5 장르 프리셋 (자동 분기)

장르를 Step 1에서 결정하면 Step 2·3의 세부 작업이 자동 분기됩니다.

### 4-1. 실용서·자기계발

| 차원 | 분기 |
|------|------|
| Step 1 (의도) | 문제 해결 페르소나, "어떻게 더 잘 살 것인가" |
| Step 2 (리서치) | 시장 사이즈·Q&A·실습 박스 분석, 베스트셀러 차트 비중 高 |
| Step 3 (USP) | 방법론 신규성·시간 단축·결과 보장 |
| 한 줄 카피 | 베네핏 직접 노출 ("30일 만에…") |
| 분량 가이드 | 250-350매 (200자 기준) |
| 대표 출판사 | 웅진지식하우스·다산북스·길벗·메가스터디북스·알에이치코리아·흐름출판 |

### 4-2. 인문·교양·에세이

| 차원 | 분기 |
|------|------|
| Step 1 (의도) | 공감·서사 페르소나, "왜 이렇게 사는가" |
| Step 2 (리서치) | 동시대 작가 문체·인용·각주, 시·소설 인용 |
| Step 3 (USP) | 관점 신규성·문장력·서사 구조 |
| 한 줄 카피 | 감정·은유 활용 |
| 분량 가이드 | 300-500매 |
| 대표 출판사 | 민음사·문학동네·창비·은행나무·돌베개·휴머니스트·사계절 |

### 4-3. 기술·전문서·IT 입문서

| 차원 | 분기 |
|------|------|
| Step 1 (의도) | 학습 페르소나 (입문/중급/고급 명시) |
| Step 2 (리서치) | 경쟁작 코드 예제·도표·실습 챕터·용어 사전 분석 |
| Step 3 (USP) | 기술 스택 신규성·완결성·검증된 실습 |
| 한 줄 카피 | "X 책 1권으로 Y까지" 패턴, 버전 명시 |
| 분량 가이드 | 400-800매 (코드·도표 포함) |
| 대표 출판사 | 한빛미디어·인사이트·제이펍·길벗 IT·비제이퍼블릭·영진닷컴 |

### 4-4. 소설·문학

| 차원 | 분기 |
|------|------|
| Step 1 (의도) | 인물·플롯 중심, 저자도 인물 한 명 |
| Step 2 (리서치) | 동일 장르 베스트셀러 플롯·시점·배경 분석, 문학상 응모 트렌드 |
| Step 3 (USP) | 세계관·시점·문체 신규성 |
| 한 줄 카피 | 인물·갈등 압축, 호기심 유발 |
| 분량 가이드 | 500-1500매 |
| 대표 출판사 | 민음사·문학동네·창비·은행나무·문학과지성사·자음과모음·안온북스 |

### 4-5. 경험서·실무회고 (GIL 오리지널)

- **포지셔닝 축**: 경력 깊이(주니어 ↔ 시니어) × 서술 거리(기록 ↔ 해설)
- **USP 방향**: "방법"이 아니라 "판단 기준·시행착오 회피"를 파는 책
- 집필 단위는 장(章), 구조는 장면→맥락→원칙 — `book-chapter-writer §4-5`

> **UZ 듀얼** — UZ 한인·고려인 실무 회고 맥락은 `book-chapter-writer/references/uz-fieldnote.md`.

## 6. 사용 예시

### 예시 1 — 실용서·자기계발

```
"30일 챌린지로 시작하는 부업" 도서 컨셉서 작성해줘. 
직장인 30-40대 대상, 다산북스·길벗 투고 목표.
```

→ Step 1 의도파악 (5 질문)  
→ Step 2 리서치 (베스트셀러 차트 + 부업 시장 KPIPA + 1:1 직장인 5명 인터뷰)  
→ Step 3 컨셉 도출 (한 줄: "퇴근 후 30일이면 충분합니다" / USP 3축 / 시장 포지셔닝)  
→ 출력: A4 1-2장 컨셉서

### 예시 2 — IT 입문서

```
"Next.js 15 + shadcn/ui 실전 가이드" 책 컨셉서 작성해줘. 
한빛미디어·인사이트 투고 목표, 챕터 12개·분량 600매 기준.
```

→ Step 1 의도파악 (학습 페르소나, 중급 개발자 대상)  
→ Step 2 리서치 (경쟁작 5권 코드 예제·실습 챕터 비교 + IT 출판사 신간 동향)  
→ Step 3 컨셉 도출 (한 줄: "Next.js 15, 한 권으로 끝낸다" / USP 3축 / 챕터 12개 시놉시스)  
→ 출력: 컨셉서 + 후속 book-outline-designer 호출 권장

### 예시 3 — 인문·에세이

```
민음사·문학동네 투고 예정 에세이집 컨셉서 만들어줘. 
40대 워킹맘 시선의 일·가족 이야기.
```

→ 인문 프리셋 자동 적용 → 서사·인용 중심 리서치 → 관점 신규성 USP 도출

## 8. 주의사항

### 자비 출판 vs 출판사 투고 의사결정

| 상황 | 권장 |
|------|------|
| 첫 책·자료가 충분 | 출판사 투고 우선 → 거절 시 자비 출판 |
| 강력한 팬덤·SNS 보유 | 자비 출판으로 초기 시장 검증 후 출판사 협상 |
| 시간 제약 (3개월 내 출간) | 자비 출판 (부크크·텀블벅 펀딩) |
| 인쇄·디자인·유통 전문성 부재 | 출판사 투고 권장 |
| 글로벌 시장 진출 | 한국 출판사 → 번역권 협상 (영어판 직접 출판도 고려) |

### USP가 약하면 통과 불가

- 출판사 편집장이 컨셉서를 보고 "이 책이 왜 필요한가?"에 답을 얻지 못하면 즉시 반려
- USP 3축이 모두 모호하면 **컨셉서 작성을 처음부터 다시** (Step 1 재방문)
- 경쟁작 4-8권 매트릭스 없이는 USP 검증 불가

### 외부 자료 직접 인용 금지

- 출판 시장 자료·카피 사례·시놉시스 예시는 모두 **자체 작성**
- 출처 노트의 비유·표현·예시 문장은 그대로 사용하지 않고, **개념·프레임워크만** 추출하여 재구성
- vault 자료 인용 가드 준수

## 9. 관련 스킬

### Before (앞 단계)
- 없음 — 이 스킬이 도서 워크플로우의 **첫 번째 산출물**

### After (체이닝)
- `book-target-reader` — 타깃 독자 페르소나 심화 + JTBD + 페인포인트 매트릭스
- `book-outline-designer` — 목차 설계 (부·장·꼭지) + 분량 배분 + 챕터 시놉시스
- `book-author-bio` — 저자 약력·저자의 말·SNS·강연 통합
- `book-proposal-writer` — 출판사 투고용 제안서 (출판기획서 + 샘플 챕터 + 마케팅 플랜)

### Alternative
- `gil-content:copywriting` — 도서 외 일반 카피 (광고·랜딩 페이지)
- `gil-business:strategy-planner` — 전략 컨셉 (사업·서비스 대상, 책 아님)
- `gil-marketing:campaign-planner` — 마케팅 캠페인 컨셉

### Post (검수)
- `gil-content:korean-spell-check` — 한국어 맞춤법
- `gil-content:humanize-korean` — AI 티 제거
