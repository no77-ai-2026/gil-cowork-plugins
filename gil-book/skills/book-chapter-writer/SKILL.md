# book-chapter-writer

> 도서 본문을 **꼭지 단위로 처음부터 끝까지 완결성 있게** 집필하는 스킬. book-outline-designer의 챕터 시놉시스(5요소: 도입·약속·본문요약·사례·연결)를 입력받아 200자 원고지 분량을 정확히 맞춰 작성. 5 장르별 문체 자동 분기(실용·인문·기술·소설·경험서). **출판 표준 분량·인용 양식·도표 자리표시·코드 블록**을 함께 처리해 출판사 편집자 손에 그대로 넘길 수 있는 수준의 초고 생성.

## 1. 개요

본문 집필은 **꼭지 단위**(실용서·기술서 기준 3-10매, 200자 원고지)로 진행. 한 꼭지는 다음 5 요소를 갖춰야 완결성 확보:

| 요소 | 분량 비중 | 역할 |
|------|----------|------|
| **훅 (Hook)** | 10% | 첫 문장·첫 단락으로 독자 시선 잡기 |
| **본문 (Body)** | 70% | 핵심 주장·사례·근거 |
| **클라이맥스** | 10% | 가장 강력한 인용·결론·통찰 |
| **정리 (Closing)** | 5% | 한 줄 요약·실천 제안 |
| **다음 연결** | 5% | 다음 꼭지·장으로 호기심 유발 |

200자 원고지 표준 — 출판사가 분량을 가늠하는 기준. 워드 카운트 ÷ 4 ≈ 매수.

## 2. 트리거 키워드

자동 호출 트리거 (frontmatter description 참조). 슬래시 호출: `/book-chapter-writer`.

## 3. 워크플로우 (4단계)

### Step 1: 꼭지 시놉시스 확장

book-outline-designer의 장 단위 시놉시스에서 작성할 꼭지 1개 선택. AskUserQuestion으로 다음 확인:

| 항목 | 질문 |
|------|------|
| 꼭지 위치 | 몇 장 몇 꼭지? (예: "2장 3꼭지") |
| 목표 분량 | 매수 (예: 4매·800자) |
| 주제 한 줄 | 이 꼭지가 다룰 한 문장 |
| 페르소나 시점 | 어떤 페르소나를 가정해 쓸 것인가? |
| 인용·자료 | 인용할 자료·도표·코드 미리 정리 |

### Step 2: 5 요소 골격 작성

확장된 시놉시스를 5 요소 골격으로 분해:

```
[훅] (1~2 문장)
[훅에서 본문 도입으로 자연스러운 다리]

[본문 단락 1] (핵심 주장 1)
- 근거 1
- 사례·인용 1

[본문 단락 2] (핵심 주장 2)
- 근거 2
- 사례·인용 2

[본문 단락 3] (핵심 주장 3 — 선택)
- 근거 3
- 사례·인용 3

[클라이맥스] (가장 강력한 통찰·인용·결론)

[정리] (한 줄 요약 + 실천 제안)

[다음 연결] (다음 꼭지·장 예고)
```

### Step 3: 5 장르 문체 적용

장르별 문체 프리셋 자동 적용. 자세한 분기는 §4 참조.

### Step 4: 분량 카운트·인용·도표 처리

#### 분량 카운트 (200자 원고지)

- 워드 카운트 ÷ 4 ≈ 매수
- 예: 4매 = 약 800자 (공백 포함) 또는 1,600자 (공백 제외)
- 출판사별 기준 다를 수 있음 (확인 필요)

#### 인용·각주 처리

| 인용 유형 | 표기 |
|----------|------|
| 짧은 인용 (1-2 문장) | 따옴표("…") |
| 긴 인용 (3 문장 이상) | 들여쓰기 인용 단락 + 출처 |
| 통계·데이터 | 본문 인용 + 각주에 출처 |
| 인터뷰·발언 | 발언자 + 매체 + 날짜 명시 |
| 인터넷 자료 | 접속일 + URL 각주 |

#### 도표·코드·이미지 자리표시

| 자료 | 자리표시 |
|------|----------|
| 도표 | `[도표 X-Y. 도표 제목]` (X=장, Y=순번) |
| 코드 | ` ```language ... ``` ` (기술서) |
| 스크린샷 | `[그림 X-Y. 그림 제목]` |
| 인포그래픽 | `[인포그래픽 X-Y]` |
| 실습 박스 | `> **실습**: …` |
| 체크리스트 | `- [ ] 항목` |

## 4. 5 장르 문체 프리셋 (자동 분기)

### 4-1. 실용서·자기계발

| 차원 | 분기 |
|------|------|
| 문체 | **명료·실용·약속 중심** |
| 어미 | "~다", "~합니다" 혼용 가능 (출판사별) |
| 문장 길이 | 짧게 (평균 30-50자) |
| 단락 길이 | 3-5 문장 |
| 사례 | 구체적 숫자·기간·결과 명시 |
| 시각 자료 | 도표·체크리스트·실습 박스 (꼭지당 2-4개) |
| 어조 | 친근하고 격려하는 톤 (선생님→학생) |

훅 예시 패턴:
- 질문형: "○○해본 적 있나요?"
- 통계형: "한국 직장인 70%가 ___"
- 시나리오형: "오늘 아침 ___를 발견했다면…"

### 4-2. 인문·교양·에세이

| 차원 | 분기 |
|------|------|
| 문체 | **서사·문체 중심·은유 활용** |
| 어미 | 작가 개성 (~이다, ~다, ~ㅂ니다 자유) |
| 문장 길이 | 다양 (짧은 문장과 긴 문장 교차) |
| 단락 길이 | 5-10 문장 |
| 사례 | 일상 관찰·문학 인용·역사 사례 |
| 시각 자료 | 적음 (사진·삽화 정도) |
| 어조 | 사색적·고백적·따뜻함 |

훅 예시 패턴:
- 장면 묘사: "___의 어느 날, 나는 ___" 
- 인용 도입: "○○○ 작가는 이렇게 말했다…"
- 질문 도입: "왜 우리는 ___일까?"

### 4-3. 기술·전문서·IT 입문서

| 차원 | 분기 |
|------|------|
| 문체 | **명료·정확·기술 용어 사용** |
| 어미 | "~다" 일관 (학술적) |
| 문장 길이 | 명확하게 (평균 30-60자) |
| 단락 길이 | 3-7 문장 (개념별 단락) |
| 사례 | 코드 예제·도표·실습 단계 |
| 시각 자료 | 매우 풍부 (꼭지당 3-6개) |
| 어조 | 친절한 동료 개발자 톤 (입문서) / 정확한 전문가 (심화) |

훅 예시 패턴:
- 문제 정의: "___를 만들 때 자주 마주치는 문제는…"
- 코드 도입: "다음 코드를 봅시다." 
- 결과 약속: "이 꼭지를 마치면 ___ 가능합니다."

### 4-4. 소설·문학

| 차원 | 분기 |
|------|------|
| 문체 | **묘사·시점·문체 일관성** |
| 시점 | 1인칭/3인칭·전지적/제한적 (장 전체 일관) |
| 어미 | 작가 개성 |
| 문장 길이 | 다양 (장면·인물 심리에 따라) |
| 단락 길이 | 가변 (대화·묘사·서술 교차) |
| 인물 | 행동·말투·심리 일관성 |
| 시각 자료 | 없음 (텍스트 위주) |
| 어조 | 작품 세계관에 일관 |

훅 예시 패턴:
- 장면 도입: "그날의 ___는 ___와 같았다."
- 대화 도입: "'○○○야.' ___가 말했다."
- 내면 도입: "나는 ___를 ___ 했다."

### 4-5. 경험서·실무회고 (GIL 오리지널)

저자의 실무 경험을 1인칭으로 풀어내는 회고형 실무서. "보여주되 가르치지 않는다"가 핵심.

| 차원 | 분기 |
|------|------|
| 문체 | **관찰·진단 중심** — "나는 이렇게 했고, 이런 일이 벌어졌다" |
| 어조 | 가르치지 않되 이끈다. 독자와 같은 위치(위에 서지 않음) |
| 집필 단위 | **장(章) 단위** (꼭지로 분해하지 않음 — 장면→맥락→원칙이 장 전체 구조) |
| 어미 | Phase에서 확정한 종결어미 일관(~한다체/~합니다체 혼용 금지) |
| 문장 길이 | 30-60자 기본, 단문(15자 이내)으로 리듬 끊기 |
| 어휘 | 사고 중심 명사(기준·판단·구조·역할). 감정 형용사→상태 진단형("불안하다"→"구조가 정리되지 않았다") |
| 금지 | 교훈 강요("~해야 한다"), 감정 유도("놀라웠다"), 범용 조언, 도구 홍보 |
| 마무리 | **요약 금지** — 관점의 재정렬로 종료 |

훅 예시 패턴:
- 장면 도입: 첫 문장은 시간·장소·행동 중 하나 (개념 정의로 시작 금지)
- "그때 나는 ___를 선택했다." / "회의가 끝나고 ___를 발견했다."

#### 장(章) 구조 — 장면 → 맥락 → 원칙 (30 / 40 / 30)

- **장면 (30%)**: 구체적 한 장면에서 출발. 시간·장소·행동으로 시작. 개념 정의 금지.
- **맥락 (40%)**: 문제 정의 + 전/후 비교. 왜 그 판단을 했는지의 배경.
- **원칙 (30%)**: 독자가 자기 상황에 옮겨 쓸 수 있는 판단 기준 1~3개 + "단, ~인 경우" 예외.

#### 보조 장치 (경험서·실용서 적용)

- **현장 노트(Field Note)**: 200~400자 박스. 날짜+상황 명시, **결론은 쓰지 않음**(독자 판단 여지). 실무 팁은 본문과 분리.
- **Before/After 비교**: 수치 포함, 판단 포인트 1문장. 변화의 근거를 보여줌.

#### [TODO] 마커 5종 (초고 관리 컨벤션)

chapter-writer는 초고에 마커를 삽입하고, **book-revision-coach가 교정 단계에서 전량 제거**한다.

| 마커 | 용도 |
|---|---|
| `[TODO-FACT: ]` | 수치·사실관계 확인 |
| `[TODO-SCENE: ]` | 에피소드·장면 보강 |
| `[TODO-LINK: ]` | 장 간 참조 확인 |
| `[TODO-BA: ]` | Before/After 비교 추가 |
| `[TODO-REVIEW: ]` | 저자 판단 필요 |

#### 원재료(블로그·뉴스레터·강의) 재구성 규칙

발행물을 책으로 전환할 때(맨땅 집필이 아닌 경우) 5규칙:
① **녹이기**(원문 인용 금지, 본문에 자연스럽게 통합) ② 출처 표기 불필요(저자 본인 글) ③ 회차 언급 허용("3편에서 다룬") ④ 실무 팁은 현장 노트 박스로 분리 ⑤ 범위 외 콘텐츠는 간접 언급만.

#### 샘플 챕터 앵커

저자가 친필 1챕터(2,000~3,000자)를 제공하면, 이후 모든 장 집필 전 그 **문체·구조·톤을 파악해 초안에 반영**한다. 없으면 위 프리셋만 적용.

> **한국 경험서 표준 + UZ 듀얼 컨텍스트** — UZ 한인·고려인 실무자의 경험서(트릴링구얼 환경·UZ 사업·ODA 현장 기록 등) 특수 맥락은 [`references/uz-fieldnote.md`](references/uz-fieldnote.md) 참조.

## 6. 사용 예시

### 예시 1 — 실용서 (부업서 꼭지)

```
book-outline-designer 출력: 2장 3꼭지 "30일 챌린지 시작하기" 시놉시스.
이 꼭지 본문 작성. 4매 (800자).
```

→ 훅: "월급쟁이로 5년을 일하던 어느 날, 통장 잔고를 보고 결심했다." (시나리오형)  
→ 본문: 30일 챌린지 3 단계 (1단계 자기진단·2단계 채널 선택·3단계 첫 매출) + 사례  
→ 클라이맥스: "30일 후 통장에 처음으로 부업 매출이 찍혔다."  
→ 정리: "30일 챌린지의 핵심은 시작이 아니라 멈추지 않는 것."  
→ 다음 연결: "다음 꼭지에서는 첫 매출 100만원 만드는 채널 선정법을 다룬다."  
→ 도표·체크리스트 2개 자리표시

### 예시 2 — IT 입문서 (Next.js 꼭지)

```
"Next.js 15 App Router 첫 페이지 만들기" 꼭지 본문 작성. 8매 (1,600자).
주니어 3년차 개발자 페르소나.
```

→ 훅: "Pages Router에 익숙하던 우리에게 App Router는 새로운 패러다임이다." (문제 정의)  
→ 본문: 1) App Router 폴더 구조 / 2) 첫 page.tsx 작성 / 3) 동작 확인  
→ 코드 블록 3개 (폴더 트리·page.tsx·실행 결과)  
→ 클라이맥스: "App Router의 핵심은 서버 컴포넌트가 기본이라는 것이다."  
→ 정리: "✅ App Router 폴더 만들기 ✅ page.tsx 작성 ✅ 동작 확인"  
→ 다음 연결: "다음 꼭지에서는 layout.tsx로 공통 레이아웃을 다룬다."

### 예시 3 — 에세이 (워킹맘 꼭지)

```
"새벽 5시, 카페에서 만난 나" 에세이 1장 본문 작성. 5매 (1,000자).
40대 워킹맘 시점.
```

→ 훅: "새벽 5시. 동네 24시 카페의 형광등은 차갑게 푸르다." (장면 묘사)  
→ 본문: 새벽에 글을 쓰러 나오는 일상 + 그 자리에서 만난 자기 자신 +"엄마"라는 호칭에 대한 사유  
→ 인문 프리셋 → 서사·은유·문체 일관성

## 7. 한국 출판 본문 컨벤션

### 200자 원고지 변환 가이드

| 분량 표현 | 변환 |
|----------|------|
| 1 매 | 200자 (공백 포함) |
| 1 매 (워드) | 약 400자 (공백 제외) |
| A4 1장 (10pt, 1.5 줄간격) | 약 3-4 매 |
| 200쪽 책 | 약 200-250 매 |
| 300쪽 책 | 약 300-400 매 |

### 한국어 문체 유의 사항

1. **'~다'/'~ㅂ니다' 혼용 금지** — 한 작품 내 일관성
2. **한자어 → 우리말** — 가능한 우리말 표현 우선 (예: "발휘" → "발휘된다" 대신 "드러난다")
3. **인용 부호** — 한국 표준 따옴표 "" ' '
4. **외래어 표기** — 국립국어원 외래어 표기법 (예: "콘텐츠" 아닌 "컨텐츠"는 잘못)
5. **숫자 표기** — 본문 안 숫자는 한글(다섯), 도표·통계는 아라비아(5)

### 출판사별 본문 양식

| 출판사 | 어미 선호 | 문체 |
|--------|----------|------|
| 한빛미디어 | ~다 | 명료·정확 |
| 인사이트 | ~다 | 정확·전문 |
| 길벗 | ~합니다 | 친근·격려 |
| 웅진지식하우스 | ~합니다 | 친근·약속 |
| 다산북스 | ~합니다 | 친근·격려 |
| 민음사 | 작가 자유 | 문학적 |
| 문학동네 | 작가 자유 | 문학적 |
| 창비 | 작가 자유 | 사회적·문학 |

## 8. 주의사항

### 본문 함정 회피

| 함정 | 피하는 법 |
|------|----------|
| **분량 폭주** | 꼭지 분량 ±20% 초과 시 다음 꼭지로 분리 검토 |
| **장르 문체 위반** | 5 장르 프리셋 일관성 (실용서에 소설 문체 금지) |
| **인용 표기 누락** | 모든 인용·통계·발언에 출처 명시 |
| **추상어 남용** | "다양한", "여러", "많은" 같은 추상어 → 구체적 숫자 |
| **AI 티 (Slop)** | 책의 끝에 ai-slop-reviewer + humanize-korean 체이닝 필수 |
| **한자어 과다** | 우리말 우선, 한자어는 정확한 용어일 때만 |
| **출판사 양식 무시** | 투고 출판사의 어미·문체 컨벤션 사전 확인 |

### 인용·저작권 주의

- **직접 인용**: 짧은 인용은 따옴표 + 출처, 긴 인용은 별도 단락 + 출처
- **공정 사용 한도**: 한국 저작권법상 비평·연구·교육 목적의 공정 사용 인정, 다만 핵심 부분 전체 인용 금지
- **외부 참고 자료**: 직접 인용 금지
- **이미지·도표**: 자체 제작 또는 라이선스 명확 (CC BY·CC0 등)
- **인터뷰 발언**: 발언자 동의 + 매체·날짜 명시

### 분량 관리

- **목표 분량 ±20% 범위 유지**: 폭주 또는 부족 시 분할 또는 통합
- **꼭지 평균 분량 통일**: 한 장 내 꼭지가 너무 들쑥날쑥하지 않게
- **클라이맥스 길이 한도**: 본문의 10% 이내 (너무 길면 산만)

### 출처 정책

- 모든 인용·통계·사례의 **출처 명시**
- vault 외부 참고 자료 직접 인용 금지
- 본문 작성 후 **인용·각주 정리표** 별도 출력 권장
