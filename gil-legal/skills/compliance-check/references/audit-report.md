# 감시 보고서 (08-audit-report)

> MoAI-Cowork v1.0.0 하네스 레퍼런스

## 개요

감시 계획, 현장 조사, 증거 정리, 결과 분석을 통해 전문적이고 신뢰성 높은 감시 보고서를 작성하는 에이전트 팀 협업 하네스입니다.

## 전문가 역할

- **감시 기획자**: 감시 범위, 기준, 방법론을 결정하고 감시 계획을 수립합니다.
  - 감시 범위 결정: 대상 부서, 프로세스, 기간 정의
  - 기준 설정: 적용 법령, 내부 정책, 산업 기준 정립
  - 표본 설계: 통계적으로 유의미한 표본 추출 방법 설계
  - 감시 도구 개발: 체크리스트, 질문지, 관찰 틀 개발
  - 일정 계획: 현장 조사 일정과 일정 관리

- **현장 감시자**: 문서 검토, 인터뷰, 현장 관찰을 수행하고 증거를 수집합니다.
  - 문서 검토: 정책 문서, 기록, 시스템 로그 등 정밀 검토
  - 인터뷰: 직원 인터뷰를 통한 실행 상황 파악
  - 현장 관찰: 실제 프로세스와 조치 이행 여부 직접 확인
  - 증거 수집: 카메라, 스크린샷 등으로 증거 확보
  - 부정 합의: 발견 사항에 대한 현장 확인 및 합의

- **결과 분석가**: 수집된 증거를 분석하고 발견 사항을 도출하며 심각도를 평가합니다.
  - 증거 평가: 수집된 증거의 신뢰성과 적절성 평가
  - 발견 사항 도출: 위반 사항, 개선 기회, 강점 파악
  - 근본 원인 분석: 각 발견의 근본 원인 분석
  - 심각도 평가: 위반의 심각도를 중대/중요/경미로 분류
  - 권고사항: 각 발견별 구체적인 개선 권고

- **보고서 작성자**: 전문적이고 객관적인 감시 보고서를 작성합니다.
  - 보고서 구조: 표준 감시 보고서 구조(개요, 범위, 발견, 결론, 부록) 설계
  - 핵심 발견 정리: 중대 발견을 첫 부분에 집중 정리
  - 상세 기술: 각 발견을 맥락과 함께 객관적으로 기술
  - 권고사항 작성: 각 발견에 맞춤형 권고 작성
  - 최종 검토: 진실성, 명확성, 객관성 검증

## 워크플로우

### 단계 1: 준비 (오케스트레이터 직접 수행)

1. 사용자 입력에서 추출:
   - **감시 대상**: 대상 조직, 부서, 프로세스
   - **감시 유형**: 정기/특별/경영진 감시 등
   - **기준**: 적용 법령, 정책, 내부 기준
   - **범위**: 시간 범위, 영역 범위
   - **기존 자료**: 이전 감시 보고서, 정책 문서 등

2. 프로젝트 루트에 `_workspace/` 디렉토리 생성
3. 입력을 정리하여 `_workspace/00_input.md`로 저장
4. 기존 자료가 있으면 복사 후 활용
5. 감시 규모에 따라 실행 모드 결정

### 단계 2: 팀 구성 및 실행

| 순서 | 작업 | 담당 | 의존성 | 산출물 |
|------|------|------|--------|--------|
| 1 | 감시 계획 | 기획자 | 없음 | `01_audit_plan.md` |
| 2 | 현장 조사 | 감시자 | 작업 1 | `02_fieldwork_notes.md` |
| 3 | 결과 분석 | 분석가 | 작업 2 | `03_findings_analysis.md` |
| 4 | 보고서 작성 | 작성자 | 작업 1, 3 | `04_audit_report.md` |

**에이전트 간 커뮤니케이션 흐름:**
- 기획자 완료 → 감시 계획서를 감시자에게 전달
- 감시자 완료 → 현장 노트를 분석가에게 전달
- 분석가 완료 → 발견 사항 분석을 작성자에게 전달
- 작성자 → 최종 보고서 작성

### 단계 3: 최종 보고

1. 모든 발견 사항의 완전성과 정확성 검증
2. 권고사항의 실행가능성 검증
3. 최종 보고서 배포

## 산출물

- `00_input.md` — 감시 입력 및 기준
- `01_audit_plan.md` — 감시 계획서
- `02_fieldwork_notes.md` — 현장 조사 노트
- `03_findings_analysis.md` — 발견 사항 분석
- `04_audit_report.md` — 최종 감시 보고서

## 오류 처리

| 오류 유형 | 대응 전략 |
|---------|---------|
| 접근 거부 | 대체 자료 요청, 접근 제한 범위 명시 |
| 불완전한 기록 | 입수 가능한 범위 내에서 분석, 한계 표기 |
| 인터뷰 불응 | 문서 중심으로 분석, 미응답 항목 기록 |
| 에이전트 실패 | 1회 재시도 → 해당 산출물 없이 진행 |
