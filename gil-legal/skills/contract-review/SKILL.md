# 계약서 검토 (Contract Review)

## 개요

계약서 검토 및 법률 문서 작성을 지원한다. 한국 민법·상법 기준 계약서 10대 리스크 패턴 분석, 이용약관·개인정보처리방침·SLA 작성, 2026년 개정 개인정보보호법 반영, 불리한 조항 수정 권고안을 제공한다.

**면책 고지: AI 보조 분석이며 전문 법률 자문을 대체하지 않습니다**

## 트리거 키워드

계약서, 계약 검토, 위험 조항, 이용약관, 개인정보처리방침, SLA, 민법, 상법, 법률 문서, 용역계약, 공급계약, 파트너십 계약, 협약서

## 워크플로우

### 1단계: 요청 유형 확인

**작업 유형:**
- **계약서 검토**: 기존 계약서의 위험 조항 분석 및 수정 권고
- **법률 문서 작성**: 이용약관, 개인정보처리방침, SLA 등 신규 작성

**문서 유형:**
- 용역계약 (SI, 개발 외주, 컨설팅)
- 공급계약 (제품, 소프트웨어 라이선스)
- NDA (비밀유지계약)
- 파트너십·제휴 계약
- 이용약관, 개인정보처리방침, SLA

**한국법 또는 외국법 적용 여부 확인**

### 2단계: 계약서 검토 (10대 리스크 체크)

**① 계약 당사자 정보 정확성**
- [ ] 상호명, 사업자등록번호, 대표자, 주소 확인
- [ ] 계약 당사자의 권한 범위 (대표이사·대리인 확인)

**② 계약 목적 및 범위 명확성**
- [ ] 용역·공급의 대상과 범위가 구체적일까?
- [ ] 성과물의 명세 ( Deliverables )가 명시되어 있을까?

**③ 대금·지급 조건 및 지연 이자**
- [ ] 계약금·중도금·잔금 지급 일정
- [ ] 지급 조건 (검수·승인 후 지급 등)
- [ ] 지연 이자율 (연 20% 법정 최고금리 이내 권장)

**④ 납기·이행 조건 및 위약 조항**
- [ ] 납품·완료 기일이 명확한가?
- [ ] 지연 시 위약벌(손해배상 예정)이 합리적인가?
- [ ] 불가항력(Force Majeure) 조항 포함 여부

**⑤ 지적재산권 귀속**
- [ ] 성과물의 지적재산권(특허, 저작권, 영업비밀) 귀속
- [ ] 기존 IP와 신규 IP의 구분
- [ ] 라이선스 범위(독점·비독점)

**⑥ 비밀유지 조항**
- [ ] 비밀정보의 정의와 범위
- [ ] 비밀유지 의무 기간 (일반: 2-5년, 영업비밀: 영구도 가능)
- [ ] 예외 조항 (공지 정보, 독자 개발, 제3자 취득, 법령 공개)

**⑦ 손해배상 범위 및 한도**
- [ ] 직접 손해 vs 간접 손해(특별 손해)
- [ ] 손해배상 한도 설정 (예: 계약금 총액의 100%)
- [ ] 특약사항(면책 조항)

**⑧ 계약 해지 조건 및 효과**
- [ ] 사전 해지 통지 기간 (통상 30일)
- [ ] 해지 사유 (사안, 쌍방 해지 권한)
- [ ] 해지 시 효과 (비용 정산, 성과물 반환)

**⑨ 분쟁 해결 (관할 법원, 준거법, 중재)**
- [ ] 합의 관할 법원 (당사자 일방所在地 법원 권장)
- [ ] 준거법 (한국법 적용 명시 권고)
- [ ] 조정·중재 조항 (선택 사항)

**⑩ 불공정 조항 (약관법 위반 가능성)**
- [ ] 고객에게 부당하게 불리한 조항
- [ ] 고객의 권리를 제한하는 조항
- [ ] 면책 조항의 과도한 제한

### 3단계: 법률 문서 작성 (선택)

**이용약관:**
- 전자상거래법·소비자보호법 준수
- 필수 포함 사항:
  - 서비스의 이용 방법과 이용 조건
  - 이용 요금 및 결제 방법
  - 이용 제한 및 해지 절차
  - 책임 한계 (면책 조항)
  - 분쟁 해결 (관할 법원 등)

**개인정보처리방침:**
- 개인정보보호법 (2026.9.11. 개정) 반영
- 필수 포함 사항:
  - 수집 항목 (필수·선택)
  - 이용 목적
  - 보유 기간
  - 제3자 제공 (유무, 대상, 목적)
  - 파기 방법 및 절차
  - 정보주체의 권리 (열람, 정정, 삭제)
  - 쿠키 사용 여부

**SLA (Service Level Agreement):**
- 가용성 목표 (예: 99.9% 업타임)
- 장애 등급별 대응 시간 (P1/P2/P3)
- 위반 시 보상 방법 (서비스 크레딧, 위약벌)
- 모니터링·보고 방법

### 4단계: 수정 권고안 제시

**리스크 조항 표시 및 대안 문구 제안:**
- 🔴 **즉시 수정 권고**: 법적으로 명백히 불리한 조항
- 🟡 **협상 권고**: 합의 가능한 조항
- 🟢 **수용 가능**: 일반적인 수준의 조항

**협상 포인트 우선순위:**
1. 지적재산권 귀속 (성과물 IP 소유)
2. 손해배상 한도 (무제한 책임 방지)
3. 계약 해지 조건 (불리한 해지 방지)
4. 관할 법원 (원거리 법원 방지)

## 사용 예시

**예시 1**: "IT 용역 계약서 검토해줘. 갑이 우리야. [계약서 내용]"
→ 10대 리스크 체크 → 불리한 조항 표시 → 수정 권고안 → 협상 포인트

**예시 2**: "SaaS 서비스 이용약관이랑 개인정보처리방침 만들어줘"
→ 서비스 범위 확인 → 이용약관 초안 → 개인정보처리방침 초안 → 검토

**예시 3**: "API 제휴 계약서 초안 작성해줘"
→ 제휴 조건 확인 → 계약서 초안 → 주요 조항 설명

## 산출물

- 계약서 검토 보고서 (10대 리스크 분석)
- 조항별 수정 권고안
- 협상 포인트 우선순위 목록
- 이용약관·개인정보처리방침·SLA 초안
- 계약서 작성 가이드

## 주의사항

**면책 고지: AI 보조 분석이며 전문 법률 자문을 대체하지 않습니다**

**계약서 검토 시 주의사항:**
- 서명 전 반드시 검토 (사후 수정은 어렵습니다)
- 전문 법률 용어는 쉬운 한국어로 해설
- 복잡한 계약(합작투자, M&A 등)은 법무법인 자문 권고

**법률 문서 작성 시 주의사항:**
- 전자상거래법·소비자보호법·개인정보보호법 등 관련 법령 준수
- 약관의 규제에 관한 법률(약관법) 위반 조항 방지
