# 계약서 검토 AI (Contract Reviewer)

기업 계약서, 이용약관, 계약 조건을 AI가 분석하여 **위험 조항**, **불리한 조건**, **누락된 조항**을 발견하는 스킬입니다.

## Overview

변호사 검토를 기본으로 하되, **빠른 초기 검토**, **비용 절감**, **법률 교육**을 목표로 합니다.

**주요 기능:**
- 위험 조항 자동 검출
- 개선 제안 (새로운 문구)
- 업계 표준 비교
- 체크리스트 제공
- 법적 위험도 평가

> **주의**: 이 스킬은 법률 정보 제공입니다.
> **실제 계약 체결 시 변호사 또는 법무사 검토 필수.**

## 지원 계약서 유형

| 유형 | 내용 | 검토 포인트 |
|------|------|-----------|
| **고용/근로** | 고용계약서, 근로계약서 | 급여, 근로시간, 퇴직금, 비밀유지, 경업금지 |
| **사업/용역** | 발주/수주계약, 용역계약 | 요금, 일정, 변경 절차, 하자책임, 배상 |
| **라이선스/IP** | 라이센스 계약, IP 양도 | 사용권 범위, 제한사항, 유효기간, 로열티 |
| **임차/임대** | 건물임차, 사무실임차 | 임차기간, 임차료, 보증금, 중도해지, 수선의무 |
| **구매/판매** | 매매계약, 공급계약 | 상품/서비스 명세, 가격, 납기, 하자책임, 반품 |
| **금융/대금** | 대출계약, 신용카드약관 | 금리, 수수료, 담보, 연체료, 전자서명 |
| **파트너십** | 파트너 계약, 협력 계약 | 역할, 수익배분, 해지 조건, 경쟁금지 |
| **이용약관** | 웹사이트, 앱 이용약관 | 책임 제한, 개인정보, 지적재산권, 분쟁해결 |

## 검토 프로세스

### Step 1: 계약서 업로드 & 기본 정보 수집

```
사용자 입력:
1. 계약서 파일 (PDF, DOCX, 이미지)
2. 계약 유형 선택
3. 당사자 입장 (발주자? 수주자? 임대인? 임차인?)
4. 핵심 조건:
   - 계약금액
   - 계약기간
   - 당사자명
```

### Step 2: 자동 분석
```
AI 검토 항목:

■ 위험 조항 탐색
  - "당 회사는 책임을 지지 않는다" (책임 회피)
  - "상대방이 마음대로 해지 가능" (일방적 해지)
  - "무제한 손해배상" (과도한 배상)
  - "합의 없는 가격 변경" (변경 자유도)

■ 누락 조항 검사
  - 분쟁 해결 절차
  - 보험 요구사항
  - 기밀 유지 조항
  - 지적재산권 소유권
  - 중도 해지 조건

■ 업계 표준 비교
  - 일반적인 금액 범위
  - 계약기간 관례
  - 배상 한도 기준
```

### Step 3: 결과 제공

```
리포트 구성:

1. 위험도 평가
   ■ 전체 위험도: 🔴 높음 / 🟡 중간 / 🟢 낮음
   ■ 점수: 65/100

2. 주요 문제점 (우선순위)
   ■ 🔴 [높음] 책임 제한 조항이 한쪽에만 유리함
   ■ 🟡 [중간] 분쟁해결 절차 미규정
   ■ 🟢 [낮음] 보험 요건 확인 필요

3. 개선 제안
   [기존 문구]
   "당 회사는 모든 손해에 책임을 지지 않습니다."
   
   [개선 제안]
   "당 회사는 본인의 중과실로 인한 손해에만 책임을 지며,
   간접손해, 영업이익 손실은 제외합니다.
   손해배상액은 이 계약의 총 금액을 초과하지 않습니다."

4. 체크리스트
   □ 양 당사자의 역할이 명확한가?
   □ 계약금액과 결제 방식이 명시되어 있는가?
   □ 기한 및 납기가 합리적인가?
   □ 분쟁 해결 절차(중재/소송)가 있는가?
   □ 양쪽 모두 해지 가능 조건이 있는가?

5. 법률 팁
   "한국 법상 계약서의 '책임제한 조항'은 
   현저히 불공정하면 효력이 없을 수 있습니다.
   (상법 제108조, 약관규제법)"
```

## 주요 검토 항목

### 1. 당사자 및 기본 조건
```
확인 사항:
□ 당사자명이 정확하고 법적 이름인가? (상호명, 사업자등록번호)
□ 계약 날짜가 명시되어 있는가?
□ 계약금액이 숫자와 문자로 명확히 기재되어 있는가?
□ 계약기간이 명확한가? (예: 2024.04.01 ~ 2024.12.31)
```

### 2. 의무 및 책임
```
체크포인트:
□ A의 의무는? (명확한가? 달성 가능한가?)
□ B의 의무는? (동등한가? 불평등하지는 않은가?)
□ 각 의무 미충족 시 위반 조항이 있는가?
□ 손해배상 대상과 한도가 명확한가?
□ 책임 제한이 일방적이지는 않은가?
```

### 3. 지적재산권 (IP)
```
확인:
□ 저작물, 특허, 상표의 소유권이 명확한가?
□ 사용 라이선스의 범위가 제한되어 있는가? (상용, 비상용, 지역, 기간)
□ 제3자 저작권 침해 시 책임이 규정되어 있는가?
□ 계약 종료 후 사용 라이선스는 종료되는가?
```

### 4. 비밀 유지 (NDA)
```
점검:
□ 비밀 정보의 정의가 명확한가?
□ 의무 기간이 명시되어 있는가? (예: 계약 종료 후 2년)
□ 법적 공개, 강제 공개의 예외가 있는가?
□ 위반 시 손해배상 범위는?
```

### 5. 분쟁 해결
```
필수 요소:
□ 협의/조정 절차가 있는가?
□ 중재 조건이 명시되어 있는가? (중재기관, 비용)
□ 소송 관할권이 명확한가? (어느 법원?)
□ 준거법이 명시되어 있는가? (한국 법?)
```

### 6. 해지 및 종료
```
점검:
□ 정상 만료 조건이 명확한가?
□ 조기 해지 조건이 있는가? (일방 해지? 합의 해지?)
□ 해지 통지 기간이 있는가? (예: 30일 전 통지)
□ 계약 종료 후 처리 사항(데이터, 자산 반환 등)이 명시되어 있는가?
```

## 위험 조항 패턴

### 🔴 높은 위험
```
1. "회사는 어떤 책임도 지지 않습니다"
   → 법적으로 무효일 가능성. 동의 불가

2. "상대방이 언제든 일방적으로 해지 가능"
   → 계약의 안정성 없음. 대등한 조건 추가 필요

3. "배상액은 무제한"
   → 기한 없는 손해배상. 한도 설정 필수

4. "변경/취소는 회사 판단에 따름"
   → 일방적 권한 집중. 협의 절차 추가

5. "모든 분쟁은 상대방이 지정한 법원"
   → 소송 부담 증가. 중립 관할권 협상
```

### 🟡 중간 위험
```
1. "합리적 범위 내에서 가격 변경 가능"
   → 정의 불명확. 구체적 조건 명시 필요

2. "최선을 다해 서비스 제공"
   → 책임 범위 모호. 명확한 기준 필요

3. "분쟁 발생 시 협의"
   → 절차 없음. 중재/소송 조건 추가

4. "계약서는 영문이 우선"
   → 한글 해석 어려움. 한글 정확성 확인
```

### 🟢 낮은 위험
```
1. "상호 합의로 계약 변경 가능"
   → 정상적 조건. 구체적 절차 추가 권장

2. "업계 표준 관례에 따름"
   → 명확. 업계 표준 문서 참조

3. "양 당사자는 동등한 의무를 가짐"
   → 공정함. 특정 의무 확인
```

## 체크리스트 템플릿

### 일반 계약서 (모든 유형)
```
□ 당사자명 (정식 법인명/사업자등록번호)
□ 계약금액 (숫자 + 문자 명시)
□ 계약기간 (시작일~종료일)
□ A의 의무 / B의 의무
□ 대금 지급 방식 및 일정
□ 하자/문제 발생 시 처리 절차
□ 손해배상 대상 및 한도
□ 비밀 유지 조항
□ 지적재산권 소유권
□ 해지 조건 (정상/조기/강제)
□ 분쟁 해결 절차 (협의→중재→소송)
□ 관할 법원 (준거법)
□ 양 당사자 서명 (또는 전자서명)
□ 발효일
```

## 절대 규칙 (Hard Rules)

### 1. 법률 자문 아님
```
⚠️ 이 검토는 법률 정보/교육입니다.
실제 계약 체결/분쟁 발생 시 반드시:
- 변호사 또는 법무사 상담
- 공식 법률 의견 수렴
- 계약 체결 전 전문가 검토
```

### 2. 관할권 제한
```
한국법 기준 검토 제공
- 계약서가 한국 당사자를 포함
- 준거법이 한국법 또는 명시 안 됨

해외 계약서:
- 해당 국가의 변호사 검토 필수
- 이 스킬은 참고만 가능
```

### 3. 비밀 보호
```
계약서 내용 기밀 보호:
- 사용자 계약서 저장 안 함
- 분석 후 삭제
- 제3자 공유 안 함
```

### 4. 면책 조항
```
이 스킬 제공 정보:
□ 법률 조언이 아님
□ 변호사-의뢰인 특권 없음
□ 정확성 보장 안 함
□ 손해 배상 책임 없음
```

```

자동 분석 (위험도 평가)

주요 문제점 식별
```

## 트러블슈팅

| 증상 | 원인 | 해결 |
|------|------|------|
| "분석 불가" | 파일 손상 또는 이미지 불명확 | 깨끗한 PDF/DOCX 재업로드 |
| 한글 오류 | 이미지 텍스트 인식 오류 | 텍스트 가능한 PDF 사용 |
| 결과 부정확 | 문맥 부족 | 전체 계약서 업로드 (일부만 아님) |
| 범위 밖 계약 | 지원하지 않는 유형 | 지원 유형 목록 확인 |
| 법률 상담 필요 | AI 분석만으로 충분 않음 | 변호사 상담 권장 |
