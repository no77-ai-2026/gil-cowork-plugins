# UX 연구 (harness-107)

> MoAI-Cowork v1.0.0 하네스 참고 자료

## 개요
UX 연구(사용자 면접 설계→사용성 테스트→페르소나 개발→여정 매핑→인사이트 합성)를 통해 사용자 중심 설계를 주도하는 하네스입니다.

## 전문가 역할

- **사용자 면접 설계자**: 면접 가이드 개발, 질문 설계, 참여자 모집
  - 면접 가이드: 반구조화된 면접 가이드 개발 (시간, 질문, 프로토콜)
  - 질문 설계: 편향 최소화, 깊이 있는 탐색, 행동 기반 질문
  - 대화 기법: 공감, 경청, 깊이 있는 탐구 기법
  - 참여자 모집: 타겟 사용자 선정 및 모집 전략
  - 분석 프레임워크: 정성적 데이터 합성 방법

- **사용성 테스트 조정자**: 테스트 계획, 작업 설계, 참여자 모집
  - 테스트 계획: 목표, 가설, 시나리오, 측정 항목 정의
  - 작업 설계: 자연스러운 작업 시나리오 및 성공 기준
  - 테스트 환경: 테스트 설정, 기록 방법, 옵저버 가이드
  - 참여자 모집: 타겟 사용자 프로필, 모집 경로
  - 분석 및 보고: 관찰, 오류, 사용성 이슈 기록 및 분류

- **페르소나 개발자**: 사용자 세그먼트, 특성, 목표, 행동 패턴 정의
  - 데이터 수집: 정성/정량 데이터로부터 패턴 식별
  - 세그먼트 정의: 주요 사용자 세그먼트 구분
  - 페르소나 작성: 배경, 목표, 통증점, 선호도 정의
  - 시나리오: 각 페르소나의 사용 시나리오 개발
  - 우선순위: 주요 페르소나 vs 부차 페르소나 구분

- **여정 매핑 전문가**: 터치포인트 식별, 통증점, 개선 기회 발굴
  - 여정 정의: 사용자 목표 달성 전체 경로 매핑
  - 터치포인트: 제품과의 모든 상호작용 지점 식별
  - 감정 곡선: 각 터치포인트에서의 사용자 감정 시각화
  - 통증점: 사용자의 문제, 좌절, 장애 요인 식별
  - 개선 기회: 각 터치포인트별 개선 및 기회 제시

- **인사이트 합성자**: 연구 결과 종합, 패턴 도출, 권고사항 제시
  - 데이터 종합: 면접, 테스트, 조사 결과 통합 분석
  - 패턴 도출: 공통 행동, 태도, 문제점 도출
  - 인사이트 정의: 설계 의사결정에 영향을 미칠 발견
  - 권고사항: 구체적이고 실행 가능한 설계 권고
  - 보고 및 전파: 팀 전체에 결과 공유 및 이해

## 워크플로우

### Phase 1: 준비

- 연구 목표 및 가설 정의
- 타겟 사용자 및 참여자 선정 기준 수립
- 기존 사용자 데이터 및 분석 결과 수집
- 연구 방법 및 일정 계획

### Phase 2: 사용자 연구 수행

- 사용자 면접 실행 및 기록
- 사용성 테스트 실행 및 관찰
- 사용자 설문 조사 실행 (필요 시)
- 데이터 정리 및 초기 분석

### Phase 3: 분석 및 설계 권고

- 페르소나 개발 및 여정 매핑
- 통증점 및 개선 기회 식별
- 설계 권고사항 도출
- 팀에 결과 공유 및 토론

## 산출물

- 사용자 면접 가이드 및 분석 결과
- 사용성 테스트 보고서 (문제, 영향도)
- 사용자 페르소나 (3-5개)
- 사용자 여정 맵 (페르소나별)
- UX 인사이트 및 설계 권고사항
- 연구 발표 자료 (팀, 경영진)

## 오류 처리

| 오류 유형 | 대응 전략 |
|----------|----------|
| 참여자 모집 어려움 | 조정자가 대체 모집 경로 및 인센티브 전략 |
| 데이터 부족 | 각 전문가가 기존 연구 자료 활용 제안 |
| 참여자 불충분 | 설계자가 3-5명 기본 면접으로 패턴 도출 |
| 분석 복잡성 | 합성자가 친화성 다이어그램 등 시각화 기법 |
| 팀 수용성 저조 | 합성자가 이해하기 쉬운 요약 및 영상 제공 |
