# 채용 파이프라인 (90-hiring-pipeline)

> MoAI-Cowork v1.0.0 하네스 참고 자료

## 개요
채용 프로세스(공고 작성→소싱→스크리닝→면접→평가→오퍼)를 에이전트 팀이 협업하여 완성하는 하네스입니다.

## 전문가 역할

- **면접 설계자**: 면접 구조 설계, 역량 기반 질문, 면접관 가이드, 평가 양식 설계 전문가
  - 면접 단계별 시간, 면접관 구성 설계
  - STAR 기법 기반 행동 면접 질문 설계
  - 직무별 기술 역량 평가 방식 설계
  - 면접관 교육 자료 및 진행 가이드 작성
  - 구조화된 평가 양식 설계

- **공고 작성자**: 직무 분석, 역량 정의, 직무 기술서(JD), 채용 공고 작성 전문가
  - 핵심 책임, 기대 성과, 보고 관계 정의
  - 필수(Must-have) 및 우대(Nice-to-have) 역량 구분
  - 정식 공고 작성
  - 외부 채용 공고 작성
  - 평가 및 면접 기준이 되는 역량 기준 수립

- **오퍼 조정자**: 최종 평가 보고서, 오퍼 패키지 설계, 오퍼 작성, 온보딩 담당
  - 전체 단계 평가 결과 통합 분석
  - 등급별, 인원별 혜택 포함 패키지 설계
  - 정식 채용 제안서 작성
  - 등급/조건 협상 전략 수립
  - 파이프라인 일관성 검증

- **스크리닝 전문가**: 평가 기준 정의, 과제 설계, 스크리닝 가이드 작성
  - 역량별 포트폴리오 평가 설계
  - 직무 관련 과제 또는 테스트 설계
  - 면접 질문 및 평가 기준 설계
  - 후보자 비교 스코어카드 설계
  - 단계별 시간 기준 설정

- **소싱 전문가**: 채용 채널 전략, 탐색 시스템 구축, 메시지 설계
  - 직무별 양질의 소싱 채널 선정
  - 후보자 검색 및 부울 검색 전략 설계
  - 후보자 접근 메시지/이메일 템플릿 작성
  - 회사/팀 역량 홍보 콘텐츠 설계
  - 채널별 반응, 비용, 기간 추적 지표 설정

## 워크플로우

### Phase 1: 준비 (오케스트레이터 직접 수행)

1. 사용자 입력에서 추출:
   - **직급**: 채용 직급
   - **조직 정보**: 회사 규모, 팀 구성
   - **채용 조건**: 예산, 인원, 기한
   - **기존 자료** (선택): 기존 공고, 현황

2. 프로젝트 루트에 `_workspace/` 디렉토리 생성
3. 입력값을 정리하여 `_workspace/00_input.md`에 저장
4. 기존 자료가 있으면 적용 Phase 결정
5. 요청 범위에 따라 **실행 모드 결정**

### Phase 2: 팀 구성 및 실행

| 순서 | 작업 | 담당 | 의존성 | 산출물 |
|------|------|------|--------|--------|
| 1 | 공고 작성 | 공고 작성자 | 없음 | `_workspace/01_job_description.md` |
| 2 | 소싱 전략 | 소싱 전문가 | 작업 1 | `_workspace/02_sourcing_strategy.md` |
| 3 | 스크리닝 설계 | 스크리닝 전문가 | 작업 1 | `_workspace/03_screening_framework.md` |
| 4 | 면접 설계 | 면접 설계자 | 작업 1, 3 | `_workspace/04_interview_design.md` |
| 5 | 평가·오퍼 | 오퍼 조정자 | 작업 1-4 | `_workspace/05_evaluation_offer.md` |

작업 2(소싱)와 3(스크리닝)은 **병렬 실행**, 작업 1에만 의존합니다.

**팀 간 협업 흐름:**
- 공고 작성자 완료 → 소싱 전문가에 타겟/EVP, 스크리닝 전문가에 역량·평가 기준 전달
- 소싱 전문가 완료 → 오퍼 조정자에 시장 보고 수준 전달
- 스크리닝 전문가 완료 → 면접 설계자에 필요 역량 확인 전달
- 면접 설계자 완료 → 오퍼 조정자에 평가 양식·채용 권고 기준 전달
- 오퍼 조정자는 전체 파이프라인 일관성 검증, 문제 발견 시 수정 요청 (최대 2회)

### Phase 3: 통합 및 최종 산출

1. `_workspace/` 모든 파일 검증
2. 오퍼 일관성 검증 결과 반영
3. 최종 요약 사용자에 보고

## 산출물

모든 산출물은 `_workspace/`에 저장됩니다:
- `00_input.md` — 사용자 입력 정리
- `01_job_description.md` — 공고 및 채용 공고
- `02_sourcing_strategy.md` — 소싱 전략
- `03_screening_framework.md` — 스크리닝 프레임워크
- `04_interview_design.md` — 면접 설계
- `05_evaluation_offer.md` — 최종 평가 및 오퍼 가이드

## 오류 처리

| 오류 유형 | 대응 전략 |
|----------|----------|
| 직무 정보 부족 | 공고 작성자가 템플릿 및 가이드 제공 |
| 예산 정보 없음 | 시장 조사 기반, 3단계 범위 제안 |
| 검색 실패 | 일반적인 JD/소싱 전략 기반, "[시장 데이터 미반영]" 표시 |
| 에이전트 오류 | 1회 재시도 → 해당 산출물 없이 진행 |
| 일관성 문제 | 오퍼 조정자 수정 요청 → 재작업 (최대 2회) |
