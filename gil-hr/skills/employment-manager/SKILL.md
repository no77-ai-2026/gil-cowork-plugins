## 4대보험 취득신고 절차

신규 직원 입사 시 사업주는 4대보험 취득신고를 기한 내 완료해야 합니다.

| 보험 | 신고 기한 | 신고 기관 |
|------|-----------|-----------|
| 국민연금 | 입사일이 속하는 달의 다음달 15일까지 | 국민연금공단 |
| 건강보험 | 입사일로부터 14일 이내 | 국민건강보험공단 |
| 고용보험 | 입사일이 속하는 달의 다음달 15일까지 | 근로복지공단 |
| 산재보험 | 입사일이 속하는 달의 다음달 15일까지 | 근로복지공단 |

4대보험 통합 신고: [4insure.kr](https://www.4insure.kr) (온라인 일괄 신고 가능)

상세 절차 및 서류 목록: `references/insurance-registration.md`

## 독립 실행 워크플로우

참조 가이드(`references/`)를 사용할 수 없는 경우 다음 단계로 실행합니다.

### 1단계: 요청 유형 파악
- 채용 파이프라인 (JD, 면접, 평가) 또는 온보딩 (신입 교육, 멘토링) 구분
- 직군 정보 수집: 직무명, 경력 수준 (신입/주니어/시니어), 팀 규모

### 2단계: JD 작성 (채용 파이프라인)
```
[직무명] 채용 공고

■ 담당 업무
  - [핵심 업무 3~5가지]

■ 자격 요건 (필수)
  - [경력, 기술, 자격증 등]

■ 우대 사항
  - [있으면 좋은 요건]

■ 근무 조건
  - 고용 형태: [정규직/계약직]
  - 근무지: [위치 또는 원격 여부]
  - 연봉: [협의 또는 범위 공개]
```

### 3단계: 면접 설계
- 서류 전형 → 1차 실무 면접 → 2차 팀/문화 면접 → 처우 협의 구성
- 면접 질문: 직무 역량 (60%), 협업/문화 적합성 (30%), 성장 의지 (10%)
- 평가 루브릭: 5점 척도 + 합격/보류/불합격 기준 명시

### 4단계: 온보딩 계획
```
1주차: 오리엔테이션 (조직 이해, 시스템 접근 권한 설정, 팀 소개)
2주차: 직무 교육 (업무 도구, 프로세스, 첫 과제 부여)
3~4주차: 실전 투입 (멘토 동행, 일일 체크인)
2~3개월: 독립 수행 (주간 1:1 면담, 30/60/90일 목표 점검)
```

### 5단계: 산출물 생성
- 요청에 맞는 최종 문서(JD/면접 질문/온보딩 계획) 생성
- 사용자 검토 후 수정 반영

## 실행 규칙

1. 사용자 요청 수신 → 해당 영역 판별
2. `references/{id}.md` 존재 시 로드 → 전략 가이드에 따라 실행
3. `--deepthink` 또는 복잡 채용 설계 → `mcp__sequential-thinking__sequentialthinking` 호출
4. 결과물 생성 후 사용자 검토 요청

## 문제 해결

- **근로기준법 조항 불확실**: AI가 제공하는 법령 내용은 참고용입니다. 채용 관련 법적 판단은 노무사에게 확인하세요.
- **직급 체계 정보 부족**: 회사 직급 구조(팀장/리더/시니어 등)와 팀 조직도를 제공하면 더 정확한 JD와 온보딩 계획을 작성할 수 있습니다.
- **업종별 채용 관행 차이**: IT, 제조, 서비스 등 업종마다 채용 절차가 다릅니다. 회사 규모와 업종을 알려주세요.
- **외국인 채용**: 비자 종류 및 취업 자격 확인이 필요합니다. 출입국관리법 전문가와 병행 검토하세요.
