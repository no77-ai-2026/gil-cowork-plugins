# OKR/MBO 설정 가이드

> gil-hr | performance-review 참조 가이드 | 2026 기준

## 개요

OKR(Objectives & Key Results)과 MBO(Management by Objectives)는 국내 기업에서 가장 많이 사용하는 목표 관리 프레임워크입니다. 이 가이드는 실제 설정부터 평가 주기 운영까지 실무 절차를 안내합니다.

---

## 1. OKR vs MBO 선택 기준

| 기준 | OKR | MBO |
|------|-----|-----|
| 평가 연동 | 원칙적 분리 (도전적 목표 설정 유도) | 연동 (급여·인사 결정) |
| 설정 주기 | 분기 | 반기 또는 연간 |
| 달성 기준 | 70% 달성이 이상적 | 100% 달성 목표 |
| 적합 기업 | 스타트업·성장기 기업 | 안정적 운영 기업 |
| 투명성 | 전사 공개 원칙 | 팀/개인 수준 관리 |

---

## 2. OKR 작성법

### Objective (목적) 작성 원칙

- 정성적이고 영감을 주는 문장
- "왜 중요한가?"에 답해야 함
- 측정값 없이 방향성만 표현
- 예: "고객이 우리 서비스를 사랑하게 만든다"

### Key Results (핵심 결과) 작성 원칙

- 반드시 측정 가능한 수치 포함
- Objective당 2-5개
- 현재 상태 → 목표 상태 형식
- 예: "NPS 45점 → 62점 달성"

### OKR 작성 체크리스트

- [ ] Objective는 팀의 분기 우선순위를 반영하는가?
- [ ] Key Results는 모두 수치로 검증 가능한가?
- [ ] 달성 시 Objective가 실질적으로 전진하는가?
- [ ] 상위 팀 OKR과 정렬되어 있는가?
- [ ] 개인이 주도적으로 달성할 수 있는 범위인가?

## 3. MBO 설계

### KPI 설계 원칙 (SMART)

- **Specific**: 구체적인 성과 지표
- **Measurable**: 수치로 측정 가능
- **Achievable**: 도전적이지만 현실적
- **Relevant**: 팀/조직 목표와 연계
- **Time-bound**: 명확한 마감 기한

### MBO 목표 배분 비율 예시

| 구분 | 가중치 | 설명 |
|------|--------|------|
| 정량 목표 | 60-70% | 매출, 건수, 달성률 등 |
| 정성 목표 | 20-30% | 프로세스 개선, 역량 개발 |
| 공통 목표 | 10-20% | 팀/조직 공통 KPI |

## 4. 평가 주기 운영

### 분기 OKR 사이클

| 시기 | 활동 |
|------|------|
| 분기 시작 1-2주 | OKR 설정 및 정렬 완료 |
| 중간 점검 (6-7주차) | 진척도 체크인, 조정 필요 시 수정 |
| 분기 종료 1주 전 | 자가 평가 작성 |
| 분기 종료 시 | OKR 리뷰 + 다음 분기 설정 |

### 연간 MBO 사이클

| 시기 | 활동 |
|------|------|
| 1월 | 연간 목표 설정 (상향식+하향식 조율) |
| 4월 | 1분기 점검 |
| 7월 | 상반기 중간 평가 |
| 10월 | 3분기 점검 |
| 12월 | 연간 최종 평가, 내년 목표 초안 |

---

## 5. 공통 실수와 해결책

| 실수 | 해결 방법 |
|------|-----------|
| Key Results가 활동(to-do) 수준 | 성과(outcome) 중심으로 재작성 |
| 너무 많은 OKR (5개 이상) | 3개 이하로 압축, 우선순위 재정렬 |
| 평가와 OKR 강제 연동 | OKR은 학습용, 평가는 별도 프로세스로 분리 |
| 목표 변경 없이 방치 | 분기 중반 공식 조정 기회 제공 |
