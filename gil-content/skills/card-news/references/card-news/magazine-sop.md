# 매거진형 카드뉴스 SOP 템플릿

매거진 편집실의 역할 분담 구조를 카드뉴스 자동화에 적용한다.
블로그 참고: hyojin.rgb의 "카드뉴스 자동화를 만든 여정"

---

## 프로세스 개요

```
브랜딩 전략 수립 → 에디토리얼 전략 → 완성본 레퍼런스 확보 → SOP 기반 실행
```

각 단계를 담당하는 AI 페르소나:

| 역할 | 담당 업무 | 핵심 원칙 |
|------|----------|----------|
| **에디터** | 글 구조 설계, 텍스트 작성 | 구조 중심, 감정 절제, 속도감 |
| **콘텐츠 마케터** | 슬라이드 레이아웃, 디자인 실행 | 폰트·레이아웃 엄격 준수, Figma 호환 |
| **포토그래퍼** | 이미지 프롬프트, 스타일 일관성 | 무드보드 기반, 입출력 기록, 스타일 통일 |

---

## 1. 브랜딩 전략 템플릿

사용자가 브랜딩 전략을 제공하지 않았을 때, 아래 항목을 빠르게 수집:

### 필수 항목 (최소한)
- **브랜드명**: 카드뉴스 하단/표지에 표기할 이름
- **핵심 가치**: 한 줄로 "무엇을 전달하는 채널인가?"
- **타겟 독자**: 누구를 위한 콘텐츠인가? (예: 3-10년차 마케터)

### 선택 항목 (있으면 품질 향상)
- **브랜드 인격**: 어떤 톤으로 말하는 채널인가? (예: 차분한 분석가, 친근한 선배)
- **비주얼 키워드**: 색상, 분위기 (예: 미니멀, 다크모드, 매거진풍)
- **경쟁자 vs 차별점**: 다른 카드뉴스 채널과 뭐가 다른가?

## 2. 에디터 SOP

### 글쓰기 7단계 플로우

```
1. 소스 분석   → URL/주제에서 핵심 포인트 추출
2. 구조 설계   → 슬라이드 수·순서 결정 (스토리보드)
3. 후킹 작성   → 표지 문구 3안 → 사용자 선택 또는 자동 결정
4. 본문 작성   → 1슬라이드=1아이디어, 속도감 있는 문장
5. CTA 작성    → 마무리 슬라이드 (저장/공유 유도)
6. 톤 점검     → anti-ai-writing.md 체크리스트 적용
7. 캡션 작성   → 인스타 캡션 + 해시태그
```

### 글쓰기 전략 (매거진 B 방식)

**문장 원칙:**
- 숫자·장면·구조로 시작. 감정으로 시작하지 않는다.
- 형용사 최소화. 사실과 맥락 중심.
- 문단은 3-4문장. 접속사 최소화.
- 추상어 → 구체적 고유명사·수치·소재로 대체.

**구조 원칙:**
- 평가하지 않는다. 사실을 제시한다.
- 결정 → 실행 → 결과의 순서로 서술.
- 정보로 끝내지 않고 해석의 여지를 남긴다.

**인터뷰/인물 콘텐츠:**
- 감정 고백이 아니라 의사결정의 맥락을 드러내는 장치.
- "가장 큰 전환점은?", "그 결정의 변수는?", "오판이었던 선택은?"

## 3. 콘텐츠 마케터 SOP

### 디자인 실행 플로우

```
1. 색상 결정   → 3색 법칙 적용, 브랜딩 전략 참조
2. 폰트 설정   → Pretendard 필수, 크기 체계 적용
3. 레이아웃    → 패턴 선택 (중앙형/상하형/분할형/매거진형)
4. HTML 생성   → 1080×1350px, base64 이미지 인라인
5. 가독성 점검 → 텍스트 오버레이 + 반투명 배경 확인
6. 반응 확인   → 브라우저에서 실제 렌더링 확인
```

### 디자인 안정성 확보 방법

블로그에서 효과적이었던 3가지:

1. **완성본 예시의 폰트와 레이아웃을 엄격히 따르라**고 명시
   - 사용자가 레퍼런스 이미지를 제공하면 최우선 참조
2. **HTML로 먼저 생성** 후 필요 시 Figma 변환
   - Figma 직접 생성보다 HTML→스크린샷→Figma가 안정적
3. **스타일 변수를 CSS 커스텀 프로퍼티로 관리**
   - `--bg-color`, `--text-color`, `--accent-color` 등

## 4. 포토그래퍼 SOP

### 이미지 프롬프트 개발 플로우

```
1. 스타일 키워드 정의 → 전체 슬라이드에 적용할 통일 스타일
2. 슬라이드별 프롬프트 작성 → 스타일 접미사 일관 적용
3. 이미지 생성 → Nano Banana Pro API
4. 결과 확인 → 스타일 일관성 점검
5. 프롬프트 기록 → 재사용을 위해 입출력 쌍 저장
```

### 스타일 일관성 유지 방법

**통일 스타일 접미사 예시:**
```
editorial magazine style, muted earth tones, soft film grain,
natural lighting, minimal composition
```

이 접미사를 모든 슬라이드 프롬프트 끝에 동일하게 붙인다.

**프롬프트 기록 형식:**
```
[슬라이드 1 - 표지]
프롬프트: "..."
결과: 성공/실패, 특이사항
스타일 노트: 색감이 너무 밝음 → "desaturated" 추가

[슬라이드 2 - 본문]
프롬프트: "..."
결과: 성공, 텍스트 공간 충분
```

최소 2개 이상의 예시를 기록하여 다음 제작 시 참조한다.

## 5. 재사용 패턴

### 패턴 A: SOP 파일 직접 첨부 (현재 기본)
- 이 파일(magazine-sop.md)을 세션에 로드
- 브랜딩 전략 + 에디토리얼 전략을 함께 첨부
- Claude에게 "이 SOP대로 카드뉴스를 만들어줘" 지시

### 패턴 B: 스킬 단축 실행
- SOP가 충분히 정제되면 스킬 자체에 설정값 포함
- "카드뉴스 만들어줘"만으로 브랜딩·스타일 자동 적용

### 패턴 C: 장시간 작업 시 인수인계
상호작용이 길어져 컨텍스트가 무거워지면:
1. "지금까지 작업 내용을 인수인계서로 정리해줘" 요청
2. 새 세션 시작
3. 인수인계서를 투입하여 이어서 작업
