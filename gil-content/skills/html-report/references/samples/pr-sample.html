핵심 설계 결정

1

Redis Pub/Sub 기반 인스턴스 간 이벤트 전파

단순 인메모리 이벤트 버스로는 로드 밸런서 뒤에서 특정 인스턴스에 연결된 클라이언트에게만 이벤트가 전달됩니다. Redis 채널을 허브로 삼아 모든 인스턴스가 구독하도록 설계해 수평 확장 시에도 유실 없이 브로드캐스트됩니다.

2

지수 백오프 자동 재연결

모바일 네트워크 불안정 시 폴링 폭풍(polling storm)을 막기 위해 클라이언트 측에서 1s → 2s → 4s → 8s (최대 30s) 지수 백오프를 적용합니다. @stomp/stompjs의 reconnectDelay 콜백에서 구현, 탭이 비활성이면 재연결 시도를 중단합니다.

3

피처 플래그로 단계적 롤아웃

모든 사용자에게 즉시 노출하지 않고 Unleash 피처 플래그 ws_notification_v1로 10% → 50% → 100% 단계 배포합니다. 플래그가 꺼진 사용자는 기존 폴링(30초) 방식으로 폴백됩니다.
