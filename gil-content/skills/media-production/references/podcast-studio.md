# 팟캐스트 스튜디오 (podcast-studio)

> MoAI-Cowork v1.0.0 하네스 참고자료

## 개요

기획, 조사, 스크립팅, 쇼 노트, 배포 전략을 통해 팟캐스트 콘텐츠를 협력하여 제작하는 하네스입니다.

## 전문가 역할

- **배포 관리자**: 플랫폼별 메타데이터 생성, 에피소드 설명, 홍보 카피, 소셜 미디어 포스트, 뉴스레터 소개 작성
  - 플랫폼별 메타데이터: Apple Podcasts, Spotify, YouTube Podcasts 등 주요 플랫폼 최적화
  - 에피소드 설명: 플랫폼 검색과 발견 최적화 키워드 풍부 설명
  - 소셜 미디어 홍보 카피: Twitter/X, Instagram, LinkedIn, Facebook용 홍보 포스트 생성
  - 뉴스레터 소개: 이메일 구독자용 에피소드 소개 작성
  - 홍보 캘린더: 발행 전/후 홍보 스케줄 제안

- **프로덕션 검수자**: QA 역할, 조사-스크립트-쇼노트-배포 일관성 검증, 갭과 모순 파악
  - 조사-스크립트 정렬: 조사 브리프의 핵심 사실과 포인트가 스크립트에 정확히 반영되었는가?
  - 스크립트-쇼노트 정렬: 쇼노트 타임스탬프와 요약이 스크립트 구조와 일치하는가?
  - 스크립트-배포 정렬: 홍보 카피가 실제 에피소드 콘텐츠를 정확히 반영하는가?
  - 팩트 정확도: 스크립트에 인용된 통계/사실이 조사 브리프의 출처와 일치하는가?
  - 품질 체크리스트: 톤 일관성, 길이, CTA, 법률 검토 등

- **조사자**: 깊이 있는 주제 조사, 팩트 체킹, 게스트 배경 조사, 참고자료 컴파일, 포인트 추출
  - 깊이 있는 주제 조사: 사용자 주제에 대한 최신 트렌드, 통계, 케이스 스터디 수집
  - 팩트 체킹: 웹 검색으로 인용된 수치, 조사 결과, 전문가 의견의 정확성 검증
  - 게스트 배경 조사: 게스트 관련 시 경력, 출판물, 최근 성명, 소셜 미디어 활동 조사
  - 경쟁사 팟캐스트 분석: 동일 주제의 기존 에피소드 조사, 차별화 기회 파악
  - 포인트 추출: 스크립트 작가가 즉시 사용 가능한 구조화된 포인트 생성

- **스크립트 작가**: 조사 브리프 기반 청자 유지 최적화 에피소드 스크립트 작성, 타임코드 기반 스크립트 생성
  - 오프닝 훅: 처음 30초 내 청자 주의 집중 오프닝 설계 (충격 통계/질문/스토리)
  - 세그먼트 구조: 포인트를 논리적 흐름으로 배열, 자연스러운 전환
  - 대화 큐: `[호스트 질문]`, `[게스트 응답 프롬프트]`, `[청자 참여 프롬프트]` 등 지시
  - 톤 및 페이싱: 에피소드 유형(솔로/인터뷰/패널)에 맞춘 대화 리듬 설계
  - CTA 설계: 구독, 리뷰, 공유, 커뮤니티 참여를 대화에 자연스럽게 녹이기

- **쇼노트 편집자**: 에피소드 요약, 타임스탬프, 참고자료 링크, 게스트 소개, 주요 인용구를 구조화된 쇼노트로 정리
  - 에피소드 요약: 3-5문장 요약 + 1줄 엘리베이터 피치로 시청자가 청취 여부 판단하도록 도움
  - 타임스탬프 생성: 스크립트의 세그먼트 구조 기반 정확한 타임스탬프 생성
  - 참고자료 컴파일: 에피소드에서 언급된 책, 논문, 웹사이트, 도구를 클릭 가능한 링크로 구성
  - 주요 인용구 추출: 소셜 미디어 공유에 적합한 1-3개 주요 인용구 선정
  - 게스트 소개: 게스트 웹사이트 및 소셜 프로필 링크 포함 게스트 이력서 작성

## 워크플로우

### Phase 1: 준비 (오케스트레이터 직접 수행)

1. 사용자 입력에서 추출:
   - **주제/키워드**: 에피소드가 다룰 주제
   - **팟캐스트 정보** (선택): 쇼 톤, 청자 기반, 기존 에피소드 방향
   - **에피소드 유형** (선택): 솔로/인터뷰/패널/스토리텔링/Q&A
   - **게스트 정보** (선택): 게스트 이름, 전문 분야
   - **제약사항** (선택): 에피소드 길이, 특정 요구사항
   - **기존 파일** (선택): 스크립트, 조사 자료 등 사용자 제공 자료

2. `_workspace/` 디렉토리 생성
3. 입력을 정리하여 `_workspace/00_input.md`로 저장
4. 기존 파일 제공 시 `_workspace/`로 복사 후 해당 Phase 건너뛰기
5. 요청 범위에 기반해 **실행 모드** 결정

### Phase 2: 팀 편성 및 실행

| 순서 | 작업 | 담당자 | 의존사항 | 산출물 |
|------|------|--------|---------|--------|
| 1 | 주제 조사 | 조사자 | 없음 | `_workspace/01_research_brief.md` |
| 2 | 스크립트 작성 | 스크립트 작가 | 작업 1 | `_workspace/02_script.md` |
| 3a | 쇼노트 | 쇼노트 편집자 | 작업 1, 2 | `_workspace/03_shownotes.md` |
| 3b | 배포 패키지 | 배포 관리자 | 작업 1, 2 | `_workspace/04_distribution_package.md` |
| 4 | 프로덕션 검수 | 프로덕션 검수자 | 작업 2, 3a, 3b | `_workspace/05_review_report.md` |

작업 3a(쇼노트)와 3b(배포)는 **병렬 실행**. 둘 다 작업 2(스크립트)만 의존하므로 동시 시작 가능.

**에이전트 간 커뮤니케이션:**
- 조사자 완료 → 포인트와 사실 스크립트 작가에게, 참고자료 쇼노트 편집자에게, 트렌딩 키워드 배포 관리자에게 전달
- 스크립트 작가 완료 → 세그먼트 타임코드 쇼노트 편집자에게, 주요 인용구 배포 관리자에게 전달
- 쇼노트 편집자 완료 → 에피소드 요약과 인용구 배포 관리자에게 전달
- 검수자 모든 산출물 교차 검증, 🔴 반드시 수정할 항목 발견 시 담당자에게 수정 요청 → 재작업 → 재검증 (최대 2라운드)

### Phase 3: 통합 및 최종 산출물

검수 리포트 기반 최종 산출물 컴파일:

1. `_workspace/` 내 모든 파일 검증
2. 검수 리포트의 모든 🔴 반드시 수정할 항목 해결 확인
3. 사용자에게 최종 요약 보고:
   - 조사 브리프 — `01_research_brief.md`
   - 에피소드 스크립트 — `02_script.md`
   - 쇼노트 — `03_shownotes.md`
   - 배포 패키지 — `04_distribution_package.md`
   - 검수 리포트 — `05_review_report.md`

## 산출물

- `00_input.md` — 정리된 사용자 입력
- `01_research_brief.md` — 조사 브리프
- `02_script.md` — 팟캐스트 스크립트
- `03_shownotes.md` — 쇼노트
- `04_distribution_package.md` — 배포 패키지
- `05_review_report.md` — 검수 리포트

## 에러 처리

| 에러 유형 | 대응 전략 |
|---------|---------|
| 웹 검색 실패 | 조사자가 일반 지식으로 진행, "데이터 제한" 기록 |
| 게스트 정보 불가용 | 일반 인터뷰 프레임워크 제공, 사용자에게 게스트 정보 요청 |
| 에이전트 실패 | 1회 재시도 → 계속 실패 시 해당 산출물 제외하고 진행, 검수 리포트에 기록 |
| 🔴 검수 항목 발견 | 담당자에게 수정 요청 → 재작업 → 재검증 (최대 2라운드) |
