# 책 출판 (book-publishing)

> MoAI-Cowork v1.0.0 하네스 참고자료

## 개요

원고 편집, 표지 설계, 목차, 메타데이터, 배포 설정을 통해 전자책 출판 결과물을 협력하여 제작하는 하네스입니다.

## 전문가 역할

- **표지 디자이너**: 책의 장르, 톤, 타겟 오디언스에 맞춘 표지 컨셉 설계, Gemini 이미지 생성으로 실제 표지 제작 (타이포그래피, 색상, 구성 포함)

- **원고 편집자**: 구조 편집(장 구성, 흐름), 스타일 수정(톤 통일, 가독성), 콘텐츠 일관성 검증 (발달 편집과 문장 편집 모두 포함)

- **메타데이터 관리자**: ISBN, 책 분류(BISAC/KDC), 책 설명, 키워드, 가격, 배포 플랫폼 설정 담당 (전자책 배포에 필요한 모든 메타데이터 관리)

- **교정자**: 맞춤법, 문법, 간격, 외국어 표기, 숫자 표기, 구두점, 용어 표준화 검수 (언어 관례에 기반한 정밀 교정)

- **출판 검수자**: QA 역할, 원고-교정-표지-메타데이터 일관성 교차 검증, 출판 사양 준수, 법률 요구사항, 배포 준비도 최종 검증

## 워크플로우

### Phase 1: 준비 (오케스트레이터 직접 수행)

1. 사용자 입력에서 추출:
   - **원고**: 편집할 원고 파일
   - **장르**: 비즈니스/자기계발/픽션/에세이/기술 서적
   - **출판 목표**: 자가 출판/출판사 제출/내부 배포
   - **배포 플랫폼** (선택): 교보문고, 리디북스, Amazon KDP 등
   - **기존 파일** (선택): 편집된 원고, 표지 등

2. `_workspace/` 디렉토리 및 `_workspace/covers/` 하위디렉토리 생성
3. 입력을 정리하여 `_workspace/00_input.md`로 저장
4. 기존 파일 제공 시 `_workspace/`로 복사 후 해당 Phase 건너뛰기
5. 요청 범위에 기반해 **실행 모드** 결정

### Phase 2: 팀 편성 및 실행

| 순서 | 작업 | 담당자 | 의존사항 | 산출물 |
|------|------|--------|---------|--------|
| 1 | 원고 편집 | 원고 편집자 | 없음 | `_workspace/01_edited_manuscript.md` |
| 2a | 교정 | 교정자 | 작업 1 | `_workspace/02_proofread_report.md` |
| 2b | 표지 설계 | 표지 디자이너 | 작업 1 | `_workspace/03_cover_concept.md` |
| 3 | 메타데이터 | 메타데이터 관리자 | 작업 1, 2a | `_workspace/04_metadata.md` |
| 4 | 출판 검수 | 출판 검수자 | 작업 2a, 2b, 3 | `_workspace/05_review_report.md` |

작업 2a(교정)와 2b(표지)는 **병렬 실행**. 둘 다 작업 1(원고 편집)만 의존하므로 동시 시작 가능.

**팀 간 커뮤니케이션:**
- 원고 편집자 완료 → 용어집 교정자에게, 톤/장르/타겟 표지 디자이너에게, 제목/키워드 메타데이터 관리자에게 전달
- 교정자 완료 → 최종 표기 메타데이터 관리자에게 전달
- 표지 디자이너 ↔ 메타데이터 관리자: 정확한 제목/부제/저자 표기 상호 확인
- 검수자 모든 산출물 교차 검증, 🔴 반드시 수정할 항목 발견 시 담당자에게 수정 요청 → 재작업 → 재검증 (최대 2회)

### Phase 3: 통합 및 최종 산출물

검수 리포트 기반 최종 산출물 컴파일:

1. `_workspace/` 내 모든 파일 검증
2. 검수 리포트의 모든 🔴 반드시 수정할 항목 해결 확인
3. 사용자에게 최종 요약 보고:
   - 편집된 원고 — `01_edited_manuscript.md`
   - 교정 리포트 — `02_proofread_report.md`
   - 표지 컨셉/이미지 — `03_cover_concept.md`
   - 메타데이터/배포 — `04_metadata.md`
   - 검수 리포트 — `05_review_report.md`

## 산출물

- `00_input.md` — 정리된 사용자 입력
- `01_edited_manuscript.md` — 편집된 원고
- `02_proofread_report.md` — 교정 리포트
- `03_cover_concept.md` — 표지 컨셉/이미지
- `04_metadata.md` — 메타데이터/배포 설정
- `05_review_report.md` — 검수 리포트

## 에러 처리

| 에러 유형 | 대응 전략 |
|---------|---------|
| 원고 미제공 | "계획 모드"로 전환하여 제목/주제에서 목차와 장 개요 설계 |
| 표지 이미지 생성 실패 | 텍스트 컨셉과 프롬프트만 진행, 사용자 재시도 가능하도록 제공 |
| 에이전트 실패 | 1회 재시도 → 계속 실패 시 해당 산출물 제외하고 진행, 검수 리포트에 기록 |
| 🔴 검수 항목 발견 | 담당자에게 수정 요청 → 재작업 → 재검증 (최대 2회) |
| ISBN 미발급 | ISBN 발급 절차 가이드 제공, 나머지 메타데이터는 정상 진행 |
