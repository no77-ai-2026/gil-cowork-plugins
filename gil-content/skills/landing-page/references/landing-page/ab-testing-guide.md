# A/B 테스트 가이드

AI가 생성한 랜딩 페이지 카피의 최적안은 실제 사용자 테스트로만 확인 가능합니다.

## 우선순위 테스트 항목

| 우선순위 | 테스트 대상 | 후보 수 | 기대 효과 |
|---------|-----------|---------|----------|
| 1 | 헤드라인 | 3개 | 전환율 20-30% 차이 |
| 2 | CTA 버튼 텍스트 | 3개 | 클릭률 10-20% 차이 |
| 3 | 히어로 이미지 | 2-3개 | 이탈률 영향 |
| 4 | 섹션 순서 | 2개 | 스크롤 깊이 영향 |
| 5 | 가격 표시 방식 | 2개 | 구매 전환 영향 |

## 최소 표본 가이드

- 각 변형당 **100 클릭** 이상
- 최소 **2주** 테스트 기간
- 통계 신뢰도 **95%** 이상
- 시즌/요일 효과 제거를 위해 전체 주기 포함

## 테스트 설계

### 한 번에 1가지만 변경
```
# 좋은 테스트
A: "3개월 만에 매출 40% 성장"
B: "10,000개 팀이 선택한 생산성 도구"
→ 헤드라인만 변경, 나머지 동일

# 나쁜 테스트
A: 파란 배경 + "무료 시작" 버튼 + 짧은 카피
B: 흰 배경 + "지금 가입" 버튼 + 긴 카피
→ 3가지 동시 변경, 원인 불명
```

## 측정 지표

| 지표 | 정의 | 목표 |
|------|------|------|
| 전환율 (CVR) | CTA 클릭 / 총 방문 | 3-5% (B2B SaaS 기준) |
| 이탈률 (Bounce Rate) | 즉시 이탈 / 총 방문 | < 40% |
| 체류 시간 | 평균 페이지 체류 | > 2분 |
| 스크롤 깊이 | 75% 이상 스크롤 비율 | > 50% |

## 도구 추천

- **Google Optimize 대체**: VWO, Optimizely, AB Tasty
- **간단한 테스트**: Unbounce, Instapage (노코드)
- **직접 구현**: Next.js middleware + 쿠키 기반 분기

## AI 카피에서 A/B 후보 요청 방법

"헤드라인 후보 3가지를 각각 다른 감정 소구로 만들어줘:
1. 숫자/성과 중심
2. 공감/고통점 중심
3. 호기심/질문 중심"
