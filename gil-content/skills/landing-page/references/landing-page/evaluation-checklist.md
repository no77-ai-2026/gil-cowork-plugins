# 랜딩 페이지 평가 체크리스트

## 1. 카피 무결성 (Must-Pass)

- [ ] 헤드라인이 카피 원본과 정확히 일치
- [ ] 후기가 실명 + 직책 포함 (가공 금지)
- [ ] 숫자 주장이 구체적 ("3배" vs "훨씬")
- [ ] AI 슬롭 표현 없음 (안티패턴 참조)
- [ ] 면책/보증 문구가 정확히 구현됨

## 2. 디자인 품질 (30% 가중)

- [ ] 히어로가 전체 사이트 톤을 설정
- [ ] 이후 섹션이 히어로 디자인 언어 따름
- [ ] 색상 대비 WCAG 2.1 AA 이상 (4.5:1)
- [ ] 아이콘이 generic stock 아닌 브랜드 맞춤
- [ ] 타이포그래피 위계가 명확 (display > headline > body > caption)
- [ ] 여백이 충분하고 일관됨 (spacing scale 준수)

## 3. 전환 최적화 (25% 가중)

- [ ] 헤드라인이 10초 내 가치 전달
- [ ] CTA가 Above the Fold에 보임
- [ ] 모바일(375px)에서도 CTA 터치 가능 (최소 44x44px)
- [ ] FAQ가 상위 3개 반대 의견 해소
- [ ] 소셜 프루프가 구체적 (수치 + 실명)
- [ ] 스크롤 유도 요소 존재

## 4. 기술 검증 (20% 가중)

- [ ] Lighthouse Performance >= 80
- [ ] Lighthouse Accessibility >= 80
- [ ] 모바일 viewport 정상 (375px에서 가로 스크롤 없음)
- [ ] 모든 링크 작동 (404 없음)
- [ ] 폼 제출 테스트 완료
- [ ] 이미지 WebP, 100KB 이내
- [ ] LCP < 2.5s, CLS < 0.1

## 5. 독창성 (25% 가중)

- [ ] 템플릿 복사 느낌 아닌 브랜드 고유 디자인
- [ ] purple gradient + white card 콤보 아님
- [ ] 3개 이상 경쟁 accent 색상 사용하지 않음
- [ ] 중앙 정렬 텍스트 블록이 3줄 이내

## 평가 기준

| 등급 | 점수 | 의미 |
|------|------|------|
| PASS | >= 0.75 | 배포 가능 |
| CONDITIONAL | 0.60-0.74 | 수정 후 재평가 |
| FAIL | < 0.60 | 재작업 필요 |

Must-Pass 항목(카피 무결성) 하나라도 FAIL이면 전체 FAIL.
