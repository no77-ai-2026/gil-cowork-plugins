# 카테고리 분기: B2B / SaaS (Category — B2B / SaaS)

B2B 솔루션·SaaS 제품 상세페이지는 **B2C와 근본적으로 다름**. 별도 패턴 필수.

---

## 1. B2C vs B2B/SaaS 차이

| 축 | B2C | B2B/SaaS |
|---|---|---|
| 의사결정자 | 1명 (구매자=사용자) | **다수** (사용자·관리자·경영진·법무) |
| 검토 시간 | 5분-1일 | **2주-6개월** |
| 가격 민감도 | 절대 가격 | **ROI·TCO** (총소유비용) |
| 페이지 목적 | 구매 클릭 | **데모 신청·문의** (영업 리드) |
| 본문 길이 | 짧게·즉시 | **길게·상세** (기술 스펙·보안·SLA) |
| 톤 | 캐주얼·감성 OK | **격식·전문** |

---

## 2. SaaS 상세페이지 필수 8 섹션

```
[1] 히어로 — 핵심 USP 한 문장 + 데모 CTA + 사용 화면 GIF
[2] 사회적 증명 — 고객 로고 / 사용자 수 / 도입 기업 수
[3] 핵심 기능 3~6개 — UI 캡처 + benefit 카피
[4] 비교 / 차별화 — 경쟁 SaaS와 매트릭스 비교 (기능 ✓/✗)
[5] 가격 페이지 (별도) — Free / Pro / Enterprise tier 표
[6] 보안·인증 — SOC2·ISO27001·개인정보보호법 준수
[7] 고객 후기 / 케이스 스터디 — 도입 효과 정량 (예: "리드 3배 증가")
[8] CTA 2 (메인 + 보조) — "14일 무료" + "영업팀 문의"
```

---

## 3. 가격 페이지 — Tier 매트릭스

SaaS 가격은 별도 페이지 (또는 상세페이지 하위 섹션) 권장. Tier 3종 표준:

| Tier | 타깃 | 가격 모델 | 핵심 차이 |
|---|---|---|---|
| **Free / Trial** | 개인·신규 | 무료 | 기능 제한·사용량 제한 |
| **Pro** | 팀·스타트업 | 월 $X / 사용자 | 핵심 기능 모두 |
| **Enterprise** | 대기업 | "문의" | 보안·SLA·전담 지원·커스텀 |

가격 표시 원칙:
- **연간 결제 할인** (예: "월간 결제 대비 -20%")
- **사용자당 가격** vs **고정 가격** 명확
- **추가 비용** (스토리지·API 호출) 투명 공개
- **Enterprise 가격 비공개**라도 "문의 후 1일 내 회신" 약속

---

## 4. 복수 의사결정자 매핑

B2B는 한 페이지에 **여러 페르소나용 콘텐츠** 동시 배치.

| 페르소나 | 관심사 | 페이지 배치 |
|---|---|---|
| **사용자** | 사용성·생산성 | 히어로·기능 섹션 |
| **관리자 / IT** | 보안·통합·관리 도구 | 보안 섹션·관리 화면 |
| **경영진** | ROI·도입 효과·비즈니스 가치 | 케이스 스터디·도입 효과 |
| **법무·구매** | 약관·계약·청구 | FAQ·약관 링크 |

각 섹션마다 페르소나 표기 권장: "[관리자용]·[CTO용]" 등.

---

## 5. CTA 전략 — 리드 수집 우선

B2B는 **구매 버튼 X → 영업 리드 수집**.

### CTA 유형 5종

| 유형 | 적용 |
|---|---|
| **무료 체험 시작** | Self-service 가능 SaaS |
| **데모 신청** | 영업 주도 SaaS |
| **영업팀 문의** | Enterprise tier |
| **가격 확인 (문의)** | 비공개 가격 |
| **백서 다운로드** | 리드 자석 (이메일 수집) |

### Form 설계

- 최소 필드 우선 (이메일만)
- 점진적 프로파일링 (다음 단계에서 회사·역할 수집)
- "1분 이내" 약속 표시

---

## 6. ROI / TCO 강조

B2B 구매 의사결정의 핵심은 **ROI 계산**. 페이지에 다음 도구 권장:

- **ROI 계산기** (입력: 팀 인원·연봉 → 출력: 절감 시간·금액)
- **TCO 비교 표** (현재 도구 비용 vs 우리 도구)
- **도입 효과 정량 사례** (예: "도입 후 6개월, 회의 시간 40% 감소")

---

## 7. 보안·컴플라이언스 (B2B 필수)

| 인증 | 의미 |
|---|---|
| **SOC2 Type II** | 보안·가용성·기밀성 감사 |
| **ISO 27001** | 정보보안 국제 표준 |
| **GDPR 준수** | EU 개인정보보호 |
| **개인정보보호법 (한국)** | 한국 시장 |
| **HIPAA** | 의료 정보 (해당 시) |

페이지 footer 또는 별도 보안 페이지에 인증 로고 + 백서 링크.

---

## 8. 사회적 증명 — 로고 + 숫자

B2B는 "다른 누가 쓰는가"가 결정적.

### 표시 우선순위

```
[1] 도입 기업 로고 6~12개 (상단 배치)
[2] 사용자 수 / 도입 기업 수 ("10,000+ 팀 사용")
[3] 케이스 스터디 3~5건 (각 1페이지 분량)
[4] 후기 인용 (이름·직책·회사 명시)
[5] G2·Capterra 별점 (영문 시장)
```

---

## 9. 본문 톤 매너

- **격식 + 전문 + 직접**
- "당신·우리" 대신 "팀·조직"
- 감정 표현 최소
- 데이터·수치 풍부
- 비교 우위 명확
