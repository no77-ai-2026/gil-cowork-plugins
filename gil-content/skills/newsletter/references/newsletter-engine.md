# 뉴스레터 엔진 (newsletter-engine)

> MoAI-Cowork v1.0.0 하네스 참고자료

## 개요

큐레이션, 저작, A/B 테스트, 발송 최적화를 통해 뉴스레터 콘텐츠를 협력하여 제작하는 하네스입니다.

## 전문가 역할

- **분석가**: A/B 테스트 설계, 발송 시간 최적화, 세그먼트 전략 개발, 성과 예측 모델 구축
  - A/B 테스트 설계: 제목, 전제 텍스트, CTA, 발송 시간 등 주요 변수별 테스트 설계
  - 발송 최적화: 요일과 시간대별 최적 발송 시간 추천
  - 세그먼트 전략: 구독자 특성별 맞춤 콘텐츠 배포 전략 개발
  - 성과 예측: 업계 벤치마크 기반 KPI 예측
  - 개선 제안: 이전 뉴스레터 데이터에서 개선점 도출 (해당 시 적용)

- **카피라이터**: 큐레이션 브리프 기반 뉴스레터 본문 저작, 구독자의 끝까지 읽기와 액션 유도
  - 제목/전제 텍스트: 오픈율을 결정하는 제목과 미리보기 텍스트 작성
  - 인트로 작성: 처음 2문장 내에 독자 주의 집중
  - 본문 카피: 큐레이션된 콘텐츠를 독자 친화적 톤으로 재편집
  - CTA 설계: 클릭, 댓글, 공유 등 독자 액션 유도 CTA 배치
  - 레이아웃 설계: 제목, 불릿, 볼드 텍스트로 스캔 가능한 레이아웃 구성

- **큐레이터**: 다양한 소스에서 콘텐츠 수집, 트렌드 분석, 뉴스레터 포함 콘텐츠 선정 및 우선순위 지정
  - 소스 수집: 웹 검색으로 주제 관련 최신 기사, 블로그, 리포트, 소셜 미디어 트렌드 수집
  - 트렌드 분석: 수집 콘텐츠에서 패턴과 트렌드 파악
  - 콘텐츠 선정: 구독자 관심사와 뉴스레터 톤에 맞는 콘텐츠 우선순위 지정
  - 각도 설정: 각 콘텐츠를 뉴스레터에서 다룰 관점 정의
  - 카테고리 구성: 뉴스레터 섹션별 콘텐츠 배치

- **편집국장**: 브랜드 톤 일관성, 콘텐츠 흐름, 레이아웃 구조 최종 검수, 발행 준비 버전 최종화
  - 톤 일관성 검수: 전체 뉴스레터가 브랜드 음성과 일치하는지 확인
  - 콘텐츠 흐름 최적화: 섹션 순서, 가중치, 전환 매끄러움 검토
  - 맞춤법 및 문법 수정: 오타, 어색한 표현, 문법 오류 수정
  - 법률 검수: 저작권, 저작자 표기, 광고 공시, 구독 취소 링크 등 법률 요구사항 확인
  - 최종 편집: 분석가의 A/B 테스트 권장사항 반영한 버전 최종화

- **품질 검수자**: QA 역할, 큐레이션-저작-분석-편집 일관성 교차 검증, 갭과 모순 파악
  - 큐레이션-저작 정렬: 큐레이션 브리프의 각도가 저작에 충실히 반영되었는가?
  - 저작-분석 정렬: A/B 테스트 변형이 초안에 제대로 반영되었는가?
  - 편집-법률 정렬: 모든 법률 요구사항(구독 취소, 발신자 정보 등) 충족되었는가?
  - 링크 검증: 본문의 모든 URL이 유효하고 올바른 대상을 가리키는가?
  - 품질 체크리스트: 맞춤법, 일관성, 모바일 최적화 등

## 워크플로우

### Phase 1: 준비 (오케스트레이터 직접 수행)

1. 사용자 입력에서 추출:
   - **주제/테마**: 뉴스레터가 다룰 주제
   - **뉴스레터 정보** (선택): 브랜드 톤, 구독자 수, 발행 빈도
   - **타겟 독자** (선택): 구독자 특성, 관심사
   - **제약사항** (선택): 길이, 특정 섹션 요구사항
   - **기존 파일** (선택): 콘텐츠, 이전 호 등 사용자 제공 자료

2. `_workspace/` 디렉토리 생성
3. 입력을 정리하여 `_workspace/00_input.md`로 저장
4. 기존 파일 제공 시 `_workspace/`로 복사 후 해당 Phase 건너뛰기
5. 요청 범위에 기반해 **실행 모드** 결정

### Phase 2: 팀 편성 및 실행

| 순서 | 작업 | 담당자 | 의존사항 | 산출물 |
|------|------|--------|---------|--------|
| 1 | 콘텐츠 큐레이션 | 큐레이터 | 없음 | `_workspace/01_curation_brief.md` |
| 2 | 뉴스레터 저작 | 카피라이터 | 작업 1 | `_workspace/02_newsletter_draft.md` |
| 3 | A/B 테스트 설계 | 분석가 | 작업 1, 2 | `_workspace/03_ab_test_plan.md` |
| 4 | 최종 편집 | 편집국장 | 작업 2, 3 | `_workspace/04_editorial_final.md` |
| 5 | 품질 검수 | 품질 검수자 | 작업 2, 3, 4 | `_workspace/05_review_report.md` |

**에이전트 간 커뮤니케이션:**
- 큐레이터 완료 → 각도와 콘텐츠 카피라이터에게, 트렌딩 키워드 분석가에게 전달
- 카피라이터 완료 → A/B 변형 자료 분석가에게, 초안 편집국장에게 전달
- 분석가 완료 → 발송 최적화 및 A/B 권장사항 편집국장에게 전달
- 편집국장 완료 → 최종 버전 품질 검수자에게 전달
- 검수자 모든 산출물 교차 검증, 🔴 반드시 수정할 항목 발견 시 담당자에게 수정 요청 → 재작업 → 재검증 (최대 2라운드)

### Phase 3: 통합 및 최종 산출물

1. `_workspace/` 내 모든 파일 검증
2. 모든 🔴 반드시 수정할 항목 해결 확인
3. 사용자에게 최종 요약 보고:
   - 큐레이션 브리프 — `01_curation_brief.md`
   - 뉴스레터 초안 — `02_newsletter_draft.md`
   - A/B 테스트 계획 — `03_ab_test_plan.md`
   - 편집국장 최종 — `04_editorial_final.md`
   - 검수 리포트 — `05_review_report.md`

## 산출물

- `00_input.md` — 정리된 사용자 입력
- `01_curation_brief.md` — 큐레이션 브리프
- `02_newsletter_draft.md` — 뉴스레터 초안
- `03_ab_test_plan.md` — A/B 테스트 계획
- `04_editorial_final.md` — 편집국장 최종 버전
- `05_review_report.md` — 검수 리포트

## 에러 처리

| 에러 유형 | 대응 전략 |
|---------|---------|
| 웹 검색 실패 | 큐레이터가 사용자 제공 자료 기반 진행, "실시간 데이터 미반영" 기록 |
| 브랜드 톤 불명확 | 기본 톤(전문적 + 친근한) 적용, 편집국장이 첫 호 후 톤 개선 권장 |
| 에이전트 실패 | 1회 재시도 → 계속 실패 시 해당 산출물 제외하고 진행, 검수 리포트에 기록 |
| 🔴 검수 항목 발견 | 담당자에게 수정 요청 → 재작업 → 재검증 (최대 2라운드) |
