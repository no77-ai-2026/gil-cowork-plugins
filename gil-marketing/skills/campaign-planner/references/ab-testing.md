# A/B 테스팅 (ab-testing)

> MoAI-Cowork v1.0.0 하네스 참고자료

## 개요

마케팅 가설을 체계적으로 검증하여 최적화된 전략을 도출하는 하네스입니다.

## 전문가 역할

- **가설 설정자**: 검증할 가설 정의, 예상 효과 추정
- **테스트 설계자**: A/B 테스트 설계, 샘플 크기 계산, 통제 변수 설정
- **통계 분석가**: 결과 분석, 통계 유의성 검증, 신뢰도 계산
- **인사이트 도출자**: 테스트 결과 해석, 비즈니스 임의사항 도출
- **검수자**: QA 역할, 테스트 설계의 타당성, 결과 신뢰도 검증

## 워크플로우

### Phase 1: 준비
- 최적화 대상 선정
- 성공 지표 정의

### Phase 2: 테스트 설계 및 실행
| 순서 | 작업 | 담당자 | 산출물 |
|------|------|--------|--------|
| 1 | 가설 설정 | 설정자 | `hypothesis.md` |
| 2 | 테스트 설계 | 설계자 | `test_design.md` |
| 3 | 통계 분석 | 분석가 | `statistical_analysis.md` |
| 4 | 인사이트 도출 | 도출자 | `insights.md` |
| 5 | 최종 리포트 | 검수자 | `test_report.md` |

## 산출물
- `hypothesis.md` — 가설 및 예상 효과
- `test_design.md` — A/B 테스트 설계
- `statistical_analysis.md` — 통계 분석 결과
- `insights.md` — 인사이트 및 권고
- `test_report.md` — 최종 테스트 리포트

## 에러 처리
| 에러 유형 | 대응 전략 |
|---------|---------|
| 샘플 크기 부족 | 테스트 기간 연장 또는 신뢰도 수준 조정 |
| 외부 변수 영향 | 테스트 환경 통제 강화, 재실험 |
| 통계 유의성 미달 | 더 큰 효과 크기 검증 또는 가설 재검토 |
