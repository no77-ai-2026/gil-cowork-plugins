## I.1 명시적 한계 (작업명세서 I.1)

모든 출력물에서 다음 한계를 명확히 인지하고 필요 시 언급합니다:

| 한계 | 설명 |
|------|------|
| **메타 보고서 데이터 의존** | 본 스킬은 메타 광고관리자 보고서 데이터를 분석합니다. 픽셀 설정 오류·어트리뷰션 창 설정에 따라 데이터 정확도가 달라질 수 있습니다. 픽셀 진단은 `pixel-audit` 스킬을 활용하세요. |
| **외부 영향 미반영** | 검색 광고·오프라인 마케팅·이메일·SNS 오가닉 등 메타 광고 외 채널의 영향이 반영되지 않습니다. 교차 채널 영향을 고려한 통합 분석이 필요한 경우 별도 진단을 권장합니다. |
| **실시간 정책 변화 미반영** | 메타의 광고 알고리즘·정책·UI는 수시로 변경됩니다. 본 스킬의 분석 기준은 업로드 시점의 데이터 기반이며, 최신 정책 변화는 직접 메타 공식 문서를 확인하시기 바랍니다. |
| **인구통계 추정값** | 메타가 제공하는 연령·성별 데이터는 사용자 자기보고 정보와 메타의 추정을 결합한 값입니다. 실제 사용자 프로필과 차이가 있을 수 있습니다. |
| **분리 보고서 교차 분석 제한** | 노출지면 보고서와 인구통계 보고서를 분리 업로드한 경우, 4차원 교차 분석(광고×지면×연령×성별)에 제한이 있습니다. 완전한 교차 분석은 통합 보고서를 권장합니다. |
| **v1 보고서 업로드 한정** | 본 스킬은 v1으로 .xlsx 보고서 업로드 방식만 지원합니다. API 직접 연동은 v2에서 지원 예정입니다 (SPEC-COMMERCE-MCP-002 참조). |

## I.3 시간 예측 금지 원칙 (REQ-META-ADS-021)

본 스킬의 모든 출력물에서 시간 예측 표현을 사용하지 않습니다:

**금지 표현 예시**:
- "2-3일 내로 개선될 것입니다"
- "1주 후에 효과가 나타납니다"
- "as soon as possible"
- "빠른 시일 내에"

**허용 표현 예시**:
- "우선순위 높음" (시간 예측 아닌 중요도 표현)
- "Phase 1 → Phase 2 순서로 진행 권장" (순서 표현)
- "데이터 충분 후 판단 권장" (조건 표현)

## I.4 시간 예측 금지 예외 사항 (SPEC §6.3)

다음 3가지는 시간 예측이 아닌 분류 기준값·절감률로 허용됩니다:

### 예외 1: 수동 대비 80% 단축 (절감률 인용)

작업명세서 §10 성공 기준 직접 인용:

> "분석 1회 소요: 수동 대비 80% 단축"

이는 시간 예측이 아닌 **절감률 인용**이며, SPEC §3.8 성공 기준에 명시된 합격 기준입니다.

### 예외 2: 2개월 연속 / 3개월 연속 (데이터 분석 기준값)

모듈 6 소재 라이프사이클 노후화 판정 기준:

- 저관여: **2개월 연속** ROAS 50% 하락 → 노후화
- 중관여: **3개월 연속** ROAS 40% 하락 → 노후화

이는 **데이터 분석 기준값**으로 미래 예측이 아닙니다.

### 예외 3: Quick Wins 15분 (분류 임계값)

부록 F.5의 Quick Win 분류 기준:

> "Critical/High 심각도 + 예상 조치 시간 15분 이하 → Quick Win"

이는 **분류 임계값**으로 개별 사용자에 대한 시간 예측이 아닙니다.
